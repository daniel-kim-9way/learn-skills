## EXPLAIN

### Working Backward란?

Working Backward는 아마존에서 신제품을 개발할 때 쓰는 방식입니다. 일반적으로는 기술부터 시작해서 제품을 만드는데, Working Backward는 반대로 **완성된 결과물을 먼저 상상하고**, 거기서 거꾸로 출발합니다.

코딩을 배울 때도 똑같이 적용됩니다. 완성된 앱을 먼저 보면:
- "아, 이런 게 만들어지는구나"라는 감각이 생깁니다
- 두려움이 줄어듭니다 ("이미 있는 걸 수정하면 되잖아!")
- 무엇을 물어봐야 할지 알게 됩니다

마치 퍼즐을 맞추기 전에 완성된 그림을 먼저 보는 것과 같습니다.

### AI 협업이란?

Claude는 여러분의 지시를 받아 코드를 작성합니다. 개발 경험이 전혀 없어도 됩니다. 중요한 것은:
- **무엇을 만들고 싶은지** 말할 수 있는 것
- **결과를 보고 피드백** 줄 수 있는 것
- **막히면 Claude에게 물어보는** 것

코드를 직접 작성할 줄 몰라도 됩니다. 여러분은 기획자이자 의사결정자입니다.

## EXECUTE

### 단계 1: Claude에게 투두 앱 만들기 요청
Claude Code에서 아래와 같이 말합니다:

```
HTML, CSS, JavaScript만 사용해서 할 일 목록(Todo) 웹앱을 하나의 파일로 만들어줘.
기능: 할 일 추가, 완료 체크, 삭제
```

Claude가 파일을 만들면 브라우저로 열어서 확인합니다.

### 단계 2: 역방향 이해 (코드 설명 요청)
완성된 앱을 보면서 Claude에게 물어봅니다:

```
이 앱에서 "할 일 추가" 기능은 어떤 부분이 담당해?
코드를 몰라도 이해할 수 있게 설명해줘.
```

Claude의 설명을 들으면서 "아, 이런 구조구나"를 느끼는 것이 목표입니다. 완벽히 이해하지 않아도 됩니다.

### 단계 3: 내 서비스 아이디어로 수정
투두 앱을 자신의 서비스 아이디어에 맞게 바꿔봅니다. Claude에게 이렇게 말합니다:

```
나는 [내 서비스 아이디어]를 만들려고 해.
이 투두 앱을 비슷한 구조로 바꿔줘.
제목이랑 버튼 텍스트를 내 서비스에 맞게 수정해줘.
```

예시:
- "독서 목록 관리 앱으로 바꿔줘. 책 제목을 추가하고 읽은 것에 체크하는 앱"
- "오늘 식단 기록 앱으로 바꿔줘. 먹은 음식을 추가하고 칼로리를 메모하는 앱"
