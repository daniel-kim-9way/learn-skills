## EXPLAIN

### 홈페이지 = 서비스의 현관문

오프라인 매장도 입구가 중요합니다. 문이 잘 안 열리거나, 어두컴컴하거나, 뭘 파는 곳인지 모르면 손님이 안 들어옵니다.

**홈페이지**는 여러분 서비스의 현관문입니다. 처음 방문한 사람이 5초 안에 이 세 가지를 알 수 있어야 합니다:
1. 이게 무슨 서비스야?
2. 나에게 도움이 될까?
3. 어떻게 시작하지?

### 인증 = 누가 들어올 수 있는가

어떤 서비스는 누구나 쓸 수 있고, 어떤 서비스는 회원만 씁니다. **인증(Authentication)**은 이 결정을 구현하는 것입니다.

- **회원가입** — 새 사람이 서비스를 사용하도록 등록
- **로그인** — 등록된 사람임을 확인
- **로그아웃** — 사용 종료

왜 인증이 필요한가? 사용자별 데이터를 저장하고, 개인화된 경험을 제공하고, 결제 같은 민감한 기능을 보호하기 위해서입니다.

### Scaffolding 방식: 완성품 먼저

Block 1의 Working Backward처럼, 완성된 예시를 먼저 보고 수정합니다.

이유:
- 처음부터 만들면 막막하고 오래 걸립니다
- 완성된 예시를 보면 "이거 색만 바꾸면 되네!"가 됩니다
- 수정하면서 구조를 자연스럽게 이해합니다

## EXECUTE

### 단계 1: 완성된 예시 페이지 요청

Claude에게 요청합니다:

```
내 서비스([서비스명])의 홈페이지 예시를 만들어줘.
포함할 내용:
- 서비스 이름과 한 줄 설명 (히어로 섹션)
- 주요 기능 3가지 소개
- 시작하기 버튼

Tailwind CSS로 예쁘게 만들어줘.
```

Claude가 페이지를 만들면 브라우저에서 확인합니다.

### 단계 2: 텍스트와 색상 수정

마음에 들지 않는 부분을 Claude에게 수정 요청합니다:

```
홈페이지에서:
- 메인 색상을 파란색에서 초록색으로 바꿔줘
- 히어로 섹션 제목을 "[내가 원하는 문구]"로 바꿔줘
- 기능 설명 카드를 3개에서 4개로 늘려줘
```

수정을 반복하면서 내 서비스에 맞는 홈페이지를 만듭니다.

### 단계 3: 회원가입/로그인 기능 추가

Claude에게 인증 기능 추가를 요청합니다:

```
이 Rails 앱에 회원가입과 로그인 기능을 추가해줘.
Devise gem을 사용해줘.
네비게이션에 로그인/회원가입 버튼도 추가해줘.
```

완료 후 테스트합니다:
1. 브라우저에서 회원가입 시도
2. 로그인 테스트
3. 로그아웃 후 재로그인
