## EXPLAIN

### Memory: CLAUDE.md는 AI에게 주는 명함

Claude Code를 새로 시작할 때마다 AI는 여러분을 처음 만납니다. "저는 누구고, 이 프로젝트는 어떤 거고, 이런 방식으로 코드를 써줘"를 매번 설명해야 할까요?

**CLAUDE.md**는 이 문제를 해결합니다. 프로젝트 폴더에 `CLAUDE.md` 파일을 만들면 Claude는 대화를 시작할 때 자동으로 이 파일을 읽습니다. 마치 새 직원에게 회사 소개서를 주는 것처럼, AI에게 여러분과 프로젝트를 소개하는 명함입니다.

CLAUDE.md에 담으면 좋은 것들:
- **프로젝트 소개**: 무엇을 만들고 있는지, 누구를 위한 서비스인지
- **AI 역할 정의**: Claude가 어떤 역할로 일해줬으면 하는지
- **선호하는 방식**: 코드 스타일, 언어(한국어/영어), 설명 방식

### Skill: 반복 작업을 명령어로 저장

매번 "현재 프로젝트 상태를 요약해줘"라고 입력하기 귀찮을 때, `/status`라고만 쳐도 같은 결과가 나오도록 설정할 수 있습니다.

**Skill**은 반복되는 지시를 저장한 단축 명령어입니다. `.claude/skills/` 폴더에 `SKILL.md` 파일을 만들면 `/파일명` 형태로 호출할 수 있습니다.

자주 쓰는 Skill 예시:
- `/status` — 프로젝트 현재 상태 요약
- `/review` — 최근 변경 내용 검토
- `/deploy` — 배포 준비 체크리스트 실행

## EXECUTE

### 단계 1: CLAUDE.md 작성

프로젝트 폴더에서 Claude에게 요청합니다:

```
내 프로젝트에 CLAUDE.md를 만들어줘.
아래 내용을 담아줘:

- 프로젝트 이름: [내 서비스 이름]
- 서비스 설명: [한 줄 설명]
- 대상 사용자: [누구를 위한 서비스인지]
- 너의 역할: 비개발자인 나를 도와 이 서비스를 만드는 AI 개발 파트너
- 선호 언어: 설명은 한국어로, 코드 주석도 한국어로
- 코드 스타일: 초보자가 읽기 쉽게, 복잡한 기술보다 단순한 방법 선호
```

### 단계 2: 기억 확인

CLAUDE.md를 만든 후 Claude Code를 재시작하고 물어봅니다:

```
우리 프로젝트가 뭐였지?
```

Claude가 CLAUDE.md 내용을 바탕으로 답하면 성공입니다.

### 단계 3: /status 커스텀 스킬 생성

Claude에게 요청합니다:

```
.claude/skills/status/ 폴더를 만들고 SKILL.md 파일을 만들어줘.
이 스킬은 /status로 호출하면 프로젝트의 현재 상태를 기능 목록으로 요약해주는 스킬이야.
파일이 있으면 나열하고, 지금 동작하는 기능과 아직 없는 기능을 구분해서 보여줘.
```

만들어진 후 테스트합니다:

```
/status
```

프로젝트 상태 요약이 나오면 성공입니다.
