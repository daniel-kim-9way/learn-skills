## EXPLAIN

### Rails = 건물 골조

집을 지을 때 벽돌 하나하나부터 쌓지 않아도 됩니다. 이미 벽, 천장, 배관이 갖춰진 **아파트 골조**를 받아서 인테리어만 하면 되죠.

**Ruby on Rails(레일즈)**는 웹 서비스 개발의 골조입니다.

웹 서비스에 필요한 것들 — 사용자 인증, 데이터베이스 연결, 페이지 라우팅, 보안 처리 — 이런 기반 작업을 Rails가 이미 해뒀습니다. 덕분에 여러분은 **내 서비스만의 독특한 기능**에 집중할 수 있습니다.

### MVC: 세 가지 역할

Rails는 MVC 패턴으로 코드를 구성합니다. 비개발자도 이해할 수 있는 방식으로 설명하면:

| 역할 | 기술 용어 | 쉬운 설명 | 예시 |
|------|----------|-----------|------|
| 데이터 관리 | Model | "창고 담당" — 데이터를 저장하고 꺼내는 역할 | 사용자 정보, 게시글 저장 |
| 화면 표시 | View | "진열 담당" — 데이터를 예쁘게 보여주는 역할 | HTML 페이지, 버튼, 폼 |
| 요청 처리 | Controller | "안내 담당" — 사용자 요청을 받아 창고-진열을 연결 | "이 버튼 누르면 저 데이터 보여줘" |

비유: 식당에서 고객(사용자)이 주문하면 → 웨이터(Controller)가 받아서 → 주방(Model)에 전달하고 → 요리가 완성되면 테이블(View)에 내놓는 것입니다.

### Rails의 장점

- **규칙이 명확해서** Claude에게 "이거 만들어줘"라고 말하면 Claude가 어디에 뭘 만들지 압니다
- **이미 많은 게 준비되어** 있어서 처음부터 만들 필요가 없습니다
- **한국에서도 많이 써서** 한국어 자료와 커뮤니티가 풍부합니다

## EXECUTE

### 단계 1: Claude에게 Rails 프로젝트 생성 요청

Claude Code에서 요청합니다:

```
Rails 프로젝트를 만들어줘.
프로젝트 이름은 [내 서비스 영문 이름]으로 해줘.
SQLite를 DB로 사용하고, Tailwind CSS를 포함해줘.
```

Claude가 아래와 비슷한 명령어를 실행합니다:
```bash
rails new [프로젝트명] --database=sqlite3 --css=tailwind
cd [프로젝트명]
```

이 과정에 몇 분이 걸릴 수 있습니다.

### 단계 2: 서버 실행 및 확인

Claude에게 서버 실행을 요청합니다:

```
서버 실행해줘
```

또는 직접:
```bash
bin/rails server
```

브라우저에서 `http://localhost:3000`을 열면 Rails 시작 화면이 보입니다. "Yay! You're on Rails!"가 보이면 성공입니다.

### 단계 3: 프로젝트 구조 투어

Claude에게 구조 설명을 요청합니다:

```
이 프로젝트 폴더 구조를 비개발자도 이해할 수 있게 설명해줘.
어떤 폴더에 뭘 넣는지만 알려줘.
```

주요 폴더만 기억하면 됩니다:
- `app/models/` — 데이터 관련 파일 (Model)
- `app/views/` — 화면 파일 (View)
- `app/controllers/` — 요청 처리 파일 (Controller)
