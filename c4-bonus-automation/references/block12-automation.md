## EXPLAIN

### 서비스 빌딩 스킬 = 업무 자동화 스킬

C3~C4를 통해 배운 것들을 돌아봅시다:
- Claude Code로 아이디어를 빠르게 구현
- CLAUDE.md로 AI에게 컨텍스트 주기
- 커스텀 Skill로 반복 작업 자동화
- MCP로 데이터베이스, 캘린더 등 외부 도구 연결
- Multi-Agent로 복잡한 작업 병렬 처리

이 스킬들은 서비스를 만드는 데만 쓰이는 게 아닙니다. **일상 업무에도 그대로 적용됩니다.**

예시:
- 매주 팀 보고서를 작성하는 업무 → Skill로 자동화
- 고객 이메일을 분류하고 답변 초안 작성 → MCP(Gmail) + Claude
- 회의록 → 할 일 목록 변환 → Multi-Agent로 자동화
- 경쟁사 가격 모니터링 → MCP(웹 스크래핑) + 알림

### MCP + Skills = 자동화 툴킷

**MCP(Model Context Protocol)**는 Claude가 외부 도구와 연결하는 규격입니다. 지금까지 배운 MCP 연결 경험을 활용합니다.

**업무 자동화 스택**:

```
트리거 (언제 실행할지)
    ↓
MCP (어떤 데이터를 가져올지)
    ↓
Claude (어떻게 처리할지)
    ↓
Skill (어떻게 저장/전달할지)
```

예시 (주간 보고서 자동화):
```
매주 금요일 오후 5시 (트리거)
    ↓
Git MCP: 이번 주 커밋 목록 가져오기
캘린더 MCP: 이번 주 미팅 기록 가져오기
    ↓
Claude: "이 정보를 바탕으로 팀장이 읽기 쉬운 주간 보고서 작성"
    ↓
/weekly-report Skill: 노션/슬랙으로 보내기
```

### 자동화할 수 있는 업무 유형

시간이 많이 걸리지만 중요도가 낮은 반복 업무:

| 업무 유형 | 자동화 방법 |
|----------|------------|
| 정기 보고서 | Skill + MCP(문서 도구) |
| 이메일 분류/답변 초안 | MCP(Gmail) + Claude |
| 미팅 요약 | 녹취 → Claude 요약 |
| 데이터 수집/정리 | MCP(웹/DB) + Claude |
| 소셜 미디어 콘텐츠 | content-digest + Claude |

## EXECUTE

### 단계 1: 자동화 아이디어 선정
가장 반복적이고 지루한 업무 하나를 고릅니다.

Claude에게 도움을 요청합니다:
```
내 업무 중 자동화하면 좋을 것들을 찾아줘.

내 업무 패턴:
- 매일/매주 반복하는 것: [설명]
- 시간이 많이 걸리는 것: [설명]
- AI가 도와줄 수 있을 것 같은 것: [설명]

자동화 가능성을 높음/중간/낮음으로 평가하고, 가장 쉽게 시작할 수 있는 것을 추천해줘.
```

### 단계 2: Template+STUB으로 자동화 스킬 제작
선정한 자동화를 **Template+STUB 방식**으로 구현합니다:

1. 스킬 구조 설계:
```
.claude/skills/[자동화이름]/SKILL.md를 만들어줘.
이 스킬은 /[자동화이름]으로 호출하면:

입력: [무엇을 받는지]
처리: [Claude가 무엇을 하는지]
출력: [어디에 어떻게 저장/전달하는지]

필요한 MCP: [어떤 외부 도구가 필요한지]
```

2. STUB이 있는 템플릿 만들기:
```
이 스킬에서 사용할 템플릿 파일을 만들어줘.
[STUB: ...]로 표시된 부분은 Claude가 매번 채워넣을 것들이야.
```

3. 테스트 실행:
```
/[자동화이름]
```

### 단계 3: GitHub에 공유
완성한 자동화 스킬을 GitHub에 올려서 다른 사람도 쓸 수 있게 합니다:

```
내 자동화 스킬을 GitHub에 올릴 준비를 해줘.

1. README.md 생성: 이 스킬이 무엇인지, 어떻게 쓰는지 설명
2. 설치 방법 작성: MCP 설정, 환경 변수 안내
3. 예시 실행 결과 추가

그리고 git commit + push 해줘.
```

GitHub에 올린 후 Vuild Showcase나 커뮤니티에 공유합니다.
