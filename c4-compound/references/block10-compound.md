## EXPLAIN

### 학습 복리란?

복리(複利)를 아시나요? 투자에서 복리는 이자에 이자가 붙는 구조입니다.

- 단리: 매년 10% → 10년 후 200%
- 복리: 매년 10% → 10년 후 259%

처음에는 차이가 작지만 시간이 지날수록 격차가 커집니다.

학습도 똑같습니다:

**단순 학습**: 새로운 것을 배우고 → 잊어버리고 → 다시 배우고...

**복리 학습**: 새로운 것을 배우고 → 기존 지식과 연결하고 → 더 빨리 다음을 배우고 → 또 연결하고...

비결은 **연결**입니다. 새로운 지식을 기존 지식과 연결할수록 학습 속도가 기하급수적으로 빨라집니다.

### content-digest: 외부 지식 흡수 도구

좋은 블로그 글, 유튜브, 책을 보고 "와, 좋다!"라고 느끼지만 3일 후에 내용이 기억 안 나는 경험, 있으신가요?

**content-digest**는 이 문제를 해결합니다:

1. 콘텐츠(URL, 텍스트)를 Claude에게 전달
2. Claude가 핵심 내용을 추출
3. 내 기존 지식과 어떻게 연결되는지 분석
4. 실천 가능한 인사이트로 변환
5. 내 compound 문서에 추가

이렇게 하면 읽은 모든 것이 내 지식 체계에 쌓입니다.

### compound 문서: 나만의 지식 백과사전

compound 문서는 내가 배운 모든 것을 연결한 **살아있는 문서**입니다.

특징:
- 시간이 지날수록 풍부해집니다
- 항목들이 서로 연결됩니다
- Claude가 새 작업을 할 때 참고 자료로 활용합니다
- 나의 사고 방식이 담겨 있습니다

예시 구조:
```
compound.md
├── 서비스 개발 인사이트
│   ├── 사용자 경험 원칙
│   └── 기술 선택 기준
├── 비즈니스 학습
│   ├── 가격 결정 원칙
│   └── 고객 인터뷰 방법
└── AI 협업 패턴
    ├── 효과적인 프롬프트
    └── Multi-Agent 활용
```

### content-digest 체험
좋아하는 글, 블로그 포스트, 또는 최근 읽은 내용을 Claude에게 전달합니다:

```
content-digest를 실행해줘.

분석할 콘텐츠:
[URL 또는 텍스트 붙여넣기]

내 컨텍스트:
- 만들고 있는 서비스: [서비스 설명]
- 현재 관심사: [관심 있는 주제]

추출해줄 것:
1. 핵심 인사이트 3개
2. 내 서비스에 어떻게 적용할 수 있는지
3. 기존에 내가 알던 것과 어떻게 연결되는지
```

### Multi-Agent로 compound 문서 생성
여러 에이전트가 동시에 기존 지식을 정리합니다:

```
Multi-Agent를 사용해서 내 compound 문서를 만들어줘.

Phase 1 (병렬):
- 에이전트 1: C3~C4에서 배운 기술 인사이트 정리
- 에이전트 2: 서비스 개발 중 발견한 비즈니스 인사이트 정리
- 에이전트 3: AI 협업에서 효과적이었던 패턴 정리

Phase 2:
- 에이전트 4: 위 내용을 연결해서 compound.md 생성
  저장 위치: docs/compound.md
```

완성된 `compound.md`를 열어서 내용을 확인하고 보완합니다.

정답:
- 1번째 질문 → "새 지식을 기존 지식과 연결하기 때문에"
- 2번째 질문 → "외부 콘텐츠의 인사이트를 내 기존 지식과 연결해서 체계에 흡수하는 것"

해설:
- 복리 효과: 단순히 많이 읽는 것보다, 새로 배운 것을 기존에 알던 것과 연결할 때 학습이 가속됩니다. "아, 이게 저번에 배운 그 개념이랑 같은 원리네!"라는 순간이 바로 복리가 일어나는 순간입니다.
- content-digest: 단순 요약은 정보를 줄여주지만 연결은 만들어주지 않습니다. content-digest의 가치는 외부 콘텐츠를 내 맥락(서비스, 목표, 기존 지식)에 연결해서 실천 가능한 인사이트로 변환하는 데 있습니다.
