# C4 Block 1: 디자인 — 신뢰를 만드는 첫인상

## 학습 목표
- 디자인이 "예쁨"이 아니라 "신뢰"를 만드는 수단임을 이해한다
- Tailwind CSS의 유틸리티 클래스 방식을 경험한다
- Claude에게 디자인 개선을 요청하고 반복적으로 다듬는 방법을 익힌다

## 사전 조건
- `/c4-warmup` 완료 (서비스 실행 및 컨텍스트 회복)

## STOP PROTOCOL

이 스킬은 2-Phase STOP Protocol을 따릅니다.

### Phase A (Turn 1)
1. references/block1-design.md의 EXPLAIN 섹션을 읽고 설명한다
   - 디자인 = 신뢰, 아름다움이 아님
   - Clean vs Impressive: 단순함이 더 효과적인 이유
   - Tailwind CSS가 무엇인지 (유틸리티 클래스 방식)
2. references/block1-design.md의 EXECUTE 섹션을 따라 실행한다
   - Before/After 비교 확인
   - Claude에게 Tailwind 디자인 적용 요청
   - 반복 다듬기
3. **STOP** — 여기서 멈추고 학생의 응답을 기다린다. AskUserQuestion을 사용하지 않는다.
4. 멈추기 전 출력: "디자인 개선을 마쳤습니다. '다음'을 입력하면 퀴즈로 넘어갑니다."

### Phase B (Turn 2)
1. references/block1-design.md의 QUIZ 섹션 퀴즈 출제
2. 학생 답변에 피드백한다 (오답 해설 포함)
3. 다음 안내:
