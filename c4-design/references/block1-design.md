## EXPLAIN

### 디자인은 예쁨이 아니라 신뢰다

처음 서비스를 만들면 "기능이 먼저고, 디자인은 나중에"라고 생각하기 쉽습니다. 하지만 실제 사용자들은 다르게 행동합니다.

사람은 **0.05초** 안에 웹사이트의 신뢰도를 판단합니다. 기능을 써보기도 전에 "이거 믿을 수 있을까?"를 눈으로 먼저 판단합니다. 어떤 조사에서는 사용자의 75%가 첫인상만으로 서비스의 신뢰도를 결정한다고 합니다.

명함을 생각해봅시다. 내용은 같아도, 구겨진 종이에 적힌 명함과 깔끔한 카드 명함은 다른 인상을 줍니다. 서비스도 마찬가지입니다.

### Clean vs Impressive

"인상적인 디자인"과 "깔끔한 디자인" 중 어느 것이 더 효과적일까요?

| | Impressive | Clean |
|--|------------|-------|
| 느낌 | 화려하고 복잡함 | 단순하고 정돈됨 |
| 사용자 반응 | "와, 멋있다" → 어디를 눌러야 하지? | "바로 이해된다" → 클릭 |
| 목적 | 인상 주기 | 행동 유도 |

비즈니스 서비스에서는 대부분 **Clean이 이깁니다.** 사용자가 "무엇을 해야 하는지" 바로 알 수 있는 디자인이 전환율을 높입니다.

핵심 원칙 3가지:
1. **여백** — 빽빽하지 않게. 숨 쉴 공간을 주세요
2. **계층** — 크고 굵게 vs 작고 얇게. 중요도를 눈으로 표현하세요
3. **일관성** — 색상 1~2개, 폰트 1~2개. 통일감이 신뢰를 만듭니다

### Tailwind CSS: 유틸리티 클래스 방식

CSS는 웹의 화장품입니다. 글자 색, 버튼 크기, 간격 등을 결정합니다.

기존 CSS는 이름을 짓고 따로 파일을 만들어야 했습니다. 번거롭고 어려웠죠.

Tailwind CSS는 다릅니다. 미리 만들어진 수천 개의 "유틸리티 클래스"를 조합해서 사용합니다. 레고 블록처럼요.

```html
<!-- 기존 CSS -->
<button class="submit-button">제출</button>
/* submit-button 클래스를 별도 CSS 파일에 정의해야 함 */

<!-- Tailwind CSS -->
<button class="bg-blue-500 text-white px-4 py-2 rounded">제출</button>
<!-- 클래스 이름이 곧 스타일 설명 -->
```

Claude에게 "Tailwind로 이 페이지를 개선해줘"라고 하면 알아서 적절한 클래스를 적용해줍니다. 코드를 몰라도 됩니다.

## EXECUTE

### 단계 1: Before 스크린샷 찍기
지금 서비스 화면을 캡처합니다 (비교를 위해). 브라우저에서:
- `http://localhost:3000` 접속
- 메인 페이지를 화면 캡처 (Windows: Win+Shift+S, Mac: Cmd+Shift+4)

### 단계 2: Claude에게 Tailwind 디자인 적용 요청
Claude Code에서 아래와 같이 요청합니다:

```
메인 페이지 디자인을 Tailwind CSS로 개선해줘.
목표: 신뢰감 있고 깔끔한 느낌
원칙:
- 여백을 충분히 줘서 숨 쉬는 느낌
- 중요한 버튼은 파란색 계열로 강조
- 텍스트 크기로 계층(중요도)을 표현
- 색상은 2가지 이하로 제한
```

Claude가 코드를 수정하면 브라우저를 새로고침해서 확인합니다.

### 단계 3: 반복 다듬기
마음에 안 드는 부분을 콕 집어서 피드백합니다:

```
좋은데, 헤더가 너무 크게 느껴져. 조금 줄여줘.
```

```
버튼이 눈에 잘 안 띄어. 더 눈에 띄게 해줘.
```

2~3번 피드백을 반복하며 마음에 드는 디자인을 찾습니다.
