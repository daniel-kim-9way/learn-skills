# C4 Block 2: 랜딩 페이지 — 서비스의 첫 문

## 학습 목표
- 랜딩 페이지의 5섹션 구조와 각 섹션의 역할을 이해한다
- 템플릿의 STUB을 채워 나만의 랜딩 페이지를 만들 수 있다
- 모바일에서도 잘 보이는지 반응형을 확인하는 방법을 익힌다

## 사전 조건
- `/c4-design` 완료 (Tailwind CSS 디자인 개선 경험)

## STOP PROTOCOL

이 스킬은 2-Phase STOP Protocol을 따릅니다.

### Phase A (Turn 1)
1. references/block2-landing.md의 EXPLAIN 섹션을 읽고 설명한다
   - 랜딩 페이지의 목적 (설득의 여정)
   - 5섹션 구조와 각 섹션의 역할
2. references/block2-landing.md의 EXECUTE 섹션을 따라 실행한다
   - templates/landing-page.md 템플릿 복사
   - STUB을 Claude의 도움으로 채우기
   - 반응형 확인
3. **STOP** — 여기서 멈추고 학생의 응답을 기다린다. AskUserQuestion을 사용하지 않는다.
4. 멈추기 전 출력: "랜딩 페이지를 완성했습니다. '다음'을 입력하면 퀴즈로 넘어갑니다."

### Phase B (Turn 2)
1. references/block2-landing.md의 QUIZ 섹션 퀴즈 출제
2. 학생 답변에 피드백한다 (오답 해설 포함)
3. 다음 안내:
   - "웹에서 '완료' 버튼을 눌러주세요."
   - "다음 블록: `/c4-ai-feature`"
