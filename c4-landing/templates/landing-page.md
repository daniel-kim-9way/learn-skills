# 랜딩 페이지 템플릿

이 템플릿을 Claude에게 주고, STUB 부분을 서비스에 맞게 채워달라고 하세요.

---

## STUB 1: Hero 섹션

```
서비스 이름: [STUB: 서비스 이름]
헤드라인: [STUB: 한 줄로 서비스 핵심 가치를 설명하세요. "누구를 위한 무엇" 형식 권장]
서브헤드라인: [STUB: 헤드라인을 뒷받침하는 1~2문장. 어떻게 도움이 되는지 구체적으로]
CTA 버튼 텍스트: [STUB: 행동을 유도하는 버튼 문구. 예: "무료로 시작하기", "지금 써보기"]
```

**Hero 작성 공식**: [누구를] [어떤 문제에서] [어떻게 구해주는가]

---

## STUB 2: Problem 섹션

```
섹션 제목: [STUB: 공감을 자아내는 제목. 예: "이런 경험 있으신가요?"]
문제 1: [STUB: 타겟 사용자가 겪는 첫 번째 불편함]
문제 2: [STUB: 타겟 사용자가 겪는 두 번째 불편함]
문제 3: [STUB: 타겟 사용자가 겪는 세 번째 불편함]
공감 마무리 문구: [STUB: "저희도 똑같이 느꼈습니다" 같은 공감 문구]
```

**작성 팁**: 사용자가 "맞아, 나 이거 불편했어!"라고 고개를 끄덕이게 만드세요.

---

## STUB 3: Solution 섹션

```
섹션 제목: [STUB: 해결책을 소개하는 제목. 예: "[서비스명]이 해결합니다"]
해결책 설명: [STUB: 어떻게 위의 문제들을 해결하는지 2~3문장으로 설명]
차별점: [STUB: 다른 대안(경쟁사, 기존 방법)과 무엇이 다른지]
```

**작성 팁**: 기술 설명 말고, 사용자가 얻는 결과를 중심으로 쓰세요.

---

## STUB 4: Features 섹션

```
섹션 제목: [STUB: 예: "주요 기능", "이런 걸 할 수 있어요"]

기능 1:
  아이콘/이모지: [STUB: 기능을 나타내는 이모지]
  제목: [STUB: 기능 이름, 3~5단어]
  설명: [STUB: 이 기능으로 무엇을 할 수 있는지 1~2문장]

기능 2:
  아이콘/이모지: [STUB: 기능을 나타내는 이모지]
  제목: [STUB: 기능 이름, 3~5단어]
  설명: [STUB: 이 기능으로 무엇을 할 수 있는지 1~2문장]

기능 3:
  아이콘/이모지: [STUB: 기능을 나타내는 이모지]
  제목: [STUB: 기능 이름, 3~5단어]
  설명: [STUB: 이 기능으로 무엇을 할 수 있는지 1~2문장]
```

**작성 팁**: 기능(무엇)보다 혜택(왜 좋은지)을 강조하세요.

---

## STUB 5: CTA 섹션

```
섹션 제목: [STUB: 마지막으로 행동을 촉구하는 제목. 예: "지금 시작할 준비가 됐나요?"]
설명 문구: [STUB: 시작하기 쉬움을 강조. 예: "5분이면 설정 완료, 신용카드 불필요"]
메인 CTA 버튼: [STUB: 주요 행동 버튼 문구]
서브 CTA (선택): [STUB: 부담 없는 대안 행동. 예: "먼저 데모 보기"]
```

**작성 팁**: 마지막 CTA는 Hero의 CTA와 연결되어야 합니다. 일관성이 신뢰를 만듭니다.

---

## Claude에게 전달하는 방법

```
위 템플릿의 모든 [STUB: ...] 부분을 채워서 완성된 랜딩 페이지 HTML 파일을 만들어줘.

내 서비스 정보:
- 서비스 이름: [여기 입력]
- 대상 사용자: [여기 입력]
- 핵심 기능: [여기 입력]
- 해결하는 문제: [여기 입력]

Tailwind CSS를 사용하고, 반응형으로 만들어줘.
색상은 [원하는 색상 계열] 계열로 해줘.
```
