## EXPLAIN

### 완벽은 완료의 적이다

서비스를 만들다 보면 이런 생각이 듭니다:
- "이 기능만 더 넣으면..."
- "디자인이 아직 좀 아쉬운데..."
- "버그가 하나 더 있는 것 같아서..."

이러다가 오픈을 계속 미루는 분들이 많습니다. 하지만 실제로 사용해보지 않으면 무엇이 진짜 중요한지 알 수 없습니다.

**페이스북의 초창기 슬로건**: "Done is better than perfect" (완료가 완벽보다 낫다)

첫 고객 10명이 서비스를 써보고 "이게 불편해요"라고 하면 그때 고치면 됩니다. 혼자 상상하는 "완벽"보다 실제 사용자의 피드백이 훨씬 가치 있습니다.

### 5개 필수 항목 vs Nice-to-have

오픈 전 **반드시** 확인해야 할 것:
1. **서비스가 돌아간다** — 핵심 기능이 오류 없이 작동
2. **결제가 된다** (결제 서비스라면) — 실제 돈이 오고 가는 흐름 확인
3. **개인정보 처리방침이 있다** — 법적 의무
4. **연락처가 있다** — 문제가 생기면 사용자가 연락할 방법
5. **보안 체크리스트 통과** — 기본 보안 항목 완료

**나중에 해도 되는 것**:
- 완벽한 디자인
- 모든 기능 완성
- SEO 최적화
- 퍼포먼스 튜닝
- 다국어 지원

처음 오픈은 MVP(최소 기능 제품)입니다. 핵심만 있으면 충분합니다.

### LAUNCH의 진짜 의미

오픈은 끝이 아닙니다. **새로운 시작**입니다.

오픈 → 실제 사용자 피드백 → 개선 → 더 많은 사용자 → 더 많은 피드백 → ...

이 사이클을 시작하는 것이 LAUNCH입니다.

## EXECUTE

### 단계 1: LAUNCH 체크리스트 실행
`skills/c4-launch/templates/launch-checklist.md`를 열어서 항목을 하나씩 확인합니다.

Claude에게 도움을 요청합니다:
```
launch-checklist.md의 항목들을 내 서비스에서 하나씩 확인해줘.
각 항목이 완료됐는지 확인하고, 미완료 항목은 지금 해결해줘.
```

### 단계 2: 런치 스킬 만들기
앞으로 새 기능을 추가할 때마다 쓸 수 있는 `/launch` 커스텀 스킬을 만듭니다:

```
.claude/skills/launch/ 폴더에 SKILL.md를 만들어줘.
이 스킬은 /launch로 호출하면:
1. launch-checklist.md의 10개 항목을 자동으로 점검
2. 미완료 항목을 목록으로 보여줌
3. 모두 완료되면 "READY TO LAUNCH!" 출력
```

### 단계 3: LAUNCH!
모든 체크리스트가 통과되면:

1. 서비스 URL을 테스트로 방문해봅니다
2. 가장 중요한 흐름 (가입 → 핵심 기능 → 결제)을 처음부터 끝까지 한 번 더 확인
3. SNS, 커뮤니티, 주변에 공유합니다!

첫 공유 메시지 템플릿:
```
안녕하세요! 제가 만든 서비스를 소개합니다.

[서비스 이름]: [한 줄 설명]
링크: [서비스 URL]

[어떤 사람에게] [어떤 문제를] 해결해주는 서비스입니다.
사용해보시고 피드백 주시면 감사하겠습니다 🙏
```
