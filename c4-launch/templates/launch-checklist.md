# LAUNCH 체크리스트

서비스를 오픈하기 전 이 10개 항목을 확인하세요.
Claude에게 `/launch` 명령으로 자동 점검을 요청할 수 있습니다.

---

## 필수 항목 (오픈 전 반드시 완료)

- [ ] **1. 핵심 기능 동작 확인**
  - 사용자가 가입하고 로그인할 수 있다
  - 서비스의 핵심 기능이 오류 없이 동작한다
  - 모바일(스마트폰)에서도 기본 기능이 사용 가능하다

- [ ] **2. 결제 흐름 확인** (결제 기능이 있는 경우)
  - 결제 버튼 클릭 → 결제창 → 결제 완료 전 과정이 동작한다
  - 테스트 결제가 아닌 실제 결제로 전환됐다
  - 결제 성공/실패 시 올바른 안내가 표시된다

- [ ] **3. 개인정보 처리방침**
  - 개인정보 처리방침 페이지가 있다 (하단 링크 등)
  - 어떤 정보를 수집하고 어떻게 사용하는지 명시되어 있다
  - 회원가입 시 동의를 받는 절차가 있다

- [ ] **4. 연락처 및 고객지원**
  - 사용자가 문의할 수 있는 이메일 또는 채널이 있다
  - 하단(footer) 또는 '문의' 페이지에 연락처가 표시된다

- [ ] **5. 보안 기본 항목**
  - 환경 변수(.env)가 GitHub에 올라가지 않는다
  - HTTPS가 적용되어 있다
  - 관리자 페이지에 인증이 걸려 있다

---

## 권장 항목 (오픈 후 개선 가능)

- [ ] **6. 오류 모니터링**
  - 서비스 오류 발생 시 알림을 받을 방법이 있다 (이메일, Slack 등)
  - 오류 로그를 확인할 수 있다

- [ ] **7. 분석 도구 연동**
  - 방문자 수, 사용 패턴을 확인할 수 있는 도구가 있다
  - Google Analytics 또는 유사 도구 설치

- [ ] **8. SEO 기본 설정**
  - 페이지 제목(title)과 설명(meta description)이 있다
  - 대표 이미지(og:image)가 설정되어 있다

- [ ] **9. 로딩 속도**
  - 메인 페이지가 3초 이내에 로딩된다
  - 이미지가 적절히 최적화되어 있다

- [ ] **10. 백업**
  - 데이터베이스 자동 백업이 설정되어 있다
  - 서비스 코드가 GitHub에 최신 상태로 저장되어 있다

---

## Claude 자동 점검 요청 방법

```
launch-checklist.md를 기반으로 내 서비스를 점검해줘.
각 항목을 확인하고:
- ✅ 완료
- ⚠️ 부분 완료 (무엇이 부족한지 설명)
- ❌ 미완료 (어떻게 해결할지 안내)

로 분류해줘.
```
