## EXPLAIN

### 결제는 어떻게 일어나는가?

"카드 번호를 입력하면 돈이 어떻게 내 통장으로 들어오지?" 사실 이 과정은 꽤 복잡합니다. 하지만 결제 게이트웨이가 대부분을 대신 처리해줍니다.

결제 흐름을 택배 시스템으로 설명하면:

```
고객 (카드 정보 입력)
   ↓
결제 게이트웨이 (Toss Payments) — "이 택배 받아서 처리할게요"
   ↓ 카드사와 협상
카드사 (결제 승인)
   ↓
결제 게이트웨이 → 나의 서비스 "승인됐어요!" 알림
   ↓
나의 서비스: 주문 완료 처리
   ↓ (며칠 후)
내 통장에 정산 입금
```

직접 카드사와 계약하고, 보안 인증을 받고, 정산 시스템을 구축하면 수억 원과 1년 이상이 필요합니다. 결제 게이트웨이는 이 모든 걸 대신해줍니다. 수수료는 보통 결제 금액의 2~3%입니다.

### 왜 Toss Payments인가?

한국에서 많이 쓰는 결제 게이트웨이들: 이니시스, KG모빌리언스, 나이스페이, 토스페이먼츠

Toss Payments를 추천하는 이유:
- **개발자 친화적 문서**: 설명이 가장 쉽고 예제가 풍부합니다
- **테스트 모드**: 실제 돈 없이도 결제 전 과정을 테스트할 수 있습니다
- **빠른 심사**: 개인 사업자도 빠른 심사가 가능합니다
- **국내 최적화**: 카카오페이, 네이버페이, 삼성페이 모두 지원

### 테스트 모드 vs 실제 모드

| | 테스트 모드 | 실제 모드 |
|--|------------|----------|
| 돈 | 가짜 돈 (테스트용 카드번호) | 실제 돈 |
| 목적 | 기능 확인, 버그 수정 | 실제 서비스 운영 |
| 전환 | 테스트 키 → 실제 키 교체 | |

테스트 카드번호: `4330000000000000` (만료: 아무 미래 날짜, CVC: 아무 숫자)

⚠️ **중요**: 실제 모드로 전환하기 전에 반드시 사업자 등록 또는 개인 계좌 인증을 완료해야 합니다.

## EXECUTE

### 단계 1: Toss Payments 테스트 계정 등록
1. https://developers.tosspayments.com 접속
2. "시작하기" 또는 "로그인" → 회원가입
3. 대시보드에서 "테스트 키" 확인:
   - 클라이언트 키 (test_ck_로 시작)
   - 시크릿 키 (test_sk_로 시작)
4. 두 키를 안전하게 복사 (`.env` 파일에 보관)

### 단계 2: Claude가 결제 기능 연동
Claude Code에서 아래와 같이 요청합니다:

```
Toss Payments를 사용해서 결제 기능을 연동해줘.
테스트 모드로 구현해줘.

요구사항:
- 결제 버튼을 누르면 Toss 결제창이 뜨는 기능
- 결제 성공 시 주문 완료 페이지로 이동
- 결제 실패 시 오류 메시지 표시

환경 변수:
- TOSS_CLIENT_KEY: 클라이언트 키
- TOSS_SECRET_KEY: 시크릿 키

공식 문서: https://docs.tosspayments.com
```

### 단계 3: 테스트 결제 실행
구현이 완료되면 실제로 테스트 결제를 해봅니다:

1. 서비스에서 결제가 필요한 화면으로 이동
2. 결제 버튼 클릭 → Toss 결제창 확인
3. 테스트 카드 입력:
   - 카드 번호: `4330000000000000`
   - 유효기간: 임의의 미래 날짜 (예: 12/28)
   - CVC: 임의 3자리 (예: 123)
4. 결제 완료 → 성공 페이지 확인
5. Toss 대시보드에서 테스트 결제 내역 확인
