## EXPLAIN

### Multi-Agent란?

지금까지 Claude 하나에게 작업을 맡겼습니다. 하지만 큰 프로젝트에서는 한 명보다 여러 명이 동시에 일하는 게 빠릅니다. AI도 마찬가지입니다.

**Multi-Agent**는 여러 AI 에이전트가 협력해서 작업하는 방식입니다.

주방을 예로 들면:
- **단일 에이전트**: 셰프 혼자 채소 자르고, 끓이고, 담아서 서빙
- **Multi-Agent**: 셰프(총괄) + 수셰프(조리) + 서버(서빙)가 동시에 진행

### 두 가지 패턴: 병렬 vs 순차

**병렬(Parallel) 패턴** — 동시에 실행

```
에이전트 A: 버그 수정
에이전트 B: 문서 작성    → 동시에 시작 → 모두 완료 → 병합
에이전트 C: 테스트 실행
```

독립적인 작업에 적합합니다. 예: 3개 페이지를 동시에 개선

**순차(Sequential) 패턴** — 앞 결과를 다음에 넘김

```
에이전트 A: 코드 분석 → 결과 전달
                              ↓
에이전트 B: 결과 바탕으로 리팩토링 → 결과 전달
                                              ↓
에이전트 C: 테스트 실행 → 통과 확인
```

앞 단계의 결과가 다음 단계에 필요할 때 사용합니다.

### 2-Phase 패턴

가장 실용적인 패턴입니다:

```
Phase 1 (병렬): 여러 에이전트가 동시에 독립 작업 실행
       ↓
Phase 2 (순차): 결과를 종합해서 최종 처리
```

**세션 마무리 스킬**에 딱 맞습니다:
- Phase 1: 에이전트 A(변경 사항 정리) + 에이전트 B(테스트 실행) + 에이전트 C(TODO 목록 갱신) 동시 실행
- Phase 2: 에이전트 D가 결과를 종합해서 세션 요약 문서 생성

### 4-에이전트 패턴 (세션 마무리)

```
에이전트 1: git diff 분석 + 변경 요약 작성
에이전트 2: 테스트 실행 + 결과 정리         → Phase 1 (병렬)
에이전트 3: 남은 TODO 항목 정리
       ↓
에이전트 4: 위 3개 결과 종합 → session-summary.md 생성  → Phase 2
```

## EXECUTE

### 단계 1: 4-에이전트 패턴 이해
Claude에게 물어봅니다:

```
세션 마무리를 위한 4-에이전트 패턴을 설명해줘.
각 에이전트의 역할과 실행 순서를 그림으로 보여줘.
```

### 단계 2: 세션 마무리 스킬 만들기
Claude에게 요청합니다:

```
.claude/skills/session-wrap/ 폴더에 SKILL.md를 만들어줘.
이 스킬은 /session-wrap으로 호출하면:

Phase 1 (병렬 실행):
- 에이전트 1: git status + git diff로 오늘 변경 사항 정리
- 에이전트 2: bin/rails test 실행 후 결과 요약
- 에이전트 3: CLAUDE.md의 TODO 항목 확인 및 완료/미완료 정리

Phase 2 (순차 실행):
- 에이전트 4: 위 3개 결과를 종합해서 session-summary.md 생성
  포함 내용: 오늘 한 일, 테스트 결과, 다음에 할 일

출력: "세션 마무리 완료. session-summary.md를 확인하세요."
```

### 단계 3: 병렬 에이전트 직접 실행
만든 스킬을 실행해봅니다:

```
/session-wrap
```

실행 중에 에이전트들이 동시에 작업하는 것을 확인합니다.
완료 후 생성된 `session-summary.md`를 열어서 내용을 확인합니다.

해설:
- 병렬 패턴: 독립적인 작업은 동시에 실행하면 시간을 절약할 수 있습니다. 예: 버그 수정 + 문서 작성 + 테스트는 서로 영향을 주지 않으므로 동시에 가능합니다. 반면 "A가 완료된 후에 B를 시작해야 하는" 경우에는 순차 패턴이 맞습니다.
- 2-Phase 패턴: Phase 1에서 여러 에이전트가 각자의 정보를 모으고, Phase 2에서 그 정보들을 하나로 합쳐 최종 결과물을 만듭니다. 세션 요약처럼 여러 곳의 정보를 취합해야 할 때 매우 효과적입니다.
