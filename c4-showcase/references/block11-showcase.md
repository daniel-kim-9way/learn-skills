## EXPLAIN

### Showcase에서의 첫인상

Vuild Showcase는 C3~C4를 완료한 분들이 만든 서비스를 공유하는 공간입니다. 투자자, 잠재 사용자, 다른 빌더들이 이 공간을 둘러봅니다.

서비스 목록에서 클릭을 받으려면 **제목과 설명 한 줄**이 전부입니다. 사람들은 5~10개의 서비스를 훑어보고 흥미로운 것만 클릭합니다.

좋은 Showcase 설명의 특징:
- 누구를 위한 서비스인지 바로 보임
- 무엇이 특별한지 느껴짐
- 클릭하고 싶은 호기심을 자극

나쁜 예: "독서 관리 앱입니다"
좋은 예: "책 읽고 나서 90%를 잊어버리는 분들을 위한 AI 독서 노트"

### 엘리베이터 피치란?

엘리베이터를 타는 30초 동안 투자자를 만났을 때 서비스를 설명할 수 있는 능력을 **엘리베이터 피치**라고 합니다.

30초 피치 구조:

```
"[문제 공감] 혹시 [이런 경험] 있으신가요?
[해결책] 저는 [이런 사람들]을 위해 [서비스 이름]을 만들었습니다.
[핵심 기능] [한 가지 핵심 기능]으로 [결과/혜택]을 줍니다.
[증거/숫자] 지금까지 [몇 명/어떤 반응]이 있었습니다.
[행동 촉구] 한번 써보시겠어요?"
```

예시:
"혹시 책을 읽고 나서 내용을 금방 잊어버리는 경험 있으신가요? 저는 독서광들을 위해 '북메모리'를 만들었습니다. AI가 핵심 내용을 정리하고 일정 간격으로 복습 알림을 보내주어 책 내용을 오래 기억할 수 있게 도와드립니다. 지금 50명이 사용 중이에요. 한번 써보시겠어요?"

### 피치의 심리

피치가 먹히는 이유: 사람은 **스토리**에 반응합니다.

데이터: "독서 앱, 기억 보조 기능 포함"
스토리: "책을 읽고 나서 90%를 잊어버리는 고통을 해결합니다"

같은 서비스이지만 스토리가 훨씬 강력합니다. 듣는 사람이 자신의 경험을 투영하기 때문입니다.

## EXECUTE

### 단계 1: Vuild Showcase 등록
1. https://vuild.kr/showcase 접속
2. "내 서비스 등록" 클릭
3. 아래 정보 입력:
   - 서비스 이름
   - 한 줄 설명 (60자 이내)
   - 서비스 URL
   - 대표 이미지 (스크린샷)
   - 카테고리

한 줄 설명 작성 팁: Claude에게 도움을 요청합니다:
```
내 서비스의 Showcase 한 줄 설명을 3가지 버전으로 써줘.
- 대상: [타겟 사용자]
- 해결하는 문제: [문제]
- 핵심 기능: [기능]
각 버전은 60자 이내로 해줘.
```

### 단계 2: 30초 피치 작성
Claude에게 내 서비스의 엘리베이터 피치를 부탁합니다:

```
내 서비스의 30초 엘리베이터 피치를 써줘.
아래 구조를 사용해줘:
1. 문제 공감 (공감 질문으로 시작)
2. 해결책 소개 (서비스 이름 + 대상)
3. 핵심 기능 + 혜택
4. 행동 촉구

말할 때 자연스럽게 읽히는 구어체로 써줘.
```

### 단계 3: Claude와 피치 연습
작성한 피치를 Claude와 함께 다듬습니다:

```
내가 방금 피치를 말했어. 어떤 점이 좋았고 어떤 점을 개선하면 좋을까?

내 피치:
[피치 내용 붙여넣기]

투자자 입장에서 피드백해줘:
- 잘 된 점
- 개선하면 좋을 점
- 가장 강력하게 전달되는 문장
```
