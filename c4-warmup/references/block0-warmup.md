## EXPLAIN

### 쉬고 나면 왜 잊어버릴까?

며칠 혹은 몇 주 동안 손을 놓으면 "내가 뭘 만들고 있었더라?" 하는 느낌이 옵니다. 이건 비개발자에게만 일어나는 일이 아닙니다. 프로의 개발자도 똑같이 겪습니다.

뇌는 컴퓨터와 달리 **자주 안 쓰는 기억을 뒤로 밀어둡니다.** 수면하는 것과 비슷합니다. 완전히 사라진 게 아니라 잠시 잠든 상태입니다.

그래서 필요한 것이 **워밍업**입니다. 다시 뜨겁게 달구는 작업이죠.

### 컨텍스트 회복이란?

컨텍스트(context)는 "맥락"이란 뜻입니다. 내가 만든 서비스의 맥락 — 왜 만들었는지, 어디까지 했는지, 다음에 뭘 해야 하는지 — 이 감각을 되찾는 것이 **컨텍스트 회복**입니다.

가장 빠른 방법은 두 가지입니다:
1. **직접 실행해보기** — 눈으로 보면 바로 기억이 돌아옵니다
2. **Claude에게 설명 시키기** — "우리가 뭘 만들고 있었어?"라고 물어보면 CLAUDE.md와 코드를 읽고 요약해줍니다

기술 용어 없이, 기능 중심으로 설명을 듣는 것이 포인트입니다. "라우팅이 이렇고 모델이 저렇고"가 아니라 "사용자가 로그인하면 이 화면이 나오고, 버튼을 누르면 저 화면으로 가고" 같은 방식입니다.

### 작은 변경 = 손 풀기

운동 선수도 경기 전에 스트레칭을 합니다. 코딩도 마찬가지입니다. 버튼 하나의 색을 바꾸거나 텍스트를 수정하는 **아주 작은 변경**을 해보면 "아, 이렇게 하는 거였지"라는 감각이 빠르게 돌아옵니다.

## EXECUTE

### 단계 1: 서비스 실행
프로젝트 폴더로 이동 후 서비스를 시작합니다:

```bash
# Rails 프로젝트인 경우
bin/rails server

# 또는 개발 서버
bin/dev
```

브라우저에서 `http://localhost:3000`을 열어 직접 클릭해보며 어디까지 됐는지 확인합니다.

### 단계 2: Claude에게 서비스 요약 요청
Claude Code를 열고 아래와 같이 말합니다:

```
우리가 만든 서비스를 기능 중심으로 요약해줘.
기술 용어 없이, 사용자 입장에서 "이걸 누르면 이게 된다" 식으로 설명해줘.
```

Claude가 요약하면 고개를 끄덕이며 맞는지 확인합니다. 틀린 게 있으면 정정해줍니다.

### 단계 3: 작은 변경 — 버튼 색 바꾸기
손을 풀기 위해 작은 변경 하나를 합니다:

```
메인 페이지에서 가장 눈에 띄는 버튼을 찾아서 색을 파란색으로 바꿔줘.
```

변경 후 브라우저를 새로고침(F5)해서 확인합니다. 바뀌었으면 워밍업 완료입니다.
