# V1: 코딩 환경 — 에디터·터미널·git

## 목표
- 코드 에디터·터미널·git 3가지 도구의 역할 차이를 한 줄로 설명할 수 있다
- "검은 화면(터미널)"이 무서운 게 아니라 "글자로 명령하는 곳"임을 이해한다
- git이 왜 "코드 시간여행"인지, 매일 한 번 commit이 왜 안전한지 안다

## 사전 조건
- 없음 (Ch.4 BUILD 전 사전 학습용)
- 실제 설치는 Ch.4에서 진행. 이 스킬은 개념 이해가 목적

## 진행 방식

감독 비유 프레임을 유지한다: 본인 = 영화 감독, AI = 배우, 에디터/터미널/git = 촬영 장비.

1. `references/main.md`를 읽고 도구별 설명 순서대로 진행한다
2. 도구마다: 짧은 설명(비유 포함) → 확인 질문 → 다음으로 넘어가는 구조
3. 3가지 모두 끝나면 본인 말로 한 줄 정의를 작성하도록 유도한다
4. 파일로 저장한다

## 산출물
- 파일명: `my-coding-env.md`
- 내용: 3가지 도구 한 줄 정의(본인 말) + Ch.4에서 설치할 도구 목록
