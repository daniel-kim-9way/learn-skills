# V1: 결제 + API — 외부 서비스와 대화하는 흐름

## 학습 목표
- 결제(PG)·외부 API 호출·카톡 알림이 모두 같은 패턴 — "외부 서비스와 대화" — 임을 안다
- 본인 DB에는 카드 정보가 아니라 "결제 결과(성공/실패 + 결제ID)"만 들어감을 이해한다
- API key가 무엇이고 왜 절대 코드에 직접 적으면 안 되는지(.env 사용 이유) 안다
- webhook이 "외부 서비스가 우리에게 거는 전화"라는 것을 안다
- 본인 SaaS의 외부 서비스 의존성 지도 1차 초안을 손에 쥔다

## 사전 조건
- v1-webflow / v1-data-flow 완료 (기본 인프라 + DB 흐름 익숙)
- 본인 SaaS에 결제·AI·알림 등 외부 서비스가 필요한지 한 줄로 떠올려볼 수 있음

## 대화형 학습 프로토콜

이 스킬은 대화형 코칭 방식을 따릅니다. 강의가 아닌 시뮬레이션 + 질문-답변 루프로 진행합니다.

### 진행 방식
1. references/main.md를 읽고 대화 흐름에 따라 진행
2. 결제 흐름을 메인 사례로 깔고, 같은 패턴이 다른 API 호출에도 적용된다는 걸 보여줌
3. "카드 정보 우리가 받지 않음"의 충격 → PG → API key → webhook 순서로 풀어줌
4. 마지막 Turn에서 본인 SaaS의 외부 서비스 지도를 파일로 저장
