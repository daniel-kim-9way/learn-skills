## EXPLAIN

### 인증 = 집 열쇠

여러분의 서비스에 아무나 들어와서 데이터를 마구 바꾸면 안 되겠죠? 음식점에 아무나 들어와서 주방을 어지럽히면 안 되듯이, 서비스에도 **문**과 **열쇠**가 필요합니다.

이것이 **인증(Authentication)**입니다.

| 일상 비유 | 서비스에서 | 하는 일 |
|----------|-----------|---------|
| 열쇠 만들기 | **회원가입** | 이메일과 비밀번호를 등록 |
| 열쇠로 문 열기 | **로그인** | 등록한 정보로 서비스에 입장 |
| 문 잠그고 나가기 | **로그아웃** | 서비스에서 퇴장 |
| 내 방은 나만, 거실은 모두 | **권한** | 로그인한 사람만 볼 수 있는 페이지 |

좀 더 구체적으로 비유하면:
- **홈페이지** = 건물 로비 → 누구나 들어올 수 있음
- **핵심 기능 페이지** = 사무실 → 열쇠(로그인)가 있어야 들어갈 수 있음
- **내 데이터** = 내 사무실 서랍 → 내 열쇠로만 열 수 있음

### Rails 8의 내장 인증 — 좋은 소식!

좋은 소식이 있습니다. Rails 8에는 인증 기능이 **이미 내장**되어 있습니다.

예전에는 인증을 만드려면 보안 전문가가 필요했습니다. 비밀번호 암호화, 세션 관리, 보안 취약점 방지... 복잡한 것들이 많았죠.

Rails 8에서는? Claude에게 "Rails 8 내장 인증을 설정해줘"라고 하면 끝입니다.

내장 인증이 처리해주는 것:
- **비밀번호 암호화**: 비밀번호가 그대로 저장되지 않음 (해킹 방지)
- **세션 관리**: 로그인 상태를 유지 (매번 다시 로그인 안 해도 됨)
- **보안 기본 설정**: 기본적인 보안이 자동으로 적용

여러분은 보안에 대해 아무것도 몰라도 됩니다. Rails가 다 해줍니다.

### 이번 레슨에서 만들 것

이번 레슨은 C3에서 **가장 깁니다** (60분). 하지만 작은 마일스톤으로 나뉘어 있어서, 하나씩 완료하면 됩니다. 중간에 쉬어도 괜찮습니다.

완성하면 이런 모습이 됩니다:

**비로그인 상태:**
- 홈페이지 보임 (멋진 디자인 + "시작하기" 버튼)
- 네비게이션에 "로그인" + "회원가입" 버튼

**로그인 상태:**
- 홈페이지의 "시작하기" → 대시보드로 이동
- 네비게이션에 "안녕하세요, [이름]님" + "로그아웃" 버튼
- 핵심 기능 페이지 접근 가능

- [ ] **대시보드**: 환영 메시지가 보인다
- [ ] **로그아웃**: 클릭하면 홈페이지로 이동
- [ ] **다시 로그인**: 기존 계정으로 로그인 성공
- [ ] **시작하기 클릭 (로그인)**: 대시보드로 이동
- [ ] **잘못된 로그인**: 틀린 비밀번호로 로그인 시 에러 메시지
