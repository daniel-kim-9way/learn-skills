## EXPLAIN

### 디자인은 예쁨이 아니라 신뢰입니다

처음 서비스를 만들면 "기능 먼저, 디자인 나중에"라고 생각하기 쉽습니다. 하지만 사용자는 다르게 행동합니다.

사람은 **0.05초** 안에 웹사이트의 신뢰도를 판단합니다.

0.05초. 눈 깜짝할 사이보다 빠릅니다. 기능을 써보기도 전에, 글을 읽기도 전에, **눈**이 먼저 결정합니다. "이거 믿을 수 있을까?" 아니면 "뒤로 가기 버튼을 누를까?"

스탠포드 대학교 연구에 따르면 사용자의 75%가 **첫인상만으로** 서비스의 신뢰를 결정합니다. 기능이 아무리 좋아도, 첫인상에서 탈락하면 기능을 경험할 기회조차 없습니다.

**명함을 떠올려봅시다.** 내용은 같아도 구겨진 종이에 적은 것과 깔끔한 카드에 인쇄한 것은 완전히 다른 인상을 줍니다. 서비스도 마찬가지입니다.

> **디자인은 예쁨이 아닙니다. 신뢰입니다.**

### Before vs After: 같은 서비스, 다른 세계

디자인 개선의 효과를 머릿속에 그려봅시다.

**Before** (C3에서 만든 직후):
- 여백 없이 글자와 버튼이 빼곡
- 색상이 뒤죽박죽 — 파란 버튼, 초록 버튼, 빨간 링크
- 중요한 것과 안 중요한 것의 구분이 없음
- 모바일에서 글자가 화면 밖으로 삐져나옴

**After** (이 레슨을 마친 후):
- 깔끔한 여백으로 눈이 편안함
- 통일된 색상 테마 — 브랜드 느낌
- 큰 제목 → 본문 → 버튼 순서로 자연스러운 시선 흐름
- 모바일에서도 완벽하게 보임

같은 서비스인데 느낌이 완전히 다릅니다. **이것이 디자인의 힘입니다.**

오늘 이 레슨이 끝나면 여러분의 서비스가 Before에서 After로 바뀝니다.

### Clean이 이긴다

| | Impressive (화려한) | Clean (깔끔한) |
|--|---------------------|---------------|
| 느낌 | 복잡하고 화려 | 단순하고 정돈됨 |
| 사용자 반응 | "와, 멋있다" → 어디를 눌러야 하지? | "바로 이해된다" → 클릭 |
| 목적 | 인상 주기 | 행동 유도 |
| 전환율 | 낮음 (혼란) | 높음 (명확) |

비즈니스 서비스에서는 대부분 **Clean이 이깁니다.** 사용자가 "무엇을 해야 하는지" 바로 알 수 있는 디자인이 전환율을 높입니다.

넷플릭스, 에어비앤비, 토스를 생각해봅시다. 화려하지 않습니다. 하지만 어디를 눌러야 하는지 즉시 알 수 있습니다. 이것이 **Clean 디자인**입니다.

### 디자인 핵심 원칙 3가지

**1. 여백 — 숨 쉴 공간을 주세요**
빽빽한 것은 답답합니다. 글자와 글자 사이, 섹션과 섹션 사이에 충분한 여백을 두면 눈이 편안해집니다. "이것도 넣고 저것도 넣고" 싶겠지만, 비우는 것이 채우는 것보다 강력합니다.

고급 레스토랑을 상상해봅시다. 테이블 위에 접시가 빼곡한 뷔페와, 접시 하나가 정중앙에 놓인 파인다이닝. 어느 쪽이 더 격이 있어 보이나요?

**2. 계층 — 중요도를 눈으로 표현하세요**
모든 것이 같은 크기면 아무것도 눈에 들어오지 않습니다.
- **큰 제목**: "이게 뭔지"를 알려주는 헤드라인
- **중간 본문**: 설명하는 텍스트
- **작은 버튼**: "이걸 해봐"라고 유도하는 CTA
- **더 작은 텍스트**: 부가 정보, 약관 등

이 계층이 있으면 사용자의 시선이 자연스럽게 위에서 아래로 흐릅니다.

**3. 일관성 — 통일감이 신뢰를 만듭니다**
- 색상: 메인 1개 + 보조 1개. 끝.
- 폰트: 제목용 1개 + 본문용 1개. 끝.
- 버튼 모양: 전체 서비스에서 같은 스타일

색상이 뒤죽박죽인 서비스 vs 일관된 색상의 서비스. 후자가 "전문적"이라는 느낌을 줍니다.

### Tailwind CSS: 레고 블록으로 디자인하기

CSS는 웹의 화장품입니다. 글자 색, 버튼 크기, 간격을 결정합니다.

기존 CSS는 이름을 짓고 별도 파일을 만들어야 했습니다. Tailwind CSS는 다릅니다. 미리 만들어진 수천 개의 **"유틸리티 클래스"**를 레고 블록처럼 조합합니다:

```html
<!-- Tailwind CSS 예시 -->
<button class="bg-blue-500 text-white px-4 py-2 rounded-lg shadow hover:bg-blue-600">
  시작하기
</button>
```

클래스 이름이 곧 설명입니다:
- `bg-blue-500` → 배경색 파란색
- `text-white` → 글자 흰색
- `px-4 py-2` → 좌우 패딩 4, 위아래 패딩 2
- `rounded-lg` → 모서리 둥글게
- `shadow` → 그림자 효과
- `hover:bg-blue-600` → 마우스 올리면 진한 파란색

**레고 블록을 끼우듯** 원하는 스타일을 조합하면 됩니다.

여러분이 이 코드를 이해하거나 직접 쓸 필요는 없습니다. Claude에게 "Tailwind로 이 페이지를 개선해줘"라고 하면 알아서 적절한 클래스를 적용해줍니다.

여러분의 역할은 **눈으로 보고 판단하는 것**입니다:
- "여기 여백이 더 필요해"
- "이 버튼 색이 너무 연해"
- "카드 모서리를 좀 더 둥글게"

이 정도면 충분합니다.

### 모바일 먼저 확인하기

한국에서 웹 트래픽의 60% 이상이 모바일에서 발생합니다. 여러분의 서비스도 절반 이상의 사용자가 폰으로 볼 가능성이 높습니다.

PC에서는 멋져 보이는데 모바일에서 깨지면 절반의 사용자를 잃는 겁니다.

모바일 확인 방법:
1. 브라우저에서 F12 (개발자 도구 열기)
2. 왼쪽 상단 모바일 아이콘 클릭 (또는 Ctrl+Shift+M)
3. 상단 드롭다운에서 "iPhone 14 Pro" 같은 기기 선택
4. 화면이 모바일 크기로 줄어듦 → 여기서 확인

Tailwind CSS는 반응형 디자인을 쉽게 만들어줍니다. Claude에게 "모바일에서도 잘 보이게 해줘"라고 하면 됩니다.

> "PLAN에서 만든 디자인 시스템(색·폰트·여백 토큰)이 있나요? 있다면 그것을 이번에 함께 적용할까요?"

옵션별 안내:
- **있어요** → "좋아요. PLAN의 디자인 토큰을 기준으로 Tailwind 클래스를 정하겠습니다. 적용 범위는 어디까지로 할까요? 전체 페이지 / 헤더·버튼만 / 랜딩 페이지만"
- **없어요** → "괜찮아요. 오늘 작업하면서 색상 테마가 자연스럽게 정해질 거예요."

확인 후 다음 단계로.

### 레퍼런스 사이트 6곳 — 스타일 참고

학습자가 "어떤 느낌으로 가야 할지 모르겠다"고 하면 다음 6곳을 보여준다:

| 사이트 | 특징 | 따라하면 좋은 것 |

| **shadcn/ui** (ui.shadcn.com) | 미니멀하고 모던 | 버튼·인풋·대화상자 스타일 |

### 단계 1: Before 스크린샷 찍기

나중에 Before/After를 비교하려면 지금 모습을 기록해야 합니다.

- `http://localhost:3000` 접속
- 메인 페이지 화면 캡처 (Windows: Win+Shift+S, Mac: Cmd+Shift+4)
- 파일로 저장해두세요 (예: `before.png`)

> **꿀팁**: 캡처를 2~3장 찍어두세요. 메인 페이지, 목록 페이지, 상세 페이지 등. 나중에 비교할 때 "와, 이렇게 달라졌어?" 하는 감동이 배가 됩니다.

### 단계 2: 프롬프트 1 — 전체 레이아웃 정리

가장 먼저 큰 틀을 잡습니다. 집을 리모델링할 때 벽부터 정리하듯이요.

```
메인 페이지 레이아웃을 Tailwind CSS로 정리해줘.

원칙:
- 여백을 충분히 줘서 숨 쉬는 느낌으로
- 콘텐츠 폭은 max-w-6xl로 가운데 정렬
- 섹션 간 간격을 넉넉하게 (py-16 이상)
- 네비게이션 바를 깔끔하게 정리
- 푸터에 저작권, 연락처 정보 포함

깔끔하고 전문적인 느낌으로.
```

변경 후 브라우저를 새로고침해서 확인합니다.

> **에러가 나면**: "이 오류가 났어: [오류 메시지]"라고 Claude에게 붙여넣으면 됩니다. 대부분 즉시 해결됩니다.

### 단계 3: 프롬프트 2 — 카드형 디자인 적용

목록이나 콘텐츠가 나열되는 부분을 카드 형태로 바꿉니다. 카드는 정보를 깔끔하게 묶어주는 마법의 틀입니다.

```
목록이나 반복되는 콘텐츠를 카드 형태로 개선해줘.

적용할 것:
- 카드에 흰 배경 + 미세한 그림자 (shadow-sm)
- 모서리 둥글게 (rounded-xl)
- 호버하면 살짝 떠오르는 효과 (hover:shadow-md hover:-translate-y-1 transition)
- 카드 안에서 정보의 계층 표현 (제목 크게, 설명 작게)
- 그리드 레이아웃 (모바일 1열, 태블릿 2열, PC 3열)
```

### 단계 4: 프롬프트 3 — 색상 테마 통일

이제 전체 서비스의 색감을 통일합니다. 이 단계에서 "브랜드" 느낌이 생깁니다.

```
전체 색상 테마를 통일해줘.

규칙:
- 메인 색상 1개 (브랜드 컬러) — 중요한 버튼, 링크에 사용
- 보조 색상 1개 — 보조 버튼, 배지에 사용
- 배경: 깨끗한 흰색(white) 또는 연한 회색(gray-50)
- 텍스트: 제목은 gray-900, 본문은 gray-600
- 성공은 초록(green-500), 경고는 노랑(amber-500), 에러는 빨강(red-500)

모든 페이지에 일관되게 적용해줘.
```

### 단계 5: 반복 다듬기

여기서부터가 진짜 디자인입니다. 완성된 결과를 보면서 마음에 안 드는 부분을 **구체적으로** 피드백합니다.

피드백 예시:
```
헤더가 너무 커서 첫 화면에서 콘텐츠가 안 보여. 헤더 높이를 줄여줘.
```

```
'시작하기' 버튼이 배경과 비슷한 색이라 안 보여. 
더 대비되는 색으로 바꾸고 크기도 키워줘.
```

```
카드 사이 간격이 너무 좁아. 좀 더 여유롭게 해줘.
```

```
글자가 너무 작아서 읽기 힘들어. 본문 폰트 크기를 키워줘.
```

**2~3번 피드백을 반복하며 마음에 드는 디자인을 찾습니다.** 정답은 없습니다. 내가 보기에 "깔끔하다"고 느끼면 됩니다.

> **꿀팁**: 한 번에 너무 많은 것을 요청하지 마세요. "헤더를 줄여줘" → 확인 → "버튼 색을 바꿔줘" → 확인. 하나씩 바꾸면 어떤 변경이 좋았는지 판단하기 쉽습니다.

### 단계 6: 모바일 확인

PC에서 디자인이 완성되었으면, 모바일에서도 확인합니다.

1. 브라우저에서 F12 → 모바일 뷰 전환 (Ctrl+Shift+M)
2. "iPhone 14 Pro" 선택
3. 천천히 스크롤하며 확인:
   - 글자가 잘 보이는지
   - 버튼이 손가락으로 누를 수 있는 크기인지
   - 이미지가 화면 밖으로 나가지 않는지
   - 가로 스크롤이 생기지 않는지

문제가 있으면:
```
모바일에서 [카드가 화면 밖으로 나가요 / 버튼이 너무 작아요 / 텍스트가 잘려요].
모바일에서도 잘 보이게 수정해줘.
```

### 단계 7: After 비교 — 감동의 순간

Before 스크린샷과 현재 화면을 나란히 놓고 비교합니다.

같은 서비스입니다. 같은 기능입니다. 하지만 느낌이 완전히 다릅니다.

Before의 서비스가 "학생 과제"처럼 보였다면, After의 서비스는 "실제 서비스"처럼 보입니다.

**이것이 디자인의 힘입니다.** 그리고 여러분은 코드를 한 줄도 직접 쓰지 않고 이것을 해냈습니다.

> **막혔을 때**: 디자인 감각이 없다고 걱정하지 마세요. "깔끔해 보이는지 아닌지"는 누구나 판단할 수 있습니다. 그 판단만 Claude에게 전달하면 됩니다. 여러분은 **디자이너가 아니라 디렉터**입니다.
