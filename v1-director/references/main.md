## CONTEXT

이 스킬은 Ch.1의 세 번째 lesson (1-3)에서 사용된다.
Ch.4 BUILD에서 실제로 에디터를 설치하고 터미널을 열기 전에 — "이 도구들이 뭔지, 왜 필요한지"를 미리 머리에 그려두는 것이 목적이다.
감독 비유 프레임이 메인: 본인=감독, AI=배우, 도구들=촬영 장비.
실제 설치는 이 스킬에서 진행하지 않는다.

### 3가지 도구 — 한 줄 정의 + 감독 비유

| 도구 | 역할 | 감독 비유 |
|---|---|---|
| 코드 에디터 (VS Code / Antigravity) | 코드 파일을 보고 수정하는 곳 | 시나리오를 펼쳐놓는 책상 |
| 터미널 | PC에 글자로 명령하는 검은 화면 | "카메라 돌려!" 외치는 마이크 |
| git + GitHub | 코드 변경 이력 저장 + 클라우드 백업 | 매 컷마다 적는 촬영 일지 + 외장 하드 |

### Turn 1: 코드 에디터

비유로 설명한다:

> "영화 촬영장에서 감독이 시나리오를 보는 책상을 상상해보세요.
> 코드 에디터가 바로 그 책상이에요. 코드 파일들을 한 눈에 보고, 수정하고, 저장하는 곳입니다.
>
> 워드(Word)나 메모장과 비슷한데, 코드에 맞게 색깔 표시(구문 강조)가 되고, 오타를 잡아주고, AI 자동완성도 붙어 있어요.
>
> LEARN에서는 **Antigravity**를 기본으로 씁니다. AI가 내장된 코드 에디터예요."

확인 질문:
> "코드 에디터가 뭔지 한 줄로 설명해볼 수 있어요? 본인 말로 편하게 해주세요."

학습자가 설명하면:
> "맞아요! 그게 전부예요. 다음으로 넘어갈게요."

---

### Turn 2: 터미널

비유로 설명한다:

> "감독이 '카메라 돌려!', '조명 올려!', '잠깐 컷!'을 외치는 마이크.
> 터미널이 바로 그 마이크예요. PC에 명령을 글자로 직접 보내는 검은 화면이에요.
>
> 처음 보면 무서워 보이지만 — 사실 그냥 '글자로 명령하는 곳'이에요.
>
> 예를 들면:
> - `ls` → '이 폴더에 뭐가 있지?' 라고 묻는 것
> - `cd myproject` → '내 프로젝트 폴더로 이동' 하는 것
> - `bin/rails server` → '서버 실행해줘' 하는 것
>
> 이 5개 정도만 알아도 90%는 됩니다. 나머지는 AI에게 '이거 어떻게 하지?' 물어보면 돼요."

확인 질문:
> "터미널이 무서운 게 아니라 '글자로 명령하는 곳'이라는 거, 이해됐어요?"

---

### Turn 3: git + GitHub

비유로 설명한다:

> "영화 감독이 매일 '오늘 찍은 씬 목록, 잘된 것, 다시 찍을 것'을 촬영 일지에 적잖아요.
> git이 바로 그 일지예요. 코드가 어떻게 변했는지 날짜별로 기록해둡니다.
>
> 그리고 GitHub는 그 일지를 클라우드에 백업하는 외장 하드예요.
>
> 왜 필요한가:
> - 어젯밤에 잘 되던 코드가 오늘 갑자기 안 된다면? → git으로 어젯밤으로 돌아갈 수 있어요.
> - 노트북이 망가지면? → GitHub에 올려뒀으면 새 PC에서 바로 복구 가능해요.
>
> 매일 한 번 git commit(일지 기록)이 안전선이에요."

확인 질문:
> "git과 GitHub의 차이를 한 줄로 말해볼 수 있어요?"

힌트가 필요하면:
> "git = 내 PC에 기록 / GitHub = 인터넷에 백업. 이게 전부예요."

---

### Turn 4: 한 줄 정의 완성 + 파일 저장

> "수고하셨어요! 3가지를 다 봤네요.
> 마지막으로 — 본인 말로 각 도구를 한 줄씩 정의해주세요. 완벽할 필요 없어요."

## 주의 사항

- 실제 설치를 시도하거나 유도하지 않는다. 이 스킬은 개념 이해만.
- 터미널 명령을 많이 가르치지 않는다. 5개만 언급하고 "나머지는 AI가 알려줌"으로 마무리.
- 감독 비유를 일관되게 유지한다.
- 도구별로 확인 질문 후 다음 단계로. 강의식 일방 전달이 되지 않게 한다.
