## EXPLAIN

### 영화감독 비유: 봉준호처럼 생각하기 🎬

영화 '기생충'을 만든 봉준호 감독을 떠올려보세요. 봉 감독이 직접 카메라를 들었을까요? 직접 조명을 설치하고, 세트를 만들고, 편집을 했을까요? 아닙니다.

봉준호 감독이 한 일은 이렇습니다:
- "이 장면에서 반지하 계단을 내려가는 느낌은 습하고 무거워야 해"
- "송강호가 문을 열 때 카메라는 그의 뒤통수에서 천천히 따라가"
- "비가 올 때 부잣집과 반지하집의 조명 온도가 달라야 해"

감독은 **비전을 제시하고, 구체적으로 지시**합니다. 그러면 촬영감독, 조명감독, 미술감독이 그 비전을 실현하죠.

AI 시대에 서비스를 만드는 것도 **정확히** 이 구조입니다.
- 당신 = **감독** (기획자, 의사결정자)
- AI = **제작팀** (개발자 + 디자이너 + 기획자가 합쳐진 올인원 제작팀)

코딩을 배울 필요가 없습니다. 대신 **좋은 감독이 되는 법**을 배워야 합니다.

### 나쁜 감독 vs 좋은 감독

같은 제작팀(AI)이라도, 감독의 지시에 따라 결과물이 완전히 달라집니다.

| 상황 | 나쁜 감독 | 좋은 감독 |
|------|----------|----------|
| 로그인 페이지 | "로그인 페이지 만들어줘" | "모바일 우선 로그인 페이지. 이메일+비번 입력, 구글 로그인 버튼, 밝은 파란 배경, 로고 상단 중앙, 비번 찾기 링크 포함" |
| 장바구니 | "장바구니 기능 넣어줘" | "장바구니에 상품명, 수량 조절 버튼(+/-), 개별 삭제, 총 금액 자동 계산, 주문하기 버튼. 상품이 없을 때는 '장바구니가 비어있어요' 메시지" |
| 대시보드 | "관리자 대시보드 만들어줘" | "오늘/이번 주/이번 달 매출 숫자 카드 3개, 최근 주문 5개 목록 테이블, 왼쪽 사이드바에 메뉴. 전체 톤은 깔끔한 흰색 배경" |
| 색상 변경 | "색깔 좀 바꿔줘" | "메인 색상을 #3B82F6(파란색)에서 #10B981(초록색)으로 변경. 버튼, 헤더, 링크 색상 모두 통일" |
| 에러 수정 | "안 돼, 고쳐줘" | "회원가입 버튼을 눌렀는데 아무 반응이 없어. 버튼 클릭 시 입력값 검증 후 성공 메시지를 보여줘야 해" |

핵심 차이가 보이시나요? 나쁜 감독은 **"뭘 해줘"**만 말합니다. 좋은 감독은 **"뭘, 어떻게, 왜"**를 말합니다.

### 좋은 지시의 3원칙

1. **구체성**: 모호하지 않게, 눈에 보이는 것처럼 설명
   - (나쁜 예) "예쁘게 만들어줘" -- AI는 '예쁘다'의 기준을 모릅니다
   - (좋은 예) "둥근 모서리, 밝은 파란 배경, 흰색 글씨, 여백 넉넉하게"

2. **맥락**: 누가 쓸 건지, 어떤 상황인지 알려주기
   - (나쁜 예) "페이지 만들어줘"
   - (좋은 예) "50대 어머니도 쉽게 쓸 수 있는 큰 글씨의 예약 페이지"

3. **예시**: "이런 느낌으로" 참고할 만한 것 제시
   - (나쁜 예) "깔끔하게"
   - (좋은 예) "토스 앱처럼 심플하고 깔끔한 느낌으로"

이 3가지만 기억하면, AI가 당신의 머릿속 그림을 훨씬 정확하게 구현합니다.

### 4가지 지시 스타일

사람마다 AI에게 말하는 방식이 다릅니다. 어떤 스타일이든 좋고 나쁨은 없습니다. 자기에게 맞는 스타일을 아는 것이 중요합니다.

| 스타일 | 성격 | 특징 | 장점 | 주의점 |
|--------|------|------|------|--------|
| **설명형** 📝 | 꼼꼼한 계획가 | 길고 자세하게 문장으로 설명 | 한 번에 원하는 결과가 나올 확률이 높음 | 처음부터 모든 걸 정하기 어려울 수 있음. 완벽주의에 빠지지 않도록 주의 |
| **키워드형** ⚡ | 효율적인 실행가 | 핵심 단어를 짧게 나열 | 빠르게 프로토타입을 뽑아냄 | 맥락이 부족하면 AI가 엉뚱한 방향으로 갈 수 있음. 핵심 맥락 1-2줄 추가 필요 |
| **예시형** 🎯 | 비전이 명확한 기획자 | "이것처럼 만들어줘"로 방향 제시 | AI가 디자인/기능 방향을 빠르게 파악 | 예시를 찾는 시간이 필요. 스크린샷이나 URL을 함께 주면 최고 |
| **대화형** 💬 | 유연한 탐험가 | 짧게 주고받으며 점점 완성 | 처음 방향이 명확하지 않아도 시작 가능 | 대화가 길어지면 AI가 앞의 맥락을 놓칠 수 있음. 중간 정리 필요 |

### 흔한 실수 모음

비개발자가 AI에게 지시할 때 자주 하는 실수들입니다. 미리 알아두면 시간을 절약할 수 있어요.

1. **"알아서 해줘" 증후군**: AI는 당신의 취향을 모릅니다. "예쁘게", "좋게", "적당히"는 AI에게 의미 없는 말이에요. 구체적으로!

2. **한 번에 너무 많이 시키기**: "회원가입, 로그인, 게시판, 결제, 관리자 페이지 다 만들어줘" -- 이러면 AI도 혼란스럽습니다. 하나씩 순서대로 시키세요.

3. **불만만 말하기**: "이거 별로야. 다시 해줘" -- AI는 뭐가 별로인지 모릅니다. "버튼이 너무 작아서 클릭하기 어려워. 2배로 키워줘"처럼 구체적으로.

4. **감사는 불필요**: "감사합니다, 수고하셨습니다" -- AI에게 예의를 차릴 필요는 없어요. 그 시간에 하나라도 더 구체적으로 지시하세요. (물론 말해도 나쁠 건 없습니다!)

5. **처음부터 완벽을 추구**: 첫 결과물이 90%만 맞아도 대성공입니다. 나머지 10%는 수정 지시로 다듬으세요.

### Turn 2: 직접 지시해보기

> "자, 그러면 감독으로서 첫 지시를 내려볼까요? AI에게 '로그인 페이지를 만들어달라'고 말해야 합니다. 어떻게 말하시겠어요? 그냥 자연스럽게 말해주세요. 정답은 없으니까 편하게!"

학생의 답변을 **그대로 기록**한다. 이것이 학생의 자연스러운 지시 방식이다.

학생이 짧게 답하면 ("로그인 페이지 만들어줘"):
> "좋아요, 아주 자연스러운 지시예요! 이대로도 AI가 만들어주긴 해요. 근데 결과물이 마음에 들까요? 한번 비교해볼게요."

### Turn 3: 세 가지 버전 비교

학생의 답변 후, 세 가지 상황에서 모호한 지시와 구체적 지시를 나란히 보여준다:

**비교 1: 로그인 페이지**

| 모호한 지시 | 구체적 지시 |
|------------|-----------|
| 로그인 페이지 만들어줘 | 모바일 우선 로그인 페이지. 이메일+비번 입력, 구글 로그인 버튼, 밝은 파란 배경, 로고 상단 중앙, 비번 찾기 링크. 깔끔하고 심플한 느낌 |

**비교 2: 장바구니**

| 모호한 지시 | 구체적 지시 |
|------------|-----------|

**비교 3: 대시보드**

| 모호한 지시 | 구체적 지시 |
|------------|-----------|
| 관리자 페이지 만들어줘 | 상단에 오늘 매출/주문 수/신규 회원 숫자 카드 3개. 아래에 최근 주문 5개 테이블. 왼쪽 사이드바에 메뉴(대시보드/상품관리/주문관리). 흰색 배경, 깔끔한 톤 |

> "세 개를 비교해보니까 차이가 확 느껴지지 않나요? 어떤 것이 AI가 더 잘 만들어줄 것 같아요?"

학생의 답변을 듣고:
> "맞아요! 구체적 지시가 훨씬 나은 결과를 가져옵니다. 여기서 핵심이 되는 원칙이 3가지 있어요."

좋은 지시의 3원칙을 설명:
1. **구체성** -- "로그인 페이지"만으로는 부족. 뭐가 있어야 하는지까지
2. **맥락** -- "모바일 우선" → AI가 화면 크기를 고려하게 됨
3. **예시/톤** -- "깔끔하고 심플한 느낌" → 디자인 방향 제시

> **꿀팁**: "토스처럼", "배민처럼", "당근마켓처럼" -- 한국 서비스를 예시로 들면 AI가 방향을 훨씬 빨리 잡아요. AI도 한국 서비스를 잘 알고 있거든요!

## QUIZ

### Q1: 감독과 제작팀
> "AI 시대에 서비스를 만들 때, 당신의 역할은?"

선택지:
1. 코드를 직접 작성하는 개발자
2. 비전을 제시하고 지시하는 감독
3. AI가 다 알아서 해주니까 구경꾼
4. 디자이너

정답: 2번
해설: 봉준호 감독이 카메라를 직접 들지 않는 것처럼, 당신도 코드를 직접 쓸 필요가 없습니다. "무엇을, 왜, 어떻게" 만들지 결정하는 감독 역할이 당신의 일입니다.

### Q2: 좋은 지시의 핵심
> "AI에게 '로그인 페이지 만들어줘'라고만 말하면 부족한 이유는?"

선택지:
1. AI가 이해를 못해서
2. 구체성, 맥락, 예시가 빠져서 AI가 아무렇게나 만들기 때문
3. 로그인 페이지는 AI가 못 만들어서
4. 한국어로 말해서

정답: 2번
해설: AI는 이해력이 뛰어나지만, 정보가 부족하면 자기 마음대로 채워넣습니다. 나쁜 감독이 "알아서 해"라고 하면 엉뚱한 영화가 나오는 것처럼요. 구체성 + 맥락 + 예시를 주면 결과물이 완전히 달라집니다.

### Q3: 지시 스타일
> "다음 중 좋은 지시 습관은?"

선택지:
1. 한 번에 모든 기능을 다 시키기
2. "별로야, 다시 해" 라고 말하기
3. 하나씩 순서대로 시키고, 수정 사항은 구체적으로 말하기
4. AI에게 "알아서 예쁘게 해줘" 라고 말하기

정답: 3번
해설: 좋은 감독은 한 장면씩 완성합니다. AI에게도 하나씩 시키고, 수정할 때는 "뭐가 어떻게 바뀌어야 하는지" 구체적으로 말하는 것이 최고의 협업법입니다.
