## EXPLAIN

### 좋은 이름의 조건

서비스 이름은 **첫인상**입니다. 좋은 이름은:
- 한번 들으면 기억에 남는다
- 무엇을 하는 서비스인지 감이 온다
- 말로 전달하기 쉽다 (전화로 알려줄 수 있는가?)

나쁜 이름의 예: "SmartLifeHubPlatformPro" — 뭐 하는 건지 모르겠고, 기억도 안 나고, 전화로 설명하기도 어렵습니다.

### 한국 서비스 이름의 좋은 예시

이미 성공한 서비스들의 이름을 분석해보면 패턴이 보여요:

**당근마켓**
- 유래: "당근이지!" (당연하다의 귀여운 표현) + 동네 거래의 신선함
- 왜 좋은가: 2글자로 짧고, 부르기 쉽고, 한국인에게 친근한 표현
- 지금은 "당근"으로 줄여 부름 → 이름이 짧을수록 줄여 불리기 쉬움

**토스(Toss)**
- 유래: 영어 "toss" = 던지다, 가볍게 보내다
- 왜 좋은가: 2음절, 발음이 간결, "돈을 간편하게 던진다"는 느낌
- 교훈: 영어라도 짧고 발음 쉬우면 OK

**배달의민족**
- 유래: "배달민족"의 말장난 (배달 + 한민족)
- 왜 좋은가: 한국적 유머, 웃기면서도 뭐 하는 건지 바로 앎
- 교훈: 유머는 기억에 남게 하는 강력한 도구

**클래스101**
- 유래: Class(수업) + 101(입문 과정을 뜻하는 미국식 표현)
- 왜 좋은가: "수업"이라는 직관 + "101"이 체계적/전문적 느낌
- 교훈: 숫자를 넣으면 독특하고 검색하기 쉬움

**밀리의서재**
- 유래: 밀리(사람 이름 느낌) + 서재(책읽는 공간)
- 왜 좋은가: 친근한 이름 + 고급스러운 공간의 조합
- 교훈: "사람이름 + 장소"는 친근하면서도 브랜드 느낌을 줌

**오늘의집**
- 유래: "오늘의" + "집" — 매일 새로운 인테리어 아이디어
- 왜 좋은가: 100% 한국어, 직관적, 따뜻한 느낌
- 교훈: 순우리말도 강력한 브랜드가 될 수 있음

### 이름 짓기 3가지 스타일

1. **직관형**: 하는 일이 바로 보이는 이름
   - 배달의민족, 직방, 오늘의집, 번개장터

2. **은유형**: 비유나 상징을 사용하는 이름
   - 토스(던지다 → 간편), 당근(당연하지 → 쉬운 거래), 리멤버(기억하다 → 명함 관리)

3. **조합형**: 두 단어를 합치거나 변형한 이름
   - 카카오톡(cacao + talk), 네이버(navigate + er), 클래스101(class + 101)

### 이름 체크리스트 (7가지)

이름 후보가 나오면 이 체크리스트로 점검하세요:

1. **발음**: 소리 내어 읽었을 때 부드럽고 자연스러운가?
2. **검색**: 구글/네이버에 검색했을 때 같은 이름의 서비스가 없는가?
3. **도메인**: .com 또는 .kr 도메인을 확보할 수 있는가? (나중에 확인)
4. **의미**: 서비스와 관련된 의미가 느껴지는가?
5. **길이**: 4글자 이하가 이상적 (줄여 부를 수 있는가?)
6. **외국어 가능성**: 영어로 표기했을 때 어색하지 않은가?
7. **부정적 연상**: 다른 뜻으로 해석될 여지가 없는가? (은어, 비속어 등)

### 이름 테스트 4가지

이름 후보가 나오면 반드시 테스트합니다:

1. **전화 테스트**: 전화로 "OOO 검색해봐"라고 말했을 때, 상대가 바로 찾을 수 있는가?
2. **검색 테스트**: 구글/네이버에 검색했을 때, 이미 같은 이름의 서비스가 있는가?
3. **5초 테스트**: 이름만 보고 5초 안에 "어떤 서비스인지" 감이 오는가?
4. **엘리베이터 테스트**: 30초 안에 "이 서비스는 ___예요"라고 설명할 수 있는가?

### 한 줄 가치 선언문

이름이 정해지면, 이 서비스가 무엇인지 한 줄로 설명합니다:

> **"[서비스명]은 [대상]이 [문제]를 [방법]으로 해결하도록 돕습니다."**

예시:
- "토스는 2030세대가 복잡한 금융을 쉬운 터치로 해결하도록 돕습니다"
- "당근은 이웃 주민이 중고 거래의 불안을 동네 인증으로 해결하도록 돕습니다"
- "오늘의집은 자취생이 인테리어 감각 부족을 실제 사례 공유로 해결하도록 돕습니다"
- "클래스101은 취미를 배우고 싶은 직장인이 시간/장소 제약을 온라인 클래스로 해결하도록 돕습니다"
- "밀리의서재는 독서하고 싶지만 책 고르기 어려운 사람이 선택 장애를 큐레이션 추천으로 해결하도록 돕습니다"

### Q1: 좋은 이름의 조건
> "서비스 이름을 고를 때 '전화 테스트'를 하는 이유는?"

선택지:
1. 전화 영업을 할 예정이라서
2. 말로 전달했을 때 상대가 바로 검색할 수 있는지 확인하기 위해서
3. 전화번호와 이름을 연결하기 위해서
4. 글자 수를 확인하기 위해서

정답: 2번
해설: 입소문은 서비스 성장의 핵심입니다. "야, [서비스이름] 써봐"라고 말했을 때 상대가 바로 찾을 수 있어야 합니다.

### Q2: 가치 선언문
> "한 줄 가치 선언문의 구조는?"

선택지:
1. [서비스명]은 최고의 서비스입니다
2. [서비스명]은 [대상]이 [문제]를 [방법]으로 해결하도록 돕습니다
3. [서비스명]은 [기술]을 사용하여 만들었습니다

### Q3: 엘리베이터 테스트
> "엘리베이터 테스트란?"

선택지:
1. 엘리베이터에서 앱을 테스트하는 것
2. 30초 안에 서비스를 한 문장으로 설명할 수 있는지 확인하는 것
3. 엘리베이터 회사에 서비스를 파는 것
4. 엘리베이터 안에서 인터넷이 되는지 확인하는 것

정답: 2번
해설: 엘리베이터에서 아는 사람을 만났을 때처럼, 짧은 시간 안에 서비스를 명확하게 설명할 수 있어야 합니다. 설명이 길어진다면 이름과 서비스의 연결이 약한 것입니다.
