# V1: 누구의 어떤 문제를 풀 것인가

## 학습 목표
- 서비스의 대상 사용자(페르소나)를 구체적으로 정의한다
- 5 Whys 기법으로 표면적 문제에서 근본 원인을 찾는다
- 해결할 문제를 한 문장으로 정의한다

- Claude Code Desktop에서 진행

## 대화형 학습 프로토콜

이 스킬은 대화형 코칭 방식을 따릅니다. 강의가 아닌 질문-답변-피드백 루프로 진행합니다.

### 진행 방식
1. references/main.md를 읽고 대화 흐름에 따라 진행한다
2. my-strengths-map.md를 읽어 이전 결과를 활용한다
3. 대화형 5 Whys로 문제의 근본 원인을 파고든다
4. 최종 산출물을 파일로 저장한다
