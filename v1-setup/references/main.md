## EXPLAIN

### 서비스 빌딩에 필요한 도구들

서비스를 만들기 위해 고가의 장비나 소프트웨어가 필요하지 않습니다. 노트북 한 대와 인터넷만 있으면 됩니다.

필요한 것:
1. **Claude Pro/Max 계정** ($20/월~) — AI 코딩 파트너
2. **Claude Code** (무료) — 터미널에서 AI와 대화하며 코딩
3. **Node.js** (무료) — JavaScript 실행 환경
4. **Git** (무료) — 코드 버전 관리
5. **호스팅** ($0~7/월) — 서비스를 인터넷에 공개

### 비용 정리 — 솔직하게

> **중요**: 비용을 과장하거나 축소하지 않겠습니다. 정확하게 알려드릴게요.

| 항목 | 비용 | 필수 여부 | 설명 |
|------|------|-----------|------|
| Claude Pro | $20/월 (약 27,000원) | **필수** | AI 코딩 파트너. 이게 없으면 서비스를 만들 수 없어요 |
| Claude Max | $100/월 (약 135,000원) | 선택 | 더 많이 사용 가능. 처음엔 Pro로 충분 |
| 호스팅 (Railway) | $5~7/월 (약 7,000~9,500원) | 출시할 때 | 서비스를 인터넷에 올리는 비용 |
| 호스팅 (Render) | $0 (무료 티어) | 출시할 때 | 무료 가능하지만 속도가 느림 |

### 환경별 설치 안내

**Windows 사용자:**
- Windows 10 또는 11 필요 (Windows 7/8은 지원 안 됨)
- Claude Code Desktop 앱 사용 (가장 쉬움)
- 또는 WSL2 + 터미널 사용 (고급)
- Node.js: nodejs.org에서 LTS 버전 다운로드
- Git: git-scm.com에서 다운로드

**Mac 사용자 (Intel):**
- macOS 12 이상 권장
- 터미널 기본 제공 (Spotlight에서 "터미널" 검색)
- Node.js: `brew install node` 또는 nodejs.org
- Git: Xcode Command Line Tools에 포함 (`xcode-select --install`)

**Mac 사용자 (Apple Silicon — M1/M2/M3/M4):**
- macOS 12 이상 권장
- 터미널 기본 제공
- Homebrew 설치: `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`
- Node.js: `brew install node`
- Git: `xcode-select --install`

> **Mac인데 Intel인지 Apple Silicon인지 모르겠다면:** 화면 왼쪽 상단 🍎 > "이 Mac에 관하여" > "칩" 항목 확인. "Apple M1/M2/M3/M4"면 Apple Silicon, "Intel"이면 Intel입니다.

**공통:**
- Claude Code Desktop: https://claude.ai/download 에서 다운로드
- Claude Code CLI: `npm install -g @anthropic-ai/claude-code` (C3에서 설치)

### Q1: 필수 비용
> "서비스를 만들기 위한 최소 월 비용은 대략 얼마인가요?"

선택지:
1. 무료 (0원)
2. 약 27,000원 (커피 3잔)
3. 약 100,000원
4. 수백만원

정답: 2번
해설: Claude Pro 구독($20/월, 약 27,000원)이 핵심 비용입니다. 호스팅은 무료 티어로 시작 가능하고, 도메인은 선택 사항입니다. 무료 경로도 있지만, 진지하게 서비스를 만들려면 Pro 구독을 권장합니다.

### Q2: Claude Code의 역할
> "Claude Code는 무엇인가요?"

선택지:
1. 코딩 학원 프로그램
2. AI와 대화하면서 코드를 작성하는 도구
3. 웹사이트 디자인 도구
4. 호스팅 서비스

정답: 2번
해설: Claude Code는 터미널에서 AI와 대화하며 코드를 작성하는 도구입니다. "이거 만들어줘"라고 말하면 AI가 코드를 작성하고 파일을 만듭니다. C3에서 본격적으로 사용합니다.

### Q3: 환경 설정 실패 시
> "Node.js를 설치했는데 `node --version`이 작동하지 않을 때 가장 먼저 해볼 것은?"

선택지:
1. Node.js를 다시 설치한다
2. 컴퓨터를 포맷한다
3. 터미널을 껐다가 다시 연다
4. 포기한다

정답: 3번
해설: 프로그램을 새로 설치한 후에는 터미널이 변경 사항을 인식하지 못할 수 있습니다. 터미널을 껐다가 다시 열면 대부분 해결됩니다. 그래도 안 되면 컴퓨터 재부팅을 시도해보세요.
