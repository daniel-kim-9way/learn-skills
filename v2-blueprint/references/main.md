## EXPLAIN

### 블루프린트 = 건축 설계도

아파트를 지을 때, 건축가가 건물주에게 설계도를 보여줍니다.

**건물주가 확인하는 것:**
- "내가 원한 3 LDK인가?" (방이 맞나?)
- "화장실이 2개 맞나?" (요구사항 반영됐나?)
- "주차장이 있나?" (빠진 건 없나?)
- "나중에 증축 가능한가?" (확장성)

**건물주가 확인하지 않는 것:**
- 철근 규격 (기술 상세)
- 배관 설계도 (내부 구조)
- 콘크리트 배합 비율 (구현 방식)
- 전기 배선도 (코드 구조)

건물주가 배관 설계도를 읽을 줄 알아야 하나요? **아닙니다.** 방 개수가 맞는지, 화장실 위치가 맞는지, 주차장이 있는지만 확인하면 됩니다.

**여러분은 서비스의 건물주입니다.** Claude가 설계도를 만들어 드릴 테니, 핵심만 확인하세요.

---

### 건물주가 봐야 할 딱 5가지

블루프린트를 받으면, 이것만 확인합니다. 5가지.

```
건물주 체크리스트

┌─────────────────────────────────────────────────┐
│                                                  │
│  1. 데이터 구조                                  │
│  "어떤 정보가 저장되는지 맞나?"                  │
│                                                  │
│  비유: 건물의 방 구조                            │
│  → "방이 몇 개인가?" = "데이터 테이블이 몇 개?"  │
│  → "각 방에 뭐가 있나?" = "각 테이블에 어떤       │
│     정보가 들어가나?"                             │
│                                                  │
│  확인할 것:                                      │
│  - 사용자 정보: 이름, 이메일 등이 다 있나?        │
│  - 핵심 데이터: 내 서비스의 핵심 정보가 빠짐없나? │
│  - 관계: 데이터끼리 연결이 맞나?                  │
│    (예: "주문"은 "사용자"에 연결되나?)            │
│                                                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  2. 화면 수                                      │
│  "필요한 화면이 다 있나?"                        │
│                                                  │
│  비유: 건물의 층수와 공간                        │
│  → "1층 로비(홈), 2층 사무실(기능),              │
│     3층 회의실(관리)" 같은 공간 배치              │
│                                                  │
│  확인할 것:                                      │
│  - my-user-flow.md의 흐름도에 나온 화면이        │
│    전부 블루프린트에 있나?                        │
│  - 에러 화면, 빈 상태 화면도 있나?               │
│  - 모바일에서도 볼 수 있나?                      │
│                                                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  3. 외부 연동                                    │
│  "다른 서비스와 연결할 것이 있나?"               │
│                                                  │
│  비유: 건물의 수도/전기/가스 연결                │
│  → 수도(결제 시스템), 전기(이메일 발송),         │
│     가스(소셜 로그인)처럼 외부에서 가져오는 것    │
│                                                  │
│  확인할 것:                                      │
│  - 결제가 필요한가? → 토스페이먼츠, 카카오페이    │
│  - 소셜 로그인? → 카카오, 네이버, 구글 로그인    │
│  - 알림? → 이메일, 카카오 알림톡                 │
│  - 지도? → 카카오맵, 네이버 지도                 │
│                                                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  4. 인증 방식                                    │
│  "로그인은 어떻게 하나?"                         │
│                                                  │
│  비유: 건물의 출입 시스템                        │
│  → 열쇠(이메일/비밀번호), 카드키(소셜 로그인),   │
│     얼굴인식(생체 인증)                          │
│                                                  │
│  확인할 것:                                      │
│  - 이메일/비밀번호 로그인인가?                   │
│  - 소셜 로그인을 지원하나? (카카오/네이버)       │
│  - 비밀번호를 잊으면 어떻게 하나?                │
│  - 비회원도 일부 기능을 쓸 수 있나?              │
│                                                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  5. 배포 환경                                    │
│  "어디에 올릴 건가?"                             │
│                                                  │
│  비유: 건물의 위치와 주소                        │
│  → "서울 강남"(클라우드 서버)처럼                │
│     서비스가 인터넷에 올라갈 장소                 │
│                                                  │
│  확인할 것:                                      │
│  - 어디에 배포하나? (Railway, Render 등)         │
│  - 도메인(주소)이 있나?                          │
│  - HTTPS(보안 연결)를 지원하나?                  │
│                                                  │
└─────────────────────────────────────────────────┘
```

이 5가지 외에 기술적인 내용은 **이해 못 해도 괜찮습니다.** 코드를 읽을 필요 없고, 기술 용어를 외울 필요 없습니다.

---

### "이해 못 해도 OK" — 건물주의 마인드셋

블루프린트에는 기술 용어가 많이 나올 수 있습니다.

```
블루프린트에서 볼 수 있는 용어:
  - PostgreSQL, SQLite → "데이터를 저장하는 창고 종류"
  - REST API → "층과 층 사이의 통신 방법"
  - JWT 인증 → "출입증 발급 방식"
  - Devise → "로그인 시스템을 만드는 도구"
  - Active Storage → "사진/파일을 저장하는 방법"
```

이 용어를 이해할 필요 없습니다. 여러분이 해야 할 것은 **의사결정**입니다:

- "이 저장 방식으로 하면 나중에 사진 업로드가 안 되나요?" → 의사결정
- "PostgreSQL이 뭔지" → 알 필요 없음

모르는 용어가 나오면, 이렇게 물어보세요:

```
유용한 질문 패턴:

"이 부분을 레스토랑으로 비유하면 뭔가요?"
→ Claude가 쉽게 설명해줍니다

"이 결정이 나중에 변경 가능한가요?"
→ 돌이킬 수 없는 결정인지 확인

"이걸 왜 이렇게 결정한 건가요? 다른 방법은 없나요?"
→ 대안이 있는지 확인

"이게 사용자에게 어떤 영향을 미치나요?"
→ 기술이 아닌 사용자 관점으로 변환

"더 쉽게 설명해주세요. 비개발자도 이해할 수 있게요."
→ 이해될 때까지 질문하는 것이 서비스 디렉터의 능력
```

---

> **꿀팁: 블루프린트 검토에서 가장 중요한 한 가지**
>
> "내가 원한 것이 다 들어갔나?"
>
> 이것만 확인하면 80%는 성공입니다. my-priority-list.md의 Must 기능이
> 블루프린트에 전부 있는지 하나하나 대조해보세요.
>
> 코드가 효율적인지, 아키텍처가 최선인지는 Claude가 알아서 합니다.
> 여러분은 "내가 원한 게 맞는지"만 확인하면 됩니다.

## EXECUTE

### Turn 1: 건물주 비유 소개

학생에게 말합니다:

"지금부터 [서비스명]의 기술 설계도를 만들어드릴게요. 코드가 보여도 걱정 마세요 — 여러분은 **건물주 체크리스트** 5가지만 확인하면 됩니다."

AI가 학생의 서비스에 맞는 기술 블루프린트를 생성합니다:

### Turn 2: 건물주 체크리스트 검증

**체크 2. 화면 수 (공간 배치)**

"화면 목록을 볼게요. my-user-flow.md에서 그린 흐름도의 화면이 다 있나요?"

각 흐름의 화면을 하나씩 대조합니다.

**체크 3. 외부 연동 (수도/전기/가스)**

"다른 서비스와 연결해야 하는 게 있나요? 결제? 소셜 로그인? 알림?"

- "사용자에게 알림을 보내야 하나요?" → 알림 연동 필요

**체크 4. 인증 방식 (출입 시스템)**

"로그인은 어떻게 할 건가요?"
- 이메일/비밀번호만?
- 카카오/네이버 소셜 로그인도?
- 비밀번호 잊으면 재설정 가능?

**체크 5. 배포 환경 (건물 위치)**

"서비스를 어디에 올릴 건가요?"
- 간단하게 설명: "인터넷에 여러분의 서비스를 올리는 장소를 정하는 거예요."
- C4에서 자세히 다룰 것이므로, 지금은 "Railway"나 "Render" 같은 서비스를 사용할 거라고 안내합니다.

- (C) 내가 원한 기능이 다 들어갔는지, 데이터가 안전한지
- (D) 프로그래밍 언어가 최신인지

정답: (C) — 서비스 디렉터는 코드의 품질이나 기술적 최적화가 아닌, "내 요구사항이 반영됐는지"와 "사용자에게 안전한지"를 확인합니다. 건물주가 철근 규격 대신 방 개수를 확인하는 것과 같습니다.

### 질문 2: 기술 소통

"블루프린트에서 이해 안 되는 기술 용어가 나왔을 때 가장 좋은 방법은?"

- (A) 무시하고 넘어간다
- (B) 인터넷에서 직접 검색한다
- (C) Claude에게 비유를 들어 설명해달라고 한다
- (D) 그 부분은 기술자에게 맡기고 신경 쓰지 않는다

정답: (C) — 이해 안 되는 부분을 그냥 넘기면 나중에 원하지 않는 결과가 나올 수 있습니다. Claude에게 비유로 설명해달라고 하면, 기술 용어를 몰라도 핵심을 이해할 수 있습니다. "이해될 때까지 질문하는 것"이 서비스 디렉터의 가장 중요한 능력입니다.

### 질문 3: 건물주의 역할

"건물주가 건축가에게 가장 많이 해야 하는 질문은?"

- (A) "이 철근의 인장 강도는 얼마인가요?"
- (B) "내가 원한 3 LDK가 맞나요? 주차장은 있나요?"
- (C) "배관을 PVC로 하나요 동관으로 하나요?"
- (D) "콘크리트 양생 기간은 며칠인가요?"

정답: (B) — 건물주는 기술적 세부사항이 아닌, 자신의 요구사항이 설계에 반영되었는지를 확인합니다. "내가 원한 게 맞나?"와 "빠진 건 없나?"가 건물주의 핵심 질문입니다.
