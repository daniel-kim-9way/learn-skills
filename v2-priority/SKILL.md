# 기능 우선순위 정하기

## 학습 목표
- MoSCoW 방법론을 활용하여 기능 우선순위를 결정할 수 있다
- "첫 버전(MVP)에 뭘 넣을 것인가"를 명확히 결정한다
- "하고 싶은 것"과 "해야 하는 것"을 구분하는 사고력을 기른다

## 사전 조건
- `/v2-feature-spec` 완료 (my-feature-spec.md 존재)
- Claude Code Desktop에서 진행

## 대화형 학습 프로토콜

### 진행 방식
1. references/main.md를 읽고 대화 흐름에 따라 진행한다
2. 이사짐 게임 비유로 우선순위 개념을 체험시킨다
3. 기능 명세서의 모든 기능을 MoSCoW로 하나씩 분류한다
4. Must-have만 모은 MVP 기능 목록을 파일로 저장한다
