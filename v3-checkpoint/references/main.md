## EXPLAIN

### "시사회" — 내 서비스의 첫 상영

영화가 완성되면 시사회를 합니다. 감독, 배우, 관계자들이 모여서 처음부터 끝까지 영화를 봅니다. 빠진 장면은 없는지, 소리가 안 나는 부분은 없는지, 흐름이 자연스러운지 꼼꼼히 점검합니다.

여러분의 서비스도 지금 "시사회" 시간입니다.

L1(에디터 설치)부터 L9(두 번째 기능)까지 만든 모든 것을 처음부터 끝까지 점검합니다. 새로운 것을 만드는 레슨이 아니라, **만든 것을 점검하고 마무리하는 레슨**입니다.

### GitHub = 클라우드 세이브

v3-git-save에서 Git으로 "로컬 세이브"를 배웠습니다. 내 컴퓨터에만 저장된 상태였죠.

그런데 만약:
- 컴퓨터가 고장나면?
- 노트북을 카페에 두고 오면?
- 하드디스크가 날아가면?

**로컬 세이브만으로는 불안합니다.**

**GitHub**는 코드를 인터넷에 저장하는 서비스입니다.

| 게임 세이브 | Git/GitHub |
|------------|-----------|
| 게임기 안에 저장 (로컬) | **Git** — 내 컴퓨터에만 저장 |
| 클라우드에 저장 (닌텐도 온라인 등) | **GitHub** — 인터넷에 저장 |
| 게임기 고장 → 세이브 사라짐 | 컴퓨터 고장 → Git 세이브 사라짐 |
| 다른 게임기에서 이어하기 가능 | 다른 컴퓨터에서 코드 가져오기 가능 |

GitHub에 올리면 얻는 것:
1. **안전**: 컴퓨터가 고장나도 코드가 안전합니다
2. **이동성**: 다른 컴퓨터에서도 코드를 가져올 수 있습니다
3. **포트폴리오**: "이것이 제가 만든 서비스입니다" 보여줄 수 있습니다
4. **협업 준비**: 나중에 다른 사람과 함께 만들 때 필요합니다

GitHub는 **무료**입니다. 개인 프로젝트는 무료로 무제한 저장 가능합니다.

### L1부터 L10까지, 여러분의 여정

돌아보면 여러분은 이런 여정을 거쳤습니다:

| 레슨 | 한 것 | 비유 |
|------|-------|------|
| L1 (v3-setup) | 에디터 + Claude Code 설치 | 작업실 차리기 |
| L2 (v3-backward) | 샘플 앱 체험 | 완성 사진 보기 (레고) |
| L3 (v3-memory) | CLAUDE.md 작성 | AI에게 온보딩 문서 주기 |
| L4 (v3-rails-start) | Rails 프로젝트 생성 | 서비스의 첫 심장 박동 |
| L5 (v3-git-save) | Git 세이브 | 게임 세이브 시작 |
| L6 (v3-database) | 데이터베이스 + 모델 | 엑셀 시트 만들기 |
| L7 (v3-auth) | 회원가입/로그인 | 집 열쇠 달기 |
| L8 (v3-feature-1) | 핵심 기능 1 | 감독의 첫 촬영 (가이드 있음) |
| L9 (v3-feature-2) | 핵심 기능 2 | 자율 촬영 (직접 주도) |
| **L10 (v3-checkpoint)** | **전체 점검 + GitHub** | **시사회 + 클라우드 세이브** |

C1에서 아이디어만 있던 사람이, C2에서 기획서를 만들었고, 이제 **실제로 동작하는 서비스**를 가지고 있습니다.

10개 레슨 전에는 "VS Code가 뭐지?" 했던 사람이, 지금은 서비스를 만들고 있습니다.

### C4에서 하게 될 것: localhost → 인터넷!

C3를 완료한 여러분의 서비스는 아직 `localhost:3000`에서만 동작합니다. 내 컴퓨터에서만 볼 수 있는 "비밀 서비스"입니다.

C4(LAUNCH)에서는 이 비밀 서비스를 **인터넷에 공개**합니다:
- 진짜 URL이 생깁니다 (예: my-service.fly.dev)
- 그 URL을 누구에게든 보내면 스마트폰으로도 볼 수 있습니다
- 카카오톡으로 공유할 수 있습니다
- 이력서에 쓸 수 있습니다

C1에서 꿈꿨던 **"URL 하나가 이력서를 이긴다"**가 현실이 됩니다.

하지만 그 전에, 지금 만든 서비스를 꼼꼼히 점검하고 안전하게 백업합시다.

### 단계 3: GitHub 계정 만들기

> 막히면:

### 단계 4: GitHub에 코드 올리기 — 첫 push!

이것이 "클라우드 세이브"의 순간입니다.

Claude에게 이렇게 요청하세요:
```
이 프로젝트를 GitHub에 올리고 싶어.

저장소(repository) 이름: [서비스이름]
공개 설정: public (포트폴리오로 쓸 거야)

GitHub에 올리는 전체 과정을 안내해줘.
```

Claude가 단계별로 안내합니다:

**과정 요약:**
1. GitHub에서 새 저장소(repository) 생성
2. 내 컴퓨터의 프로젝트와 GitHub 저장소 연결
3. `git push` — 코드를 인터넷에 업로드!

중간에 **인증(Authentication)**이 필요합니다. GitHub에 "이 사람이 진짜 이 계정 주인이다"를 증명해야 합니다.

방법이 여러 가지인데, Claude가 가장 적합한 방법을 안내해줍니다. SSH 키, Personal Access Token 등의 용어가 나올 수 있지만, Claude의 안내를 따라가면 됩니다.

> 에러가 나면: GitHub 연결은 인증 관련 에러가 자주 납니다. 당황하지 마세요.
> ```
> git push했더니 이런 에러가 나왔어: [에러 메시지]. 어떻게 해?
> ```
> Claude가 해결책을 제시합니다.

### 단계 5: GitHub에서 내 코드 확인 — "클라우드 세이브 완료!"

브라우저에서 GitHub에 접속합니다:

```
https://github.com/[내 사용자이름]/[서비스이름]
```

이 페이지에서 확인할 것:
- 내 프로젝트의 파일 목록이 보인다
- app/, config/, db/ 등 익숙한 폴더들이 보인다
- Git 커밋 기록(세이브 기록)이 보인다

**내 코드가 인터넷에 올라가 있습니다!**

이것이 "클라우드 세이브"입니다. 이제 컴퓨터가 고장나도, 노트북을 잃어버려도, 코드는 안전합니다. 다른 컴퓨터에서 GitHub에 접속해서 코드를 내려받으면 이어서 작업할 수 있습니다.

> 꿀팁: 이 GitHub URL을 메모해두세요. 나중에 포트폴리오로 쓸 수 있습니다. "제가 만든 서비스 코드입니다"라고 링크를 보낼 수 있죠.

### 단계 7: 여정 회고 — "나는 서비스 빌더입니다"

잠시 멈추고 돌아봅시다.

**C1에서의 나:**
- 서비스를 만들고 싶지만, 코딩을 모른다
- "나 같은 비개발자가 가능할까?"
- VS Code가 뭔지도 몰랐다

**지금의 나:**
- 실제로 동작하는 서비스를 가지고 있다
- 회원가입, 로그인, 핵심 기능 2개가 작동한다
- GitHub에 코드가 올라가 있다
- AI에게 지시하면 무엇이든 만들 수 있다는 걸 안다

10개 레슨 동안 여러분은:
- 코드 에디터를 설치했고 (L1)
- 완성된 서비스의 구조를 파악했고 (L2)
- AI에게 프로젝트 맥락을 심어줬고 (L3)
- Rails 프로젝트를 만들었고 (L4)
- Git으로 안전하게 저장하기 시작했고 (L5)
- 데이터베이스를 설계하고 만들었고 (L6)
- 회원 시스템을 구축했고 (L7)
- 핵심 기능을 가이드와 함께 만들었고 (L8)
- 핵심 기능을 직접 주도해서 만들었고 (L9)
- 전체를 점검하고 GitHub에 업로드했습니다 (L10)

**코딩을 배운 게 아닙니다.** Ruby 문법도, HTML 태그도, SQL 쿼리도 배우지 않았습니다.

**AI와 함께 서비스를 만드는 방법**을 배운 것입니다.

그리고 이 방법은 어떤 서비스에든 적용됩니다. 다음에 새로운 아이디어가 떠오르면, 같은 패턴으로 만들 수 있습니다. 여러분은 이미 **서비스 빌더**입니다.

- 2번째 질문 → "Git은 내 컴퓨터에 저장, GitHub는 인터넷에 저장"
- 3번째 질문 → "localhost에서만 보이던 서비스를 인터넷에 공개한다"
- 4번째 질문 → "AI와 함께 서비스를 만드는 방법"

해설:
- Git은 내 컴퓨터(로컬)에서 세이브 포인트를 관리하는 도구이고, GitHub는 그 세이브를 인터넷(클라우드)에 백업하는 서비스입니다. 게임기의 로컬 세이브와 클라우드 세이브의 관계와 같습니다. 둘 다 무료입니다.
- C4에서는 localhost:3000에서만 보이던 서비스를 실제 인터넷에 배포합니다. URL을 친구에게 보내면 스마트폰에서도 볼 수 있게 됩니다. 그리고 랜딩 페이지, AI 기능, 결제까지 추가합니다.
- C3에서 여러분은 Ruby, HTML, SQL 같은 프로그래밍 언어를 배운 게 아닙니다. "계획 → 조각내기 → 지시 → 확인 → 반복 → 세이브" 패턴으로 AI와 함께 서비스를 만드는 방법을 배웠습니다. 이 방법은 어떤 서비스에든 적용됩니다.

C3 BUILD 챕터를 완료하신 것을 진심으로 축하합니다. 10개 레슨 전에는 "나 같은 비개발자가 서비스를 만들 수 있을까?" 했던 여러분이, 지금은 실제로 동작하는 서비스와 GitHub 포트폴리오를 갖게 되었습니다. 여러분은 서비스 빌더입니다. C4에서 만나요!
