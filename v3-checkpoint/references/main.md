## EXPLAIN

### "시사회" — 내 서비스 상영

영화가 완성되면 시사회를 합니다. 감독과 관계자들이 모여 처음부터 끝까지 영화를 보면서 빠진 부분은 없는지, 흐름이 자연스러운지 점검합니다.

여러분의 서비스도 지금 "시사회" 시간입니다. L1(에디터 설치)부터 L9(두 번째 기능)까지 만든 모든 것을 처음부터 끝까지 점검합니다.

### GitHub = 클라우드 세이브

지금까지 Git으로 "로컬 세이브"를 해왔습니다. 내 컴퓨터에만 저장된 상태죠. 그런데 만약 컴퓨터가 고장나면? 노트북을 잃어버리면?

**GitHub**는 코드를 인터넷에 저장하는 서비스입니다. 비유하면:
- **Git (로컬 세이브)** = 게임기 안에 저장 (게임기가 고장나면 세이브 사라짐)
- **GitHub (클라우드 세이브)** = 클라우드에 저장 (게임기가 고장나도 다른 기기에서 이어하기)

GitHub에 올리면:
1. 컴퓨터가 고장나도 코드가 안전합니다
2. 다른 컴퓨터에서도 코드를 가져올 수 있습니다
3. 나중에 다른 사람과 협업할 수 있습니다

### L1 ~ L10 여정

돌아보면 여러분은 이런 여정을 거쳤습니다:

| 레슨 | 한 것 | 비유 |
|------|-------|------|
| L1 (v3-setup) | 에디터 + Claude Code 설치 | 작업실 차리기 |
| L2 (v3-backward) | 샘플 앱 체험 | 완성 사진 보기 |
| L3 (v3-memory) | CLAUDE.md 작성 | AI 인수인계 문서 |
| L4 (v3-rails-start) | Rails 프로젝트 생성 | 첫 심장 박동 |
| L5 (v3-git-save) | Git 세이브 포인트 | 게임 세이브 시작 |
| L6 (v3-database) | 데이터베이스 + 모델 | 엑셀 시트 만들기 |
| L7 (v3-auth) | 회원가입/로그인 | 집 열쇠 만들기 |
| L8 (v3-feature-1) | 핵심 기능 1 | 감독의 첫 촬영 |
| L9 (v3-feature-2) | 핵심 기능 2 | 자율 촬영 |
| L10 (v3-checkpoint) | 전체 점검 + GitHub | 시사회 + 클라우드 세이브 |

C1에서 아이디어만 있던 사람이, 이제 실제로 동작하는 서비스를 가지고 있습니다.

### C4 미리보기

C3를 완료한 여러분의 서비스는 아직 localhost에서만 동작합니다. 내 컴퓨터에서만 볼 수 있는 "비밀 서비스"입니다.

C4(LAUNCH)에서는 이 비밀 서비스를 **인터넷에 공개**합니다. URL을 누구에게든 보내면 스마트폰으로도 볼 수 있게 됩니다. C1에서 꿈꿨던 "URL 하나가 이력서를 이긴다"가 현실이 됩니다.

## EXECUTE

### 단계 1: 전체 체크리스트 테스트

서버를 시작하고, 브라우저에서 처음부터 끝까지 테스트합니다.

**홈페이지**
- [ ] 서비스 이름과 소개가 보인다
- [ ] "시작하기" 버튼이 있다
- [ ] 디자인이 깔끔하다

**인증**
- [ ] 회원가입이 된다
- [ ] 로그인이 된다
- [ ] 로그아웃이 된다
- [ ] 네비게이션이 상태에 맞게 변한다

**핵심 기능 1**
- [ ] 데이터 추가가 된다
- [ ] 데이터 목록이 보인다
- [ ] 데이터 수정이 된다
- [ ] 데이터 삭제가 된다

**핵심 기능 2**
- [ ] 기능이 동작한다
- [ ] 기능 1과 연결이 자연스럽다

**전체 흐름**
- [ ] 비로그인 → 홈 → 시작하기 → 회원가입 → 기능 사용 → 로그아웃 전체 흐름 가능

### 단계 2: 문제 수정

체크리스트에서 문제가 발견되면 Claude에게 수정을 요청합니다:
```
체크리스트 테스트 중에 [문제 설명]을 발견했어.
수정해줘.
```

모든 항목이 통과할 때까지 반복합니다.

### 단계 3: GitHub 계정 준비

GitHub 계정이 없다면 만듭니다:
```
GitHub 계정을 만들고 싶어. 어떻게 하면 돼?
```

이미 있다면 이 단계를 건너뜁니다.

### 단계 4: GitHub에 코드 올리기 (첫 push)

Claude에게 이렇게 요청하세요:
```
이 프로젝트를 GitHub에 올려줘.
저장소 이름은 [서비스이름]으로 해줘.
```

Claude가 안내하는 단계를 따라 진행합니다:
1. GitHub에서 새 저장소(repository) 생성
2. 로컬 프로젝트와 연결
3. git push (코드 업로드)

SSH 키나 토큰 설정이 필요할 수 있습니다. 오류가 나면:
```
이런 오류가 나. [오류 메시지]. 해결해줘.
```

### 단계 5: GitHub에서 확인

브라우저에서 `https://github.com/[사용자이름]/[서비스이름]`에 접속합니다.

내 코드가 인터넷에 올라가 있는 것을 확인합니다! 이것이 "클라우드 세이브"입니다. 이제 컴퓨터가 고장나도 코드는 안전합니다.

### 단계 6: 최종 Git 세이브 + CLAUDE.md 업데이트

Claude에게 이렇게 요청하세요:
```
CLAUDE.md의 현재 상태를 업데이트해줘.
"C3 BUILD 전체 완료. 홈페이지, 인증, 핵심 기능 2개 구현. GitHub 업로드 완료."로 바꿔줘.
그리고 Git에 저장하고 GitHub에도 올려줘. 메시지는 "C3 체크포인트: 전체 점검 완료, GitHub 업로드"로.
```

### 단계 7: 여정 회고

잠시 멈추고 돌아봅시다.

**C1에서의 나**: 서비스를 만들고 싶지만, 코딩을 모른다
**지금의 나**: 실제로 동작하는 서비스를 가지고 있다

10개 레슨을 통해 여러분은:
- 코드 에디터를 설치했고
- AI에게 기억을 심어줬고
- 프로젝트를 만들었고
- 데이터베이스를 설계했고
- 회원 시스템을 구축했고
- 핵심 기능 2개를 직접 지시해서 만들었고
- GitHub에 업로드했습니다

코딩을 배운 게 아닙니다. **AI와 함께 서비스를 만드는 방법**을 배운 것입니다.
