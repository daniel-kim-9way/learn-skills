## EXPLAIN

### "영화 감독" 마인드셋

영화 감독은 직접 연기하지 않습니다.

감독이 하는 일:
1. **비전을 세운다** — "이 장면은 이런 분위기여야 해"
2. **배우에게 지시한다** — "여기서 슬프게 연기해봐"
3. **결과를 확인한다** — 모니터로 촬영본을 본다
4. **피드백을 준다** — "좋은데, 조금 더 감정을 넣어봐"
5. **만족하면 다음으로** — "OK! 다음 장면!"

여러분도 영화 감독입니다:
- **직접 코딩하지 않습니다** — 그건 Claude의 일
- **무엇을 만들지 계획합니다** — 감독의 비전
- **Claude에게 지시합니다** — "이런 기능을 만들어줘"
- **결과를 확인합니다** — 브라우저에서 직접 써봄
- **피드백을 줍니다** — "이 부분은 이렇게 바꿔줘"
- **만족하면 다음으로** — "좋아! 다음 조각!"

이것이 AI 시대의 서비스 빌딩 방식입니다. 코딩 실력이 아니라 **지시하는 실력**이 중요합니다.

### 큰 기능을 작은 조각으로 나누기

"독서 모임에서 책을 관리하는 기능"을 Claude에게 한 번에 요청하면 어떻게 될까요?

할 수는 있습니다. 하지만 문제가 생기면 **어디서 생겼는지 찾기가 매우 어렵습니다.** 마치 10개 재료를 한꺼번에 냄비에 넣고 "맛이 이상한데 뭐가 문제지?" 하는 것과 같습니다.

대신 작은 조각으로 나눕니다:

1. 책 목록 페이지 (저장된 책들 보여주기)
2. 책 추가 폼 (새 책 등록)
3. 책 상세 페이지 (책 하나의 자세한 정보)
4. 책 수정 기능
5. 책 삭제 기능
6. 디자인 다듬기

작은 조각으로 나누면:
- **한 조각이 5~10분 안에 끝남** → 빠른 성취감
- **중간중간 확인할 수 있음** → "여기까지는 잘 되네!"
- **문제가 생겨도 찾기 쉬움** → "방금 만든 이 조각에서 문제가 생겼구나"
- **하나씩 쌓이는 느낌** → 레고 블록 쌓듯이

### 구현 패턴: 계획 → 조각내기 → 지시 → 확인 → 반복 → 세이브

이 패턴이 C3 전체에서 **가장 중요한 것**입니다:

1. **계획**: C2 기획서에서 만들 기능을 확인한다
2. **조각내기**: 기능을 작은 단위로 나눈다
3. **지시**: Claude에게 첫 번째 조각을 요청한다
4. **확인**: 브라우저에서 결과를 확인한다
5. **반복**: 다음 조각을 요청한다 (문제가 있으면 수정 요청)
6. **세이브**: 전체가 완성되면 git commit

이 패턴을 외울 필요는 없습니다. 이번 레슨에서 **몸으로 익힙니다.** 한 번 해보면 자연스럽게 체득됩니다.

### 에러가 나면?

기능을 만들다 보면 에러가 반드시 납니다. 이것은 실패가 아닙니다. 에러는 "여기가 아직 안 됐어요"라고 알려주는 **친절한 안내판**입니다.

에러가 나면:
```
이런 에러가 나왔어: [에러 메시지를 그대로 복사-붙여넣기]. 해결해줘.
```

이 한 마디면 됩니다. Claude가 에러를 분석하고 수정합니다. 프로 개발자들도 매일 이렇게 합니다.

## EXECUTE

### 단계 1: C2 핵심 기능 확인 + 조각내기

Claude에게 이렇게 요청하세요:
```
CLAUDE.md를 읽고, 내 서비스의 핵심 기능 중 첫 번째 기능이 뭔지 알려줘.

이 기능을 구현하려면 어떤 작은 조각들이 필요한지 목록으로 만들어줘.
각 조각이 뭘 하는지 한 줄로 설명해줘.
예상 소요 시간도 각각 알려줘.
```

Claude가 기능을 분석하고 조각 목록을 만들어줍니다.

예를 들어 "독서 목록 관리" 기능이라면:

| 조각 | 할 일 | 예상 시간 |
|------|-------|----------|
| 1 | 책 목록 페이지 — 저장된 책들을 카드로 보여줌 | 10분 |
| 2 | 책 추가 폼 — 제목, 저자, 장르를 입력하는 페이지 | 10분 |
| 3 | 책 저장 — 폼에서 입력한 데이터를 DB에 저장 | 5분 |
| 4 | 책 상세 보기 — 책 하나의 상세 정보 페이지 | 5분 |
| 5 | 책 수정 — 이미 등록한 책의 정보를 고침 | 10분 |
| 6 | 책 삭제 — 등록한 책을 삭제 | 5분 |
| 7 | 디자인 정리 — 전체적으로 깔끔하게 다듬기 | 10분 |

이 목록이 여러분의 **촬영 스케줄**입니다. 이제 첫 번째 장면부터 촬영합니다.

### 단계 2: 첫 번째 조각 — 목록 페이지

Claude에게 이렇게 요청하세요:
```
그러면 첫 번째 조각부터 시작하자.
[핵심 데이터]의 목록 페이지를 만들어줘.

조건:
- 로그인한 사용자만 볼 수 있어야 해
- 데이터가 없을 때 "아직 등록된 [데이터]가 없습니다. 첫 번째를 추가해보세요!" 안내 메시지
- 데이터가 있으면 카드 형태로 보여줘
- 네비게이션에 이 페이지로 가는 링크도 추가해줘
- Tailwind CSS로 깔끔하게
```

Claude가 필요한 파일들을 만듭니다:
- 컨트롤러 (요청 처리)
- 뷰 (화면)
- 라우트 (URL 경로)

**브라우저에서 확인합니다:**
1. 로그인한 상태에서 목록 페이지에 접근합니다
2. "아직 등록된 데이터가 없습니다" 메시지가 보이나요?
3. 네비게이션에서 이 페이지로 이동할 수 있나요?

> 에러가 나면: 에러 메시지를 Claude에게 그대로 보여주세요.

확인 완료! 첫 번째 조각 성공!

### 단계 3: 두 번째 조각 — 추가 폼 + 저장

Claude에게 이렇게 요청하세요:
```
다음 조각! [데이터]를 추가하는 폼 페이지를 만들어줘.

입력 필드:
- [칼럼1]: [타입 설명]
- [칼럼2]: [타입 설명]
- [칼럼3]: [타입 설명]
(v3-database에서 정한 칼럼들)

"저장" 버튼을 누르면 DB에 저장되고 목록 페이지로 돌아가야 해.
목록 페이지에 "새로 추가" 버튼도 넣어줘.
Tailwind CSS로 깔끔한 폼으로 만들어줘.
```

**브라우저에서 확인합니다:**
1. "새로 추가" 버튼이 목록 페이지에 보이나요?
2. 버튼을 클릭하면 입력 폼이 나오나요?
3. 데이터를 입력하고 "저장"을 누르면?
4. 목록 페이지로 돌아가고, 방금 추가한 데이터가 보이나요?

> 저장이 안 되면:
> ```
> 데이터를 입력하고 저장 버튼을 눌렀는데 [어떤 현상]. 해결해줘.
> ```

2~3개 데이터를 추가해보세요. 목록에 카드가 쌓이는 것을 확인합니다.

### 단계 4: 세 번째 조각 — 상세 보기

Claude에게 이렇게 요청하세요:
```
다음 조각! [데이터] 상세 보기 페이지를 만들어줘.

목록에서 카드를 클릭하면 상세 페이지로 이동하게 해줘.
모든 정보가 보여야 하고, "목록으로 돌아가기" 링크도 있어야 해.
나중에 수정/삭제 버튼도 넣을 자리를 비워둬.
```

**브라우저에서 확인합니다:**
1. 목록에서 데이터를 클릭하면 상세 페이지가 나오나요?
2. 모든 정보가 보이나요?
3. "목록으로" 링크가 동작하나요?

### 단계 5: 네 번째 조각 — 수정 기능

Claude에게 이렇게 요청하세요:
```
다음 조각! [데이터] 수정 기능을 만들어줘.

상세 페이지에 "수정" 버튼을 넣어줘.
수정 페이지는 추가 폼과 비슷하게, 기존 데이터가 미리 입력된 상태로 보여줘.
수정 완료 후 상세 페이지로 돌아가야 해.
```

**브라우저에서 확인합니다:**
1. 상세 페이지에 "수정" 버튼이 보이나요?
2. 클릭하면 기존 데이터가 채워진 폼이 보이나요?
3. 수정하고 저장하면 상세 페이지에 반영되나요?

### 단계 6: 다섯 번째 조각 — 삭제 기능

Claude에게 이렇게 요청하세요:
```

삭제 후 목록 페이지로 돌아가야 해.

**브라우저에서 확인합니다:**

### 단계 7: 디자인 다듬기

기본 기능이 다 동작하면, 이제 **디자인을 다듬습니다**.

Claude에게 이렇게 요청하세요:
```
지금 만든 [기능명] 페이지들의 디자인을 깔끔하게 정리해줘.

- 목록: 카드 형태, 깔끔한 그림자, 호버 효과
- 추가/수정 폼: 입력 필드 정렬, 적절한 여백
- 상세: 정보가 보기 좋게 정리
- 빈 목록: 이모지나 아이콘과 함께 안내 메시지
- 버튼 스타일: 홈페이지와 통일
- 전체적으로 여백과 간격 정리

홈페이지, 네비게이션과 디자인이 통일되게 해줘.
```

브라우저에서 확인하고, 마음에 안 드는 부분을 추가로 요청합니다.

**체크리스트:**

### 되돌아보기

방금 여러분이 한 것을 정리하면:

1. C2 기획서에서 기능을 **확인**했고
2. 큰 기능을 작은 **조각**으로 나눴고
3. Claude에게 하나씩 **지시**했고
4. 매 조각마다 **확인**했고
5. 문제가 있으면 **수정 요청**했고
6. 완성되면 **세이브**했습니다

**계획 → 조각내기 → 지시 → 확인 → 반복 → 세이브**

이것이 AI와 함께 서비스를 만드는 기본 리듬입니다. 다음 레슨(v3-feature-2)에서 이 패턴을 더 자율적으로 적용해봅니다.
