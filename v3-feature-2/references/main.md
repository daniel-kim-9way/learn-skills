## EXPLAIN

### 반복의 힘: 패턴을 알면 뭐든 만든다

v3-feature-1에서 배운 패턴을 기억하시나요?

**계획 → 조각내기 → 지시 → 확인 → 반복 → 세이브**

이 패턴은 어떤 기능에든 적용됩니다. 독서 목록이든, 할 일 관리든, 예약 시스템이든, 리뷰 기능이든. 패턴은 같습니다.

이번 레슨에서는 같은 패턴으로 두 번째 핵심 기능을 만듭니다. 차이가 있다면 — **이번에는 여러분이 주도합니다.**

### 점진적 자율성 (Progressive Autonomy)

v3-feature-1에서는 제가 단계별로 "이렇게 해보세요"라고 안내했습니다. 이번에는 다릅니다:

- **여러분이** 기능을 조각으로 나눕니다
- **여러분이** Claude에게 직접 지시합니다
- **여러분이** 결과를 확인하고 피드백합니다
- 막힐 때만 도움을 요청합니다

이것은 "혼자 해보세요"가 아닙니다. "이제 할 수 있으니까 직접 해보세요"입니다.

### 두 번째 기능이 첫 번째보다 쉬운 이유

1. **패턴을 이미 안다**: 조각내기 → 지시 → 확인 사이클이 익숙하다
2. **프로젝트 구조를 안다**: 어디에 뭐가 있는지 감이 잡혀있다
3. **Claude와 대화하는 법을 안다**: 어떻게 요청해야 원하는 결과가 나오는지 경험했다
4. **오류가 무섭지 않다**: 오류 메시지를 Claude에게 보여주면 된다는 걸 안다

### 기능 통합

두 번째 기능은 첫 번째 기능과 함께 동작해야 합니다. 독립적으로 만든 뒤, 두 기능이 자연스럽게 연결되는지 확인합니다.

예시: 첫 번째 기능이 "책 관리"이고 두 번째가 "독서 감상"이라면, 책 상세 페이지에서 감상을 남길 수 있어야 합니다.

## EXECUTE

### 단계 1: 패턴 복습 + 기능 확인

Claude에게 이렇게 요청하세요:
```
CLAUDE.md를 읽고, 내 서비스의 핵심 기능 2번이 뭔지 알려줘.
```

그리고 스스로 생각해봅니다: "이 기능을 어떤 조각으로 나눌 수 있을까?"

힌트가 필요하면:
```
이 기능을 작은 조각으로 나누면 어떻게 될까?
v3-feature-1에서 했던 것처럼 목록으로 만들어줘.
```

### 단계 2: 직접 구현 시작

여기서부터는 **여러분이 주도**합니다.

v3-feature-1에서 배운 패턴을 기억하면서, Claude에게 직접 지시합니다:
```
[기능의 첫 번째 조각]을 만들어줘.
[구체적인 요구사항]
```

각 조각마다 브라우저에서 확인합니다.

### 단계 3: 막힐 때 도움 받기

막히면 이렇게 요청하세요:
```
[상황 설명]. 어떻게 하면 좋을까?
```

또는:
```
지금 [이런 오류]가 나. 해결해줘.
```

도움을 요청하는 것은 실력이 부족한 게 아닙니다. 프로 개발자도 매일 합니다.

### 단계 4: 기능 1과 통합

두 번째 기능이 완성되면, 첫 번째 기능과 연결합니다.

Claude에게 이렇게 요청하세요:
```
[기능 1]과 [기능 2]를 자연스럽게 연결해줘.
예를 들어, [기능 1의 상세 페이지]에서 [기능 2]에 접근할 수 있도록.
네비게이션에도 [기능 2] 링크를 추가해줘.
```

### 단계 5: 통합 테스트

전체 서비스를 처음부터 끝까지 테스트합니다:

체크리스트:
- [ ] 회원가입 → 로그인
- [ ] 기능 1 사용 (추가/조회/수정/삭제)
- [ ] 기능 2 사용 (추가/조회/수정/삭제)
- [ ] 기능 1 ↔ 기능 2 연결 동작
- [ ] 네비게이션으로 모든 페이지 이동 가능
- [ ] 로그아웃 후 접근 제한 확인

### 단계 6: Git 세이브 + CLAUDE.md 업데이트

Claude에게 이렇게 요청하세요:
```
변경사항을 Git에 저장해줘. 메시지는 "핵심 기능 2: [기능명] 구현 + 기능 1 통합"으로.
CLAUDE.md의 현재 상태도 "v3-feature-2 완료. 핵심 기능 2개 구현 및 통합."으로 업데이트해줘.
```

### 단계 7: 깨달음의 순간

여기서 잠깐 멈추고 생각해봅시다.

방금 여러분은 **두 번째 기능을 거의 혼자 만들었습니다.** 패턴만 알면 세 번째, 네 번째 기능도 같은 방식으로 만들 수 있습니다.

"계획 → 조각내기 → 지시 → 확인 → 반복 → 세이브"

이 패턴 하나로 어떤 서비스 기능이든 만들 수 있습니다. 이것이 AI 시대의 서비스 빌딩 스킬입니다.
