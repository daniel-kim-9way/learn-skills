## EXPLAIN

### 반복의 힘: "이 패턴을 알면 뭐든 만든다"

v3-feature-1에서 배운 패턴을 기억하시나요?

**계획 → 조각내기 → 지시 → 확인 → 반복 → 세이브**

이 패턴은 어떤 기능에든 적용됩니다. 독서 목록이든, 할 일 관리든, 예약 시스템이든, 리뷰 기능이든. 서비스의 종류가 달라도 패턴은 같습니다.

피아노를 배울 때와 비슷합니다. 첫 곡은 선생님이 손가락 하나하나 짚어주면서 배우지만, 두 번째 곡부터는 "악보 읽는 법"을 알기 때문에 스스로 연습할 수 있죠.

이번 레슨이 바로 "두 번째 곡"입니다.

### 점진적 자율성 (Progressive Autonomy)

v3-feature-1에서는 제가 단계별로 안내했습니다:
- "첫 번째 조각은 이것입니다"
- "Claude에게 이렇게 요청하세요"
- "브라우저에서 이것을 확인하세요"

이번에는 다릅니다:

- **여러분이** 기능을 분석합니다
- **여러분이** 조각으로 나눕니다
- **여러분이** Claude에게 직접 지시합니다
- **여러분이** 결과를 확인하고 피드백합니다
- 막힐 때만 도움을 요청합니다

이것은 "혼자 알아서 해"가 아닙니다. **"이제 할 수 있으니까 직접 해봐"**입니다.

자전거를 배울 때 처음에는 뒤에서 잡아주지만, 어느 순간 손을 놓죠. 넘어질 수도 있지만, 그래야 진짜로 탈 수 있게 됩니다. 지금이 그 순간입니다.

### 두 번째 기능이 첫 번째보다 쉬운 이유

걱정하지 마세요. 두 번째는 확실히 더 쉽습니다:

1. **패턴을 이미 안다**: "조각내기 → 지시 → 확인" 사이클이 몸에 익었다
2. **프로젝트 구조를 안다**: 어디에 뭐가 있는지 감이 잡혀있다
3. **Claude와 대화하는 법을 안다**: 어떻게 요청해야 원하는 결과가 나오는지 경험했다
4. **에러가 무섭지 않다**: 에러 메시지를 Claude에게 보여주면 해결된다는 걸 안다
5. **자신감이 있다**: "한 번 해봤으니까 또 할 수 있다"

첫 번째 기능에서 1시간 걸렸다면, 두 번째는 40분이면 될 수 있습니다.

### 기능 통합: 두 기능이 함께 동작하기

두 번째 기능은 독립적으로 만든 뒤, 첫 번째 기능과 **자연스럽게 연결**해야 합니다.

예시:
- 첫 번째 기능: "책 관리" / 두 번째 기능: "독서 감상"
  → 책 상세 페이지에서 감상을 남길 수 있어야 함

- 첫 번째 기능: "레시피 관리" / 두 번째 기능: "재료 관리"
  → 레시피 상세에서 재료 목록이 보여야 함

- 첫 번째 기능: "운동 기록" / 두 번째 기능: "운동 통계"
  → 기록 데이터를 바탕으로 통계를 보여줘야 함

이 연결이 "서비스답게" 만드는 핵심입니다. 기능이 따로 놀면 서비스가 아니라 기능 목록일 뿐이니까요.

### 에러와 친해지기

이번 레슨에서도 에러가 날 겁니다. 특히 기능 통합 단계에서 많이 납니다.

에러가 나면 이렇게 대처하세요:

**레벨 1 — 단순 에러:**
```
이런 에러가 나왔어: [에러 메시지]. 해결해줘.
```

**레벨 2 — 복잡한 에러:**
```
[기능 2]를 만들었는데 [기능 1]이 안 되기 시작했어.
어디서 충돌이 나는지 확인하고 해결해줘.
```

**레벨 3 — 모르겠을 때:**
```
지금 뭐가 문제인지도 잘 모르겠어.
현재 서비스 상태를 전체적으로 점검하고 문제를 찾아서 해결해줘.
```

어떤 레벨이든 괜찮습니다. 중요한 건 **에러를 무시하지 않고 Claude에게 보여주는 것**입니다.

## EXECUTE

### 단계 1: 두 번째 기능 확인 + 스스로 조각내기

Claude에게 이렇게 요청하세요:
```
CLAUDE.md를 읽고, 내 서비스의 핵심 기능 2번이 뭔지 알려줘.
```

Claude가 기능을 알려주면, **먼저 스스로 생각해보세요:**

> "이 기능을 어떤 조각으로 나눌 수 있을까?"

종이에 적어보거나, 머릿속으로 목록을 만들어보세요. v3-feature-1에서 했던 것을 떠올리면 됩니다:
- 목록 페이지
- 추가 폼
- 상세 보기
- 수정
- 삭제
- 디자인

스스로 생각해본 뒤, 확인이 필요하면:
```
이 기능을 작은 조각으로 나누면 어떻게 될까?
내가 생각한 건 이거야: [자신의 목록]
빠진 게 있으면 알려줘.
```

> 꿀팁: 스스로 나눈 것과 Claude가 제안한 것이 다를 수 있습니다. 그래도 괜찮습니다. "아, 이런 조각도 필요하구나" 배우는 과정입니다.

### 단계 2: 직접 구현 시작

여기서부터는 **여러분이 주도**합니다.

v3-feature-1에서는 제가 "Claude에게 이렇게 요청하세요"라고 알려드렸죠. 이번에는 여러분이 직접 요청합니다.

**힌트만 드릴게요:**

첫 번째 조각을 시작할 때:
```
[기능의 첫 번째 조각]을 만들어줘.
[구체적인 요구사항]
로그인한 사용자만 사용할 수 있어야 해.
Tailwind CSS로 깔끔하게.
```

요구사항을 **구체적으로** 쓸수록 Claude의 결과가 좋아집니다.

나쁜 요청: "감상 기능 만들어줘" (너무 모호)
좋은 요청: "책 상세 페이지 아래에 감상을 남기는 폼을 추가해줘. 감상 내용(text)과 별점(1~5, integer)을 입력받고, 저장하면 그 책의 감상 목록에 표시되게 해줘."

**매 조각마다 반드시 브라우저에서 확인합니다.** 이것이 핵심입니다.

### 단계 3: 나머지 조각들

같은 패턴으로 나머지 조각들을 진행합니다.

패턴을 기억하세요:
1. Claude에게 조각 요청
2. 브라우저에서 확인
3. 문제 있으면 수정 요청
4. OK면 다음 조각으로

이 사이클을 모든 조각이 완성될 때까지 반복합니다.

> 막히면: 막히는 건 정상입니다. 프로 개발자도 매일 막힙니다. 
> ```
> [상황 설명]. 어떻게 하면 좋을까?
> ```
> 이 한 마디면 Claude가 방법을 제안해줍니다.

### 단계 4: 기능 1과 통합 — 서비스답게 연결하기

두 번째 기능이 완성되면, 첫 번째 기능과 연결합니다.

Claude에게 이렇게 요청하세요:
```
[기능 2]를 [기능 1]과 자연스럽게 연결해줘.

예를 들어:
- [기능 1의 상세 페이지]에서 [기능 2]에 접근할 수 있도록
- [기능 2의 데이터]가 [기능 1]과 연결되어 보이도록
- 네비게이션에 [기능 2] 링크도 추가

사용자가 두 기능을 오가면서 자연스럽게 사용할 수 있어야 해.
```

통합 후 확인할 것:
- 기능 1 페이지에서 기능 2로 자연스럽게 이동 가능한가?
- 기능 2의 데이터가 기능 1과 연결되어 보이는가?
- 네비게이션으로 양쪽 기능 모두 접근 가능한가?
- 기존에 잘 되던 기능 1이 여전히 동작하는가? (이것이 중요!)

> 에러가 나면: 통합 과정에서 에러가 가장 많이 납니다. 
> ```
> [기능 1]과 [기능 2]를 연결했더니 [이런 문제]가 생겼어. 해결해줘.
> ```

### 단계 5: 전체 서비스 통합 테스트

처음부터 끝까지, 실제 사용자처럼 테스트합니다.

**전체 체크리스트:**

홈페이지 + 인증:
- [ ] 홈페이지에서 "시작하기" → 회원가입
- [ ] 회원가입 → 자동 로그인 → 대시보드
- [ ] 로그인/로그아웃 정상 동작
- [ ] 네비게이션이 로그인 상태에 맞게 변경

기능 1:
- [ ] 데이터 추가 → 목록에 표시
- [ ] 데이터 상세 보기
- [ ] 데이터 수정 → 변경 반영
- [ ] 데이터 삭제 → 목록에서 제거

기능 2:
- [ ] 데이터 추가 → 목록에 표시
- [ ] 데이터 상세 보기
- [ ] 데이터 수정 → 변경 반영
- [ ] 데이터 삭제 → 목록에서 제거

기능 1 + 기능 2 연결:
- [ ] 기능 1 → 기능 2로 자연스러운 이동
- [ ] 데이터가 서로 연결되어 보임

접근 제어:
- [ ] 비로그인 상태에서 기능 페이지 접근 불가
- [ ] 네비게이션으로 모든 페이지 이동 가능

문제가 있으면:
```
전체 테스트 중에 [문제]를 발견했어. 해결해줘.
```

### 단계 6: 디자인 통일

두 기능의 디자인이 서로 어울리는지 확인합니다.

```
[기능 1]과 [기능 2] 페이지들의 디자인을 통일해줘.
버튼 스타일, 카드 디자인, 색상, 여백 등이 일관되게.
홈페이지와도 디자인이 어울리게 해줘.
```

### 단계 7: Git 세이브 + CLAUDE.md 업데이트

Claude에게 이렇게 요청하세요:
```
변경사항을 Git에 저장해줘. 메시지는 "핵심 기능 2: [기능명] 구현 + 기능 1과 통합"으로.
CLAUDE.md의 현재 상태도 "v3-feature-2 완료. 핵심 기능 2개 구현 및 통합 완료."로 업데이트해줘.
```

### 단계 8: 깨달음의 순간

여기서 잠깐 멈추고 돌아봅시다.

**방금 일어난 일:**

여러분은 두 번째 기능을 **거의 혼자** 만들었습니다.

- 스스로 기능을 분석했고
- 스스로 조각으로 나눴고
- 스스로 Claude에게 지시했고
- 스스로 결과를 확인했고
- 스스로 문제를 해결했습니다

v3-feature-1에서는 제가 "이렇게 하세요"라고 알려드렸지만, 이번에는 여러분이 직접 했습니다. **같은 패턴인데 주도권이 여러분에게 넘어온 것**입니다.

이것이 의미하는 바:

> **세 번째, 네 번째, 다섯 번째 기능도 같은 방식으로 만들 수 있습니다.**
> **완전히 새로운 서비스도 같은 방식으로 만들 수 있습니다.**

"계획 → 조각내기 → 지시 → 확인 → 반복 → 세이브"

이 패턴 하나로 어떤 서비스 기능이든 만들 수 있습니다. 이것이 AI 시대의 서비스 빌딩 스킬입니다.

여러분은 코딩을 배운 게 아닙니다. **AI와 함께 무엇이든 만드는 방법**을 배운 것입니다.
