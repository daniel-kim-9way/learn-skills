## EXPLAIN

### Git = 게임의 세이브 포인트

게임을 할 때 중요한 장면마다 세이브하죠?

보스전 앞에서 세이브하면, 보스에게 지더라도 세이브한 지점부터 다시 시작할 수 있습니다. 새 무기를 얻으면 세이브해두고, 위험한 던전에 들어가기 전에 또 세이브합니다.

**Git**은 코드의 세이브 시스템입니다.

| 게임 | Git |
|------|-----|
| 세이브 포인트 만들기 | `git commit` — 현재 상태를 저장 |
| 세이브 목록 보기 | `git log` — 언제, 무엇을 저장했는지 확인 |
| 세이브 메시지 쓰기 | commit message — "보스전 전" 같은 메모 |
| 새 게임 시작 시 세이브 활성화 | `git init` — 이 폴더를 세이브 관리하겠다고 선언 |

### 왜 세이브가 필요한가요?

코딩을 하다 보면 이런 일이 반드시 생깁니다:

**시나리오 1**: 잘 되던 로그인 기능이 갑자기 안 된다
- 세이브 없이: "어디서 망가진 거지?" 한참 찾아야 함
- 세이브 있으면: "3시간 전 세이브로 돌아가자" 끝

**시나리오 2**: 새 기능을 추가했는데 기존 기능이 망가졌다
- 세이브 없이: 새 기능도 기존 기능도 다 엉망
- 세이브 있으면: "새 기능 추가 전으로 돌아가자" 깨끗

**시나리오 3**: "아까 그 디자인이 더 나았는데..."
- 세이브 없이: 기억에 의존해서 다시 만들어야 함
- 세이브 있으면: "디자인 변경 전 세이브"로 돌아가면 됨

세이브 없는 게임을 상상해보세요. 실수 한 번에 처음부터 다시 시작해야 합니다. 끔찍하죠? Git은 그 끔찍함을 막아줍니다.

### 딱 세 가지만 기억하세요

Git에는 수십 가지 기능이 있습니다. 하지만 지금은 **세 가지만** 알면 됩니다:

| 명령어 | 게임 비유 | 역할 |
|--------|----------|------|
| `git init` | 새 게임에서 세이브 기능 활성화 | 이 폴더를 Git으로 관리하겠다고 선언 |
| `git add .` | 세이브할 파일들 선택 | 변경된 파일들을 세이브 대기열에 올림 |
| `git commit -m "메시지"` | 세이브 버튼 누르기 | 현재 상태를 메시지와 함께 저장 |

**이것만 기억하세요. 나머지는 전부 무시.**

branch? 나중에. push? 나중에. merge? 나중에. GitHub? 나중에.

지금은 **"내 컴퓨터에 세이브하기"**만 합니다. 게임기 안에 저장하는 것처럼요.

### 좋은 세이브 메시지 쓰기

게임 세이브에 "세이브 1", "세이브 2"라고 쓰면 나중에 어떤 세이브가 어떤 상태인지 모릅니다.

하지만 이렇게 쓰면?
- "보스전 직전 - 체력 만땅"
- "전설 무기 획득 후"
- "마을 퀘스트 3개 완료"

어떤 세이브가 어떤 시점인지 한눈에 알 수 있죠.

Git도 마찬가지입니다:

**나쁜 예:**
- "수정함" → 뭘 수정한 건데?
- "업데이트" → 무슨 업데이트?
- "asdf" → ...
- "123" → ...

**좋은 예:**
- "홈페이지 제목과 소개 문구 추가"
- "회원가입 기능 구현"
- "배경색 파란색으로 변경"
- "책 목록 페이지 완성"

규칙은 간단합니다: **"무엇을 했는지"를 한 줄로 쓰면 됩니다.**

나중에 세이브 목록을 볼 때 "아, 이때 홈페이지를 만들었구나" 하고 바로 알 수 있도록요.

## EXECUTE

### 단계 1: Git 초기화 — "새 게임에서 세이브 기능 켜기"

여러분의 서비스 프로젝트 폴더에서 시작합니다.

Claude에게 이렇게 요청하세요:
```
이 프로젝트를 Git으로 관리할 수 있게 초기화해줘.
```

Claude가 `git init` 명령을 실행합니다.

> "Initialized empty Git repository"라는 메시지가 나오면 성공입니다!

이 순간, 여러분의 프로젝트 폴더에 세이브 기능이 활성화되었습니다. 아직 세이브를 한 건 아닙니다 — 세이브를 **할 수 있는 상태**가 된 것입니다.

> 이미 git init이 되어 있다면? Rails 프로젝트를 만들 때 자동으로 초기화되는 경우가 있습니다. Claude가 "이미 Git으로 관리되고 있어요"라고 하면 그대로 단계 2로 넘어가세요.

### 단계 2: 첫 번째 세이브 — "프로젝트의 탄생 기록"

Claude에게 이렇게 요청하세요:
```
지금까지 만든 모든 파일을 Git에 저장해줘.
세이브 메시지는 "프로젝트 시작: 홈페이지 완성"으로 해줘.
```

Claude가 두 가지를 실행합니다:
1. `git add .` — 모든 파일을 세이브 대기열에 올림
2. `git commit -m "프로젝트 시작: 홈페이지 완성"` — 세이브!

화면에 여러 줄이 나타나며 어떤 파일들이 저장되었는지 보여줍니다. 숫자가 많이 나와도 놀라지 마세요 — Rails 프로젝트는 기본 파일이 많거든요.

**첫 번째 세이브 포인트가 만들어졌습니다!** 게임으로 치면 "1화 시작 지점"이 저장된 것입니다.

### 단계 3: 세이브 확인 — "정말 저장됐나?"

Claude에게 이렇게 요청하세요:
```
Git 세이브 기록을 보여줘.
```

Claude가 `git log`를 실행합니다. 이런 내용이 보입니다:

```
commit abc1234... 
Date: 2024-XX-XX
    프로젝트 시작: 홈페이지 완성
```

날짜, 시간, 세이브 메시지가 함께 표시됩니다. 방금 만든 세이브가 기록에 남아있습니다!

게임의 세이브 목록과 같죠? "언제, 무엇을 했는지" 기록이 남습니다.

### 단계 4: 파일 수정하기 — "세이브 후 변경 만들기"

이제 일부러 파일을 수정합니다. 세이브가 2개가 되는 것을 확인하기 위해서요.

Claude에게 이렇게 요청하세요:
```
홈페이지에 "현재 학습 중: Git 세이브 포인트"라는 문구를 추가해줘.
푸터 위쪽에 작은 글씨로 넣어줘.
```

브라우저에서 확인합니다. 문구가 추가되었나요?

이 상태는 "변경은 했지만 아직 세이브하지 않은" 상태입니다. 게임에서 진행은 했는데 아직 세이브 버튼을 안 누른 것과 같습니다.

### 단계 5: 두 번째 세이브 — "변경 사항 기록"

Claude에게 이렇게 요청하세요:
```
변경사항을 Git에 저장해줘.
세이브 메시지는 "홈페이지에 학습 상태 문구 추가"로 해줘.
```

두 번째 세이브가 만들어졌습니다!

### 단계 6: 세이브 2개 확인 — "기록이 쌓이는 마법"

Claude에게 이렇게 요청하세요:
```
Git 세이브 기록을 다시 보여줘.
```

이번에는 세이브가 **2개** 보입니다:

```
commit def5678...
Date: 2024-XX-XX (더 최근)
    홈페이지에 학습 상태 문구 추가

commit abc1234...
Date: 2024-XX-XX (더 이전)
    프로젝트 시작: 홈페이지 완성
```

최신 세이브가 위에, 이전 세이브가 아래에 나타납니다.

**이것이 Git의 힘입니다.** 언제, 무엇을 했는지 시간순으로 기록이 남습니다. 나중에 세이브가 10개, 20개로 늘어나도 "아, 이때 이걸 했구나" 한눈에 볼 수 있습니다.

### 단계 7: "매 레슨 끝에 세이브" 습관 만들기

앞으로 매 레슨이 끝날 때마다 이렇게 세이브합니다:

```
변경사항을 Git에 저장해줘.
세이브 메시지는 "[레슨명] 완료"로 해줘.
```

이 한 줄이면 됩니다.

**세이브 타이밍 체크리스트:**
- 레슨을 완료했을 때
- 새 기능이 잘 동작하는 것을 확인했을 때
- 큰 변경을 하기 전에 (혹시 망가질까봐)
- "지금 상태 좋다!" 싶을 때

게임에서도 "지금 세이브해야지" 싶은 순간이 있잖아요. 그때 세이브하면 됩니다.

> 꿀팁: 세이브를 너무 자주 해도 괜찮습니다. 세이브 안 하고 후회하는 것보다, 많이 해서 기록이 남는 게 훨씬 낫습니다. "이것도 세이브해야 하나?" 고민이 되면 — 세이브하세요.

### 단계 8: CLAUDE.md 업데이트 + 세이브

Claude에게 이렇게 요청하세요:
```
CLAUDE.md의 현재 상태를 업데이트해줘.
"v3-git-save 완료. Git 초기화, 2개 커밋 완료."로 바꿔줘.
그리고 변경사항을 Git에 저장해줘. 메시지는 "v3-git-save 완료: CLAUDE.md 업데이트"로.
```

> 이 패턴을 기억하세요: **레슨 끝 = CLAUDE.md 업데이트 + Git 세이브**. 앞으로 매 레슨마다 이걸 합니다.

해설:
