# V3: 새 티켓 만들기 + AI 에이전트 배정

## 학습 목표
- 빌드킷에 미리 깔려 있던 티켓 외에, 본인이 직접 떠올린 신규 기능을 작은 단위로 분해해서 티켓으로 만든다
- 4가지 에이전트(Rails 개발자 / UX·UI 디자이너 / 리뷰어 / 보안 전문가)의 역할을 구분하고, 본인 티켓에 적절한 에이전트를 1개 이상 배정한다
- 에이전트가 만든 결과를 검토하고, 부족하면 피드백해서 다시 시킨다

## 사전 조건
- Ch.4-6 완료 (빌드킷 기존 티켓 1개 이상 `/backlog [ID]`로 실행해 본 경험)
- 본인 SaaS 프로젝트가 vuild.kr에 등록되어 있고 `vuild.kr/projects/{본인ID}/build/board`에 접근 가능
- Antigravity + Claude Code가 본인 PC에 깔려 있고 본인 빌드킷 폴더에서 작업 중

## 대화형 학습 프로토콜

이 스킬은 대화형 코칭 방식을 따릅니다. 강의가 아닌 질문-답변-피드백 루프로 진행합니다.

### 진행 방식
1. references/main.md를 읽고 대화 흐름(Turn 1 ~ Turn 7)에 따라 진행
2. 학생의 신규 기능 아이디어를 받아 → 작업 단위로 분해 → 4종 에이전트 중 적합한 곳 배정 → 보드에 실제 티켓 생성 → 결과 검토 + 피드백 루프 안내
3. "잘 모르겠어요" 같은 답도 정상 — 본인 빌드킷의 기존 티켓을 참고해서 다시 풀어준다
4. 마지막 Turn에서 본인이 만든 티켓 명세 + 배정한 에이전트 + 검토 노트를 파일로 저장
