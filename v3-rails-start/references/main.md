## EXPLAIN

### Rails = 건물의 골조

집을 지을 때 처음부터 벽돌 하나하나 쌓지 않습니다. 먼저 **골조**(철근 뼈대)를 세우고, 그 위에 벽, 지붕, 인테리어를 입힙니다.

**Rails(Ruby on Rails)**는 웹 서비스의 골조입니다.

서비스에 필요한 기본 구조를 미리 준비해줍니다:
- 사용자의 요청을 받는 시스템 (누가 어떤 페이지를 열었는지)
- 데이터를 저장하는 시스템 (회원 정보, 게시글 등)
- 화면을 보여주는 시스템 (사용자가 보는 웹페이지)
- 보안, 로그인, 오류 처리 등

이 모든 것을 처음부터 만들면 몇 달이 걸립니다. Rails를 쓰면? **명령어 하나**로 기본 구조가 만들어집니다.

비유를 하나 더 하자면:
- 요리를 할 때, 밀가루부터 반죽해서 면을 만들 수도 있지만
- **라면 봉지를 뜯어서 끓이면** 5분이면 됩니다
- Rails는 서비스의 "라면 봉지"입니다 — 기본 재료가 다 들어있어요

### Ruby는 뭐고 Rails는 뭔가요?

- **Ruby** = 프로그래밍 언어 (한국어, 영어처럼 컴퓨터와 대화하는 언어)
- **Rails** = Ruby로 만든 웹 서비스 도구 (Ruby 언어를 사용하는 서비스 제작 키트)

Ruby를 직접 배울 필요는 없습니다. Claude가 Ruby를 씁니다. 여러분은 한국어로 Claude에게 지시하면 됩니다.

### localhost = 내 컴퓨터에서만 보이는 비밀 주소

서버를 시작하면 `http://localhost:3000`이라는 주소에서 서비스를 볼 수 있습니다.

**localhost**는 "내 컴퓨터"를 가리키는 특별한 주소입니다:
- 일반 웹사이트 (naver.com) = 모든 사람이 방문할 수 있는 공개 건물
- **localhost** = **내 방에서만 볼 수 있는 비밀 모형** (다른 사람은 절대 볼 수 없음)

옆에 앉은 사람에게 "localhost:3000 들어가봐"라고 해도 안 됩니다. 내 컴퓨터에서만 작동하는 비밀 주소니까요.

**3000**은 문 번호입니다. 건물(컴퓨터)에 여러 문(포트)이 있는데, Rails는 기본적으로 3000번 문을 씁니다.

나중에 C4에서 이 비밀 모형을 인터넷에 공개합니다. "localhost → 진짜 URL"로 바뀌는 순간이 옵니다. 지금은 내 컴퓨터에서만 개발하고 테스트합니다.

### 이 순간이 왜 특별한가요?

지금 여러분은 **내 서비스를 브라우저에서 처음으로 보게 됩니다.**

C1에서 아이디어를 떠올리고, C2에서 기획서를 만들고, 그동안 서비스는 "문서" 속에만 있었습니다. 오늘, 처음으로 **화면에 실제로** 나타납니다.

글자 한 줄이라도 — 이것은 여러분의 서비스입니다. 브라우저 주소창에 localhost:3000을 치고, 내 서비스 이름이 화면에 뜨는 순간 — 그 기분을 기억하세요.

이것이 서비스의 **첫 심장 박동**입니다.

## EXECUTE

### 단계 1: Ruby와 Rails 설치 확인

Ruby와 Rails가 컴퓨터에 설치되어 있어야 합니다.

Claude에게 이렇게 요청하세요:
```
내 컴퓨터에 Ruby와 Rails가 설치되어 있는지 확인해줘.
버전도 알려줘.
```

**설치되어 있는 경우:** 버전 번호가 나타납니다. 단계 2로 넘어가세요.

**설치되어 있지 않은 경우:** Claude에게 설치를 요청합니다:
```
Ruby와 Rails를 설치해줘. 내 OS는 [Windows/macOS]야.
```

> Windows 사용자 참고: WSL(Windows Subsystem for Linux)이 필요할 수 있습니다. Claude가 안내해줄 겁니다. "WSL이 뭐야?" 걱정하지 마세요 — Windows에서 개발 도구를 편하게 쓰게 해주는 장치입니다. Claude의 안내를 따라가면 됩니다.

> macOS 사용자 참고: Homebrew라는 도구를 먼저 설치할 수 있습니다. "Homebrew가 뭐야?" — Mac에서 프로그램을 쉽게 설치하는 도구입니다. 앱스토어 비슷한 거예요.

설치에 시간이 걸릴 수 있습니다 (5~15분). 중간에 오류가 나면 당황하지 마세요:
```
이런 오류가 나왔어: [오류 메시지 복사-붙여넣기]. 어떻게 해?
```

> 에러가 나면: 에러 메시지는 "적"이 아니라 "단서"입니다. 에러를 Claude에게 그대로 보여주면 Claude가 해결해줍니다. 개발자들도 매일 에러를 만나고, 매일 이렇게 해결합니다.

### 단계 2: 프로젝트 폴더 준비

서비스 프로젝트를 만들 위치를 확인합니다.

Claude에게 이렇게 요청하세요:
```
my-projects 폴더로 이동해줘.
```

v3-backward에서 만든 sample-bookclub 폴더가 보일 수 있습니다. 그건 샘플이고, 이제 **진짜 내 서비스**를 만듭니다.

### 단계 3: Rails 프로젝트 생성 — "서비스의 첫 심장 박동"

이 순간입니다.

Claude에게 이렇게 요청하세요:
```
[서비스이름]이라는 Rails 프로젝트를 새로 만들어줘.
CSS는 Tailwind CSS를 사용해줘.
데이터베이스는 SQLite로 해줘.
```

서비스이름은 C1에서 정한 이름을 영어로 쓰면 됩니다.
- 한 단어: `bookclub`, `mealplanner`, `studybuddy`
- 여러 단어는 하이픈(-) 연결: `pet-diary`, `my-recipe`, `study-helper`

> 꿀팁: 서비스 이름에 한글, 띄어쓰기, 특수문자는 넣지 마세요. 영어 소문자와 하이픈(-)만 사용합니다.

Claude가 `rails new [서비스이름]` 명령을 실행합니다.

화면에 수많은 줄이 쭉 올라갑니다. 파일이 생성되고, 라이브러리가 설치되고... 1~2분 정도 기다리세요.

완료되면 에디터 왼쪽에 수많은 폴더와 파일이 나타납니다. 무섭게 느껴질 수 있지만, v3-backward에서 본 구조와 같습니다. app/, config/, db/ — 건물의 층별 안내도, 기억나시죠?

### 단계 4: 서버 시작 — 살아있나 확인!

Claude에게 이렇게 요청하세요:
```
서버를 시작해줘.
```

Claude가 `rails server` 명령을 실행합니다.

이제 브라우저를 열고 주소창에 입력하세요:
```
http://localhost:3000
```

**Rails 기본 환영 페이지**가 나타납니다! 초록색 로고와 함께 "You're on Rails!" 같은 메시지가 보입니다.

이 페이지가 보이는 순간 — 축하합니다!

**여러분의 서비스가 살아 숨쉬기 시작했습니다.** 아직 아무 기능도 없지만, 서버가 돌아가고, 브라우저에서 접속할 수 있습니다. 건물로 치면 골조가 완성된 것입니다. 이제 여기에 벽도 세우고, 인테리어도 하고, 가구도 놓을 겁니다.

> 에러가 나면: "localhost:3000이 안 열려요" — 서버가 실행 중인지 확인하세요. Claude에게 "서버 상태 확인해줘"라고 하면 됩니다. 가장 흔한 원인은 서버가 아직 시작 안 된 것입니다.

### 단계 5: 기본 페이지 → 내 서비스 홈페이지로 교체

Rails 기본 환영 페이지는 "공사 중" 안내판 같은 것입니다. 이제 이것을 **내 서비스의 첫 화면**으로 바꿉니다.

Claude에게 이렇게 요청하세요:
```
Rails 기본 페이지를 내 서비스 홈페이지로 바꿔줘.

서비스 이름: [서비스이름]
한 줄 소개: [C1에서 만든 가치 선언문]

이런 구성으로:
- 큰 제목으로 서비스 이름
- 아래에 한 줄 소개
- "시작하기" 버튼 (아직 동작 안 해도 돼)
- 깔끔하고 현대적인 디자인 (Tailwind CSS)

배경은 깨끗한 흰색/밝은 색으로 해줘.
```

브라우저를 새로고침합니다 (Ctrl+R 또는 Cmd+R).

**내 서비스 이름과 소개가 나타납니다!**

이것은 더 이상 Rails 기본 페이지가 아닙니다. **여러분의 서비스**입니다.

> 마음에 안 드는 부분이 있으면 바로 수정하세요:
> ```
> 제목 폰트를 더 크게 해줘.
> ```
> ```
> 배경색을 연한 파란색으로 바꿔줘.
> ```
> ```
> "시작하기" 버튼을 더 눈에 띄게 해줘.
> ```

이것이 에디터 + Claude의 힘입니다. 말하면 바로 바뀝니다.

### 단계 6: 작은 수정 체험 — "요청 → 확인" 사이클 연습

홈페이지를 몇 가지 더 수정해봅시다. 이 "요청 → 확인" 사이클에 익숙해지는 것이 목적입니다.

**수정 1: 문구 추가**
```
홈페이지 서비스 소개 아래에 "지금 무료로 시작하세요"라는 문구를 추가해줘.
```
브라우저 새로고침 → 확인!

**수정 2: 색상 변경**
```
"시작하기" 버튼 색상을 [원하는 색]으로 바꿔줘.
```
브라우저 새로고침 → 확인!

**수정 3: 내용 추가**
```
홈페이지 아래쪽에 서비스의 특장점 3가지를 카드 형태로 추가해줘.
특장점은:
1. [특장점 1]
2. [특장점 2]  
3. [특장점 3]
```
브라우저 새로고침 → 확인!

이 사이클이 C3 전체에서 반복됩니다:
1. Claude에게 요청한다
2. Claude가 코드를 수정한다
3. 브라우저에서 확인한다
4. 마음에 안 들면 다시 요청한다

코드를 직접 쓸 일은 없습니다. 여러분은 "감독"이고 Claude가 "실행자"입니다.

### 단계 7: 프로젝트 구조 확인

Claude에게 이렇게 요청하세요:
```
지금 프로젝트의 구조를 간단히 설명해줘.
방금 홈페이지를 만들면서 어떤 파일이 바뀌었는지도 알려줘.
```

v3-backward에서 봤던 구조가 다시 나타납니다:
- `app/views/` — 방금 만든 홈페이지가 여기에 있습니다
- `app/controllers/` — 홈페이지를 보여주는 처리 로직
- `config/routes.rb` — "localhost:3000을 열면 홈페이지를 보여줘"라는 설정

"아, 독서 모임 앱에서 봤던 그 구조구나!" 연결이 되시나요?
