## EXPLAIN

### 데스크탑에서 에디터로: 무엇이 달라지나요?

C1과 C2에서는 Claude Desktop을 사용했습니다. 채팅창에 글을 쓰면 Claude가 답해주는 방식이었죠.

이제 한 단계 업그레이드합니다. **코드 에디터 + Claude Code 확장 프로그램**으로 넘어갑니다.

"에디터"라는 단어가 무섭게 느껴질 수 있습니다. 하지만 걱정하지 마세요. 데스크탑에서 하던 것과 똑같이 Claude와 대화합니다. 달라지는 건 딱 하나 — **Claude가 이제 직접 파일을 만들어준다**는 것입니다.

비유하면 이렇습니다:
- **Claude Desktop** = 전화로 비서에게 지시하기 (비서가 말로 답해줌, 직접 실행은 내가)
- **에디터 + Claude Code** = 비서가 내 사무실에 와서 직접 일하기 (지시하면 바로 실행)

### 처음 에디터를 열었을 때의 두려움

에디터를 처음 열면 이런 생각이 듭니다:

> "버튼이 100개는 되는 것 같은데... 나는 뭘 눌러야 하지?"

정상입니다. 개발자들도 처음엔 다 그랬습니다. 하지만 진실을 알려드릴게요:

**우리가 쓸 버튼은 딱 3개입니다:**
1. 왼쪽 파일 탐색기 (폴더 아이콘) — 파일 목록 보기
2. Claude Code 아이콘 — AI와 대화하기
3. 터미널 영역 — Claude가 명령어 실행하는 곳 (그냥 구경만 하면 됩니다)

나머지 버튼들은? 무시하세요. 진심입니다. 프로 개발자들도 대부분의 버튼은 안 씁니다.

### VS Code란?

VS Code(Visual Studio Code)는 마이크로소프트가 만든 **무료** 코드 에디터입니다.

비유하면 이렇습니다:
- 메모장 = 일반 텍스트만 쓸 수 있는 공책
- 워드(Word) = 글을 예쁘게 꾸밀 수 있는 도구
- **VS Code** = 코드를 편하게 쓸 수 있는 전용 도구

전 세계에서 가장 많이 쓰이고, 초보자부터 전문가까지 모두 사용합니다. 무료인데 성능이 좋아서 유료 도구를 밀어낸 전설적인 프로그램입니다.

### Claude Code 확장이란?

확장(Extension)은 에디터에 추가하는 플러그인입니다.

스마트폰에 카카오톡 앱을 설치하듯, VS Code에 Claude Code 확장을 설치하면 에디터 안에서 AI와 대화하면서 서비스를 만들 수 있습니다.

설치하면 왼쪽 사이드바에 Claude 아이콘이 나타납니다. 그걸 클릭하면 채팅창이 열리고, 거기서 Claude와 대화합니다. Desktop에서 하던 것과 똑같은 경험입니다.

### 데스크탑 vs 에디터: 핵심 비교

| 항목 | Claude Desktop | 에디터 + Claude Code |
|------|----------------|---------------------|
| 대화 | 채팅창에서 대화 | 에디터 안에서 대화 |
| 파일 생성 | Claude가 텍스트를 보여줌 → 내가 직접 저장 | **Claude가 직접 파일 생성/수정** |
| 코드 실행 | 불가능 — 내가 직접 해야 함 | **Claude가 직접 실행** |
| 오류 수정 | 수동으로 복사-붙여넣기 | **Claude가 자동으로 감지하고 수정** |
| 프로젝트 파악 | 매번 설명해줘야 함 | **폴더 안의 모든 파일을 자동으로 파악** |

핵심 차이: **Claude가 "말만 하는 조수"에서 "직접 일하는 조수"로 진화합니다.**

C1에서 Claude Desktop으로 기획서를 만들 때를 떠올려보세요. Claude가 텍스트를 보여주면 여러분이 복사해서 파일로 저장했죠? 에디터에서는 Claude가 직접 파일을 만들고, 수정하고, 삭제까지 합니다.

## EXECUTE

### 단계 1: 내 컴퓨터 확인

먼저 자신의 컴퓨터가 무엇인지 확인합니다:
- **Windows**: 시작 메뉴가 있는 컴퓨터 (대부분의 노트북/데스크탑)
- **macOS**: 사과(Apple) 마크가 있는 컴퓨터 (맥북, 아이맥)

> 왜 확인하나요? Windows와 Mac은 설치 방법이 조금 다릅니다. 마치 아이폰과 안드로이드에서 앱 설치하는 곳이 다른 것처럼요.

### 단계 2: VS Code 설치

**Windows 사용자:**

1. 브라우저에서 `https://code.visualstudio.com` 접속
2. 큰 파란 버튼 "Download for Windows" 클릭
3. 다운로드된 파일 실행
4. "다음(Next)" 계속 클릭 (기본 설정 그대로 OK)
5. 설치 완료 후 VS Code 실행

> 꿀팁 (Windows): 설치 중 "PATH에 추가" 옵션이 나오면 반드시 체크하세요. 나중에 편해집니다.

**macOS 사용자:**

1. 브라우저에서 `https://code.visualstudio.com` 접속
2. "Download for Mac" 클릭
3. 다운로드된 .zip 파일 더블클릭으로 압축 해제
4. "Visual Studio Code" 앱을 "응용 프로그램" 폴더로 드래그
5. 응용 프로그램에서 VS Code 실행

> 꿀팁 (Mac): 처음 실행 시 "인터넷에서 다운로드한 앱입니다" 경고가 나올 수 있습니다. "열기"를 클릭하면 됩니다. 정상입니다.

VS Code를 실행하면 "Welcome" 탭이 나타납니다. 이 화면이 보이면 성공입니다!

> 막혔을 때: 설치가 안 되거나 오류가 나면, Claude Desktop을 열고 "VS Code 설치 중에 [오류 내용]이 나왔어. 어떻게 해?"라고 물어보세요.

### 단계 3: VS Code 한국어 설정 (선택)

영어 메뉴가 불편하다면 한국어로 바꿀 수 있습니다:

1. 왼쪽 사이드바에서 네모 4개 모양 아이콘 (확장 마켓플레이스) 클릭
2. 검색창에 `Korean Language Pack` 입력
3. "Korean Language Pack for Visual Studio Code" 설치
4. VS Code 재시작

이제 메뉴가 한국어로 나타납니다.

### 단계 4: Claude Code 확장 설치

VS Code가 열린 상태에서:

1. 왼쪽 사이드바에서 네모 4개 모양 아이콘(확장)을 클릭합니다
2. 검색창에 `Claude Code`를 입력합니다
3. Anthropic이 만든 "Claude Code" 확장을 찾아 "Install" 버튼을 클릭합니다
4. 설치가 끝나면 왼쪽 사이드바에 **Claude 아이콘**이 나타납니다

> 주의: "Claude" 비슷한 이름의 확장이 여러 개 있을 수 있습니다. 반드시 **Anthropic**이 만든 공식 확장인지 확인하세요.

설치 후 Claude 아이콘을 클릭하면 로그인 화면이 나타날 수 있습니다. Anthropic 계정으로 로그인하면 됩니다.

> 막혔을 때: 로그인이 안 되면 Claude Desktop에서 "Claude Code VS Code 확장에서 로그인이 안 돼. [오류 내용]"이라고 물어보세요.

### 단계 5: 프로젝트 폴더 만들기

서비스를 만들 작업 공간을 준비합니다.

**Windows 사용자:**
1. 파일 탐색기에서 `문서` 폴더로 이동
2. 새 폴더 만들기: `my-projects`
3. VS Code에서 `파일 > 폴더 열기` → 방금 만든 `my-projects` 선택

**macOS 사용자:**
1. Finder에서 `문서` 폴더로 이동
2. 새 폴더 만들기: `my-projects`
3. VS Code에서 `File > Open Folder` → 방금 만든 `my-projects` 선택

> "이 폴더의 작성자를 신뢰하시나요?" 알림이 나오면 "예, 신뢰합니다" 클릭.

### 단계 6: 에디터에서 첫 대화

이제 드디어 Claude와 만납니다!

1. 왼쪽 사이드바에서 Claude 아이콘을 클릭합니다
2. 채팅창이 열립니다
3. 이렇게 입력합니다:

```
안녕! 나는 [내 서비스 이름]을 만들 거야. 반가워!
```

Claude가 인사하면 성공입니다! Desktop에서 대화하던 것과 같은 느낌이죠?

### 단계 7: 파일 생성 데모 — Desktop vs Editor의 결정적 차이

이제 핵심적인 차이를 직접 체험합니다.

Claude에게 이렇게 요청하세요:
```
hello.txt라는 파일을 만들어줘.
안에 "나의 첫 번째 파일입니다!"라고 적어줘.
```

**지금 일어나는 일을 주목하세요:**

왼쪽 파일 탐색기에 `hello.txt`가 **자동으로** 나타납니다. 클릭하면 파일 내용이 보입니다.

Desktop에서라면 어땠을까요? Claude가 "이런 내용을 파일로 저장하세요"라고 텍스트를 보여줬을 겁니다. 그러면 여러분이 직접 메모장을 열고, 복사하고, 붙여넣고, 저장 경로를 찾아서 저장해야 했죠.

에디터에서는? **Claude에게 말하면 끝.** 이것이 핵심 차이입니다.

### 단계 8: 파일 수정 데모

Claude에게 이렇게 요청하세요:
```
hello.txt에 두 번째 줄을 추가해줘.
"Claude Code가 직접 만들었습니다!"라고 적어줘.
```

파일이 자동으로 수정되는 것을 확인합니다. 에디터에서 파일을 클릭하면 두 줄이 보입니다.

이 패턴 — **"요청 → Claude가 파일 수정 → 확인"** — 이 앞으로 C3 전체에서 계속 반복됩니다. 서비스의 모든 코드가 이 방식으로 만들어집니다.

### 단계 9: 명령어 실행 데모

Desktop에서는 할 수 없었던 것을 해봅시다.

Claude에게 이렇게 요청하세요:
```
현재 폴더에 있는 모든 파일 목록을 보여줘.
```

Claude가 터미널에서 명령어를 실행하고 결과를 보여줍니다. Desktop에서는 "이 명령어를 터미널에 입력하세요"라고 안내만 했지만, 에디터에서는 **Claude가 직접 실행**합니다.

### 단계 10: 정리 + 핵심 확인

테스트 파일을 삭제합니다:
```
hello.txt 파일을 삭제해줘.
```

파일이 사라지는 것을 확인합니다.

지금까지 체험한 것을 정리하면:
- **파일 생성**: Claude에게 말하면 직접 만들어줌
- **파일 수정**: Claude에게 말하면 직접 고쳐줌
- **파일 삭제**: Claude에게 말하면 직접 지워줌
- **명령어 실행**: Claude에게 말하면 직접 실행해줌

앞으로 서비스를 만들 때, 여러분은 이 네 가지를 반복합니다. 코드를 직접 타이핑할 일은 없습니다. Claude에게 **무엇을 원하는지 말하면** 됩니다.

> 꿀팁: 에디터의 수많은 버튼과 메뉴가 신경 쓰이시나요? 지금은 무시하세요. 정말로요. 우리는 Claude와 대화하고, 파일 탐색기를 보고, 브라우저에서 결과를 확인하는 것만 합니다. 그 세 가지가 전부입니다.
