## EXPLAIN

### [선택] 이 레슨은 선택사항입니다

이 레슨은 **선택 레슨**입니다. AI 기능을 추가하고 싶은 분만 진행하면 됩니다.

AI 기능이 없어도 서비스는 충분히 가치가 있습니다. 하지만 AI를 더하면 사용자 경험이 한 단계 올라갑니다. "있으면 좋고, 없어도 괜찮은" 보너스라고 생각하세요.

건너뛰어도 다음 레슨(결제, 보안, 배포)에 영향이 없습니다.

### 내 서비스에 AI를 더하면?

AI가 붙으면 서비스가 **대화할 수 있는 서비스**가 됩니다. 몇 가지 예시를 볼까요?

| 서비스 유형 | AI 없이 | AI 추가 후 |
|------------|---------|-----------|
| 독서 기록 앱 | 책 제목, 감상 기록 | "이 책의 핵심 메시지가 뭐야?" 물어볼 수 있음 |
| 할 일 관리 앱 | 할 일 추가/완료 | "오늘 할 일 중 우선순위를 AI가 정해줘" |
| 식단 관리 앱 | 먹은 음식 기록 | "이 재료로 만들 수 있는 레시피 추천해줘" |
| 운동 기록 앱 | 운동 내역 저장 | "이번 주 운동 패턴 분석해줘" |
| 일기 앱 | 일기 작성 | "이번 달 감정 변화 요약해줘" |

AI는 단순한 기능이 아니라 **서비스의 두뇌**가 됩니다. 그리고 중요한 점 — **내가 AI를 직접 만드는 것이 아닙니다.** 이미 만들어진 AI(Claude)를 "불러서 쓰는" 것입니다.

### API란? 레스토랑 배달 서비스

API(Application Programming Interface). 이름이 어렵지만, 매일 하는 행동과 같습니다.

**레스토랑을 상상하세요:**

```
손님(나의 서비스)
  "파스타 하나 주세요" ← 주문서(요청)
      ↓
웨이터(API)
  주문을 받아 주방에 전달
      ↓
주방(Claude AI)
  요리 시작! (AI가 텍스트를 처리)
      ↓
웨이터(API)
  완성된 요리를 손님에게 전달
      ↓
손님(나의 서비스)
  "맛있다!" ← 결과를 받아서 화면에 표시
```

**API는 주문서입니다.** 내 서비스가 Claude에게 "이 질문에 답해줘"라고 주문서를 보내면, Claude가 답을 만들어서 돌려주는 구조입니다.

배달 앱을 쓸 때 주방에 직접 가지 않듯이, AI를 쓸 때 AI를 직접 만들지 않아도 됩니다. 주문만 잘 하면 됩니다.

### 구체적인 AI 기능 아이디어 5가지

어떤 AI 기능을 넣을지 감이 안 잡힐 수 있습니다. 가장 흔하고 효과적인 5가지 유형을 소개합니다:

**1. 텍스트 요약**
- 긴 글을 핵심만 뽑아서 정리
- 예: 독서 노트 요약, 회의록 요약, 뉴스 요약
- 난이도: 쉬움

**2. 맞춤 추천**
- 사용자의 데이터를 기반으로 개인화된 추천
- 예: 레시피 추천, 운동 루틴 추천, 학습 자료 추천
- 난이도: 보통

**3. 자동 분류**
- 입력된 데이터를 카테고리로 자동 분류
- 예: 문의 유형 분류, 감정 분석, 우선순위 판별
- 난이도: 쉬움

**4. 콘텐츠 생성**
- 주어진 조건에 맞는 텍스트를 생성
- 예: 이메일 초안, 상품 설명, 피드백 답변
- 난이도: 쉬움

**5. 데이터 분석**
- 축적된 데이터에서 인사이트를 뽑아냄
- 예: 습관 분석, 트렌드 파악, 패턴 발견
- 난이도: 보통

> **꿀팁**: 처음이라면 1번(텍스트 요약)이나 4번(콘텐츠 생성)부터 시작하세요. 가장 단순하면서도 사용자가 "오!" 하는 기능입니다.

### API 키 = 비밀번호, .env = 금고

Claude API를 쓰려면 **API 키**가 필요합니다. API 키는 "나는 이 서비스를 사용할 수 있는 사람이에요"라는 증명서입니다.

**API 키는 비밀번호와 같습니다.**

절대 해서는 안 되는 것:
- 소스 코드에 직접 넣기 → GitHub에 올리면 전 세계에 공개
- 카카오톡으로 보내기 → 유출 위험
- 스크린샷에 포함시키기 → 실수로 공유

악의적인 봇들이 GitHub를 **24시간** 스캔합니다. API 키가 공개되면 몇 분 안에 발견되어 도용됩니다. 수백만 원의 요금이 청구될 수 있습니다.

**해결책: .env 파일에 보관합니다.**

.env 파일은 **금고**와 같습니다:
```
# .env 파일 (로컬 컴퓨터에만 존재)
ANTHROPIC_API_KEY=sk-ant-xxxxx
```

그리고 `.gitignore`에 `.env`를 추가해서 **절대 업로드되지 않게** 합니다:
```
# .gitignore (이 목록의 파일은 GitHub에 안 올림)
.env
```

이렇게 하면 코드는 GitHub에 올라가되, 비밀번호(API 키)는 내 컴퓨터에만 남습니다.

### "비용이 무섭지 않나요?"

많은 분들이 API 비용을 걱정합니다. 현실적인 이야기를 합시다.

**테스트 중에는 거의 무료입니다.**

구체적인 숫자:
- Claude API는 **사용한 만큼만** 과금됩니다 (월정액 아님)
- 짧은 텍스트 처리 한 번에 약 10~50원 수준 (0.01~0.05달러)
- 하루 10번 테스트하면 하루 100~500원
- 한 달 내내 테스트해도 1~2만 원 수준
- 가입 시 무료 크레딧을 주는 경우도 있음

**비용이 늘어난다면 그건 좋은 신호입니다.** 사용자가 많아졌다는 뜻이니까요. 그때 가서 비용 최적화를 고민해도 늦지 않습니다.

지금은 **"비용 걱정 때문에 시작을 못 하는"** 상황만 피하면 됩니다.

## EXECUTE

### 단계 1: 내 서비스에 어울리는 AI 기능 선택

Claude Code에서:
```
내 서비스에 AI 기능을 추가하고 싶어.
현재 서비스의 기능을 보고, AI를 활용하면 좋을 곳을 3가지 제안해줘.

각 제안에 대해:
- 어떤 기능인지 (한 줄)
- 사용자에게 어떤 가치를 주는지
- 구현 난이도 (쉬움/보통/어려움)
- 예상 구현 시간
를 알려줘.

가장 쉬운 것부터 추천해줘.
```

3가지 중 하나를 골랍니다. **가장 쉬운 것을 추천합니다.** 첫 번째 AI 기능은 성공 경험을 쌓는 것이 목표입니다.

### 단계 2: API 키 발급

1. https://console.anthropic.com 접속 → 회원가입 또는 로그인
2. 왼쪽 메뉴에서 "API Keys" 클릭
3. "Create Key" 버튼 클릭
4. 이름 입력 (예: "내 서비스 이름")
5. 생성된 키를 **즉시 복사** (페이지를 벗어나면 다시 볼 수 없습니다!)

> **에러가 나면**: "키가 안 보여요" → API Keys 페이지에서 다시 만들면 됩니다. 키는 무제한 생성 가능합니다.

### 단계 3: .env 설정 (비밀번호 금고)

Claude Code에서:
```
.env 파일에 ANTHROPIC_API_KEY를 추가해줘.
.gitignore에도 .env가 있는지 확인해줘.
"비밀은 코드에 안 쓴다" 원칙을 지켜줘.
```

Claude가 .env 파일을 만들면, 직접 열어서 키를 입력합니다:
```
ANTHROPIC_API_KEY=여기에_복사한_키_붙여넣기
```

> **중요**: `.env` 파일은 직접 열어서 키를 넣어야 합니다. Claude에게 실제 API 키를 알려주지 마세요. Claude는 파일 구조만 만들어주고, 키 입력은 여러분이 합니다.

### 단계 4: AI 기능 구현

이제 선택한 AI 기능을 구현합니다:

```
내 서비스에 [선택한 AI 기능]을 추가해줘.
Claude API를 사용해서 구현해줘.

요구사항:
- 사용자가 [입력]하면 Claude가 [결과]를 돌려주는 기능
- API 키는 환경 변수 ANTHROPIC_API_KEY에서 읽어줘
- 결과가 나오기까지 "처리 중..." 메시지를 보여줘
- 에러가 나면 사용자에게 친절한 메시지를 보여줘 (기술 용어 없이)
- 결과가 너무 길면 적당히 요약해줘
```

### 단계 5: 테스트

서버를 재시작하고 AI 기능을 테스트합니다:

1. 서비스에서 AI 기능이 있는 화면으로 이동
2. 정상적인 입력값으로 테스트 → 결과 확인
3. 짧은 입력으로 테스트 → 잘 처리되는지
4. 긴 입력으로 테스트 → 결과가 너무 길지 않은지
5. 빈 입력으로 테스트 → 에러 메시지가 친절한지

> **에러가 나면**: 오류 메시지를 Claude에게 붙여넣으세요. API 키 관련 오류가 가장 흔한데, 대부분 .env 파일 경로나 키 형식 문제입니다.

### 단계 6: 다듬기

작동이 확인되면, 사용자 경험을 다듬습니다:

```
AI 기능의 사용자 경험을 개선해줘.
- AI 응답이 나오는 동안 로딩 애니메이션 추가
- 결과를 깔끔하게 포맷팅 (목록, 강조 등)
- "다시 물어보기" 버튼 추가
- 입력창에 플레이스홀더 텍스트 추가 (예시 입력)
```

> **막혔을 때**: AI 기능은 외부 서비스(Claude API)와 통신하기 때문에 네트워크 문제가 생길 수 있습니다. "인터넷 연결 확인", "API 키 확인", "요청 형식 확인" — 이 세 가지를 먼저 체크하세요.
