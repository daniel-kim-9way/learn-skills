## EXPLAIN

### 내 서비스에 AI를 더하면?

내 서비스에 AI가 붙으면 사용자 경험이 완전히 달라집니다:
- 독서 기록 앱 → "이 책의 핵심 메시지가 뭐야?" 물어볼 수 있음
- 할 일 관리 앱 → "오늘 할 일 중 우선순위를 AI가 정해줘"
- 식단 관리 앱 → "이 재료로 만들 수 있는 레시피 추천해줘"

AI는 단순한 기능이 아니라 **서비스의 두뇌**가 됩니다. 내가 AI를 직접 만드는 것이 아닙니다. 이미 만들어진 AI(Claude)를 "불러서 쓰는" 것입니다.

### API란? 레스토랑 주문

API(Application Programming Interface)는 어렵게 들리지만 매일 하는 행동과 같습니다.

레스토랑을 상상하세요:
- **손님(나의 서비스)**: "파스타 하나 주세요"
- **웨이터(API)**: 주문을 받아 주방에 전달
- **주방(Claude AI)**: 요리해서 다시 전달
- **웨이터(API)**: 음식을 손님에게 전달

내 서비스가 Claude에게 "이 질문에 답해줘"라고 요청하면 Claude가 답을 돌려주는 구조입니다. 그 연결 통로가 API입니다.

### "비용이 무섭지 않나요?"

많은 분들이 API 비용을 걱정합니다. 현실적인 이야기를 하자면:

- Claude API는 **사용한 만큼만** 과금됩니다 (월정액 아님)
- 짧은 텍스트 처리 한 번에 약 0.01~0.05달러 수준
- 하루 100명이 써도 월 수천 원 수준
- 무료 크레딧을 제공하는 경우도 있음

서비스가 성장해서 비용이 오른다면, 그건 좋은 신호입니다. 사용자가 많아졌다는 뜻이니까요.

### "비밀은 코드에 안 쓴다"

API 키는 비밀번호와 같습니다. 소스 코드에 직접 넣으면 GitHub에 올릴 때 전 세계에 공개됩니다. 악의적인 봇들이 GitHub를 24시간 스캔해서 노출된 API 키를 자동 수집합니다.

해결책: **.env 파일**에 보관하고, .gitignore에 추가해서 절대 업로드되지 않게 합니다.

```
# .env 파일 (로컬에만 존재)
ANTHROPIC_API_KEY=sk-ant-xxxxx

# .gitignore (이 파일은 업로드 안 함)
.env
```

## EXECUTE

### 단계 1: 내 서비스에 어떤 AI가 어울릴지 생각하기

Claude Code에서:
```
내 서비스에 AI 기능을 추가하고 싶어.
현재 서비스의 기능을 보고, AI를 활용하면 좋을 곳을 3가지 제안해줘.
각 제안에 대해:
- 어떤 기능인지
- 사용자에게 어떤 가치를 주는지
- 구현 난이도 (쉬움/보통/어려움)
를 알려줘.
```

제안 중 하나를 골랍니다. 가장 쉬운 것을 추천합니다.

### 단계 2: API 키 발급과 .env 설정

1. https://console.anthropic.com 접속 → 로그인
2. "API Keys" → "Create Key" → 이름 입력 → 생성
3. 발급된 키를 복사 (다시 볼 수 없습니다!)

Claude Code에서:
```
.env 파일에 ANTHROPIC_API_KEY를 추가해줘.
.gitignore에도 .env가 있는지 확인해줘.
"비밀은 코드에 안 쓴다" 원칙을 지켜줘.
```

.env 파일을 열어서 키를 입력합니다:
```
ANTHROPIC_API_KEY=여기에_복사한_키_붙여넣기
```

### 단계 3: AI 기능 구현

```
내 서비스에 [선택한 AI 기능]을 추가해줘.
Claude API를 사용해서 구현해줘.

요구사항:
- 사용자가 [입력]하면 Claude가 [결과]를 돌려주는 기능
- API 키는 환경 변수 ANTHROPIC_API_KEY에서 읽어줘
- 에러가 나면 사용자에게 친절한 메시지를 보여줘
```

### 단계 4: 테스트

서버를 재시작하고 AI 기능을 테스트합니다:
1. 서비스에서 AI 기능이 있는 화면으로 이동
2. 입력값을 넣고 결과 확인
3. 다양한 입력으로 2~3번 더 테스트
