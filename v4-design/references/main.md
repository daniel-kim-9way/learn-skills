## EXPLAIN

### 디자인은 신뢰다

처음 서비스를 만들면 "기능 먼저, 디자인 나중에"라고 생각하기 쉽습니다. 하지만 사용자는 다르게 행동합니다.

사람은 **0.05초** 안에 웹사이트의 신뢰도를 판단합니다. 기능을 써보기도 전에 "이거 믿을 수 있을까?"를 눈으로 결정합니다. 사용자의 75%가 첫인상만으로 서비스의 신뢰를 결정한다는 조사도 있습니다.

명함을 떠올려봅시다. 내용은 같아도 구겨진 종이에 적은 것과 깔끔한 카드에 인쇄한 것은 완전히 다른 인상을 줍니다. 서비스도 마찬가지입니다.

**디자인은 예쁨이 아닙니다. 신뢰입니다.**

### Clean이 이긴다

| | Impressive (화려한) | Clean (깔끔한) |
|--|---------------------|---------------|
| 느낌 | 복잡하고 화려 | 단순하고 정돈됨 |
| 사용자 반응 | "와, 멋있다" → 어디를 눌러야 하지? | "바로 이해된다" → 클릭 |
| 목적 | 인상 주기 | 행동 유도 |

비즈니스 서비스에서는 대부분 **Clean이 이깁니다.** 사용자가 "무엇을 해야 하는지" 바로 알 수 있는 디자인이 전환율을 높입니다.

핵심 원칙 3가지:
1. **여백** — 숨 쉴 공간을 주세요. 빽빽하면 답답합니다
2. **계층** — 크고 굵은 것 vs 작고 얇은 것. 중요도를 눈으로 표현하세요
3. **일관성** — 색상 1~2개, 폰트 1~2개. 통일감이 신뢰를 만듭니다

### Tailwind CSS란?

CSS는 웹의 화장품입니다. 글자 색, 버튼 크기, 간격을 결정합니다.

기존 CSS는 이름을 짓고 별도 파일을 만들어야 했습니다. Tailwind CSS는 다릅니다. 미리 만들어진 수천 개의 "유틸리티 클래스"를 레고 블록처럼 조합합니다:

```html
<!-- Tailwind CSS -->
<button class="bg-blue-500 text-white px-4 py-2 rounded">제출</button>
<!-- 클래스 이름이 곧 스타일 설명 -->
```

Claude에게 "Tailwind로 이 페이지를 개선해줘"라고 하면 알아서 적절한 클래스를 적용해줍니다. 코드를 몰라도 됩니다.

## EXECUTE

### 단계 1: Before 스크린샷 찍기

지금 서비스 화면을 캡처합니다. 나중에 비교하기 위해서입니다.

- `http://localhost:3000` 접속
- 메인 페이지 화면 캡처 (Windows: Win+Shift+S, Mac: Cmd+Shift+4)
- 파일로 저장해두세요

### 단계 2: 3가지 프롬프트로 디자인 개선

**프롬프트 1 — 레이아웃 정리:**
```
메인 페이지 레이아웃을 Tailwind CSS로 정리해줘.
- 여백을 충분히 줘서 숨 쉬는 느낌
- 콘텐츠를 가운데 정렬
- 섹션 간 간격을 넉넉하게
```

**프롬프트 2 — 카드/컴포넌트 개선:**
```
목록이나 카드 형태의 UI를 깔끔하게 개선해줘.
- 카드에 그림자와 라운드 적용
- 호버 효과 추가
- 중요한 정보가 눈에 들어오게 계층 표현
```

**프롬프트 3 — 색상 테마 통일:**
```
전체 색상 테마를 통일해줘.
- 메인 색상 1개, 보조 색상 1개로 제한
- 중요한 버튼은 메인 색상으로 강조
- 배경은 깨끗한 흰색 또는 연한 회색
```

### 단계 3: 반복 다듬기

마음에 안 드는 부분을 콕 집어서 피드백합니다:

```
좋은데, 헤더가 너무 커. 조금 줄여줘.
```

```
버튼이 눈에 안 띄어. 더 눈에 띄게 해줘.
```

2~3번 피드백을 반복하며 마음에 드는 디자인을 찾습니다.

### 단계 4: After 비교

Before 스크린샷과 현재 화면을 나란히 놓고 비교합니다. 같은 서비스인데 느낌이 완전히 다를 것입니다. 이것이 디자인의 힘입니다.
