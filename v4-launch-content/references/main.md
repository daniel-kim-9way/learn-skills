## EXPLAIN

### "알릴 준비"의 진짜 의미

서비스가 인터넷에 떴어도, 사람들이 본인 URL을 처음 봤을 때 **"클릭하고 싶다"**는 첫 인상이 없으면 반쪽이에요.

이 스킬은 그 ‘첫 인상’의 4가지 요소를 챙겨요.

| 요소 | 한 줄 의미 | 왜 중요 |
|---|---|---|
| OG 미리보기 | 카톡에 본인 URL 붙이면 카드 형태로 뜸 | 한국 공유의 99%는 카톡. 못생기면 클릭 안 함 |
| 한 줄 소개 | 3초 안에 "이게 뭐야"에 답 | 첫 화면 헤드라인 = 클릭률 |
| 방문자 분석 | 누가 와서 뭘 보고 갔는지 안다 | 모르면 개선 못 함 |
| 런치 글 | Disquiet / PH / SNS에 올릴 글 미리 작성 | 출시 직후 흥분 상태에서 글이 가장 잘 나옴 |

### OG가 뭐예요? — Open Graph

**OG**(Open Graph)는 페이스북이 2010년에 만든 표준이에요. "이 URL을 다른 데서 공유했을 때 어떻게 보이게 할지"를 페이지 자체가 미리 알려 주는 약속이에요. 카톡, 트위터(X), 디스코드, 슬랙 — 거의 모든 SNS와 메신저가 이 표준을 따라요.

본인이 카톡에 URL 하나 붙여 넣으면 카톡이 뒤에서 그 URL의 HTML을 가져와서 다음 4개 태그를 읽어요.

```html
<meta property="og:title" content="제목">
<meta property="og:description" content="한두 문장 설명">
<meta property="og:image" content="https://본인URL/og.png">
<meta property="og:url" content="https://본인URL">
```

이 4개가 빠지면 카톡에서 **회색 박스 + URL만** 떠요. 끔찍하게 안 예뻐서 클릭률이 절반 이하로 떨어져요. 있으면 큰 이미지 + 제목 + 설명이 카드처럼 떠서 클릭률이 2~3배.

비유하자면 **명함**이에요. 명함 없이 이름만 적힌 종이를 주면 안 받아 가지만, 디자인된 명함은 사람들이 챙겨요. OG = 본인 서비스의 명함.

### "3초 룰" 한 줄 소개

본인 서비스를 처음 보는 사람이 **3초 안에 "이게 뭐야"에 답**할 수 있어야 해요. 그게 한 줄 소개의 역할이에요.

좋은 한 줄 소개의 공식: **"누구에게 + 어떤 결과를 + 어떻게"**

```
나쁜 예                          좋은 예
"AI 기반 차세대 노트 앱"      → "독서가를 위한, 책 1권당 5문장 AI 요약 노트"
"혁신적인 솔루션"             → "1인 창업자가 30일에 본인 SaaS를 출시하는 코치"
"새로운 가능성"               → "냉장고 재료 3개로 30초 안에 레시피 1개 추천"
```

‘혁신’ ‘차세대’ ‘솔루션’ 같은 단어는 다 빼세요. 추상적이면 다 나쁜 카피.

### 런치 글 3곳 — 왜 이 3곳

| 사이트 | 대상 | 언어 + 톤 | 권장 시점 |
|---|---|---|---|
| Disquiet (디스콰이엇) | 한국 인디 메이커 커뮤니티 | 한국어, 진솔, 회고 톤 | 첫 사용자 5명 직후 1주 안 |
| Product Hunt | 글로벌 메이커 + 얼리어답터 | 영어, 짧고 간결, 가치 강조 | 화·수 한국시각 오후 5시 권장 |
| 본인 SNS | 본인 친구·팔로워 | 평소 본인 톤 그대로 | 첫 사용자 직후 즉시 |

3곳을 동시에 노리는 이유는 **세 그룹의 사람이 다 다르기** 때문이에요. Disquiet는 한국 메이커, Product Hunt는 해외 얼리어답터, SNS는 본인 친구. 한 글로 다 못 잡아요. 3개 따로 써야 해요.

런치 글의 공통 5단락 골격:

```
1. 한 줄  : 본인 + 만든 것 한 줄 (3초 룰 그대로)
2. 왜   : 본인이 겪은 문제 (Ch.2-4 pain)
3. 어떻게 다른가: 차별점 1가지 (Ch.2-6 solution)
4. 가격 + 시작 방법
5. 피드백 부탁 + 본인 연락처
```

#### og:image (OG 이미지) 만들기

> "OG 이미지가 카드의 70%를 차지해서 가장 신경 써야 해요. 사이즈는 1200x630px 권장. 첫 사용자 5명 단계에서는 다음 3가지 중 하나로 충분해요."

```
가장 빠름      → Canva 무료 템플릿 "Open Graph Image" 검색 →
                 본인 서비스명 + 한 줄 입력 → PNG 다운로드

중간          → Claude Code에 "vercel/og 같은 라이브러리로
                 OG 이미지 자동 생성 라우트 만들어 줘" 한 줄

가장 자동화    → 랜딩 페이지 첫 화면 스크린샷을 그대로 OG 이미지로
```

> "어느 쪽으로 갈래요?"

학생 답에 따라 안내:

- Canva → "https://canva.com 접속 → 검색창에 ‘Open Graph Image’ → 마음에 드는 템플릿 → 본인 서비스명 + 한 줄 소개 입력 → ‘다운로드’ 버튼 → PNG. 그 PNG 파일을 본인 빌드킷의 `app/assets/images/og.png`에 두고 git commit + push."
- vercel/og 라이브러리 → "Claude Code에 그대로 한 줄. 본인 프로젝트가 Rails라면 Ruby로 OG 이미지 라우트, Node라면 Next.js의 vercel/og 직접."
- 스크린샷 → "본인 랜딩 페이지를 데스크탑 화면 가득 띄우고 1200x630 크롭 캡쳐. 빠르고 본인 디자인 그대로라는 장점."

**"카카오 디버거에서는 OG 잘 떠요. 그런데 카톡 채팅에서는 회색 박스만 떠요."**
