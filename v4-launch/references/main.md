## EXPLAIN

### 완벽은 완료의 적이다

서비스를 만들다 보면 이런 생각이 듭니다:
- "이 기능만 더 넣으면..."
- "디자인이 아직 좀 아쉬운데..."
- "버그가 하나 더 있는 것 같아서..."

이러다가 오픈을 계속 미루는 분들이 많습니다. 하지만 실제로 써보지 않으면 무엇이 진짜 중요한지 알 수 없습니다.

**페이스북 초창기 슬로건**: "Done is better than perfect" (완료가 완벽보다 낫다)

첫 고객 10명이 "이게 불편해요"라고 하면 그때 고치면 됩니다. 혼자 상상하는 "완벽"보다 실제 사용자의 피드백이 훨씬 가치 있습니다.

### 필수 5개 vs Nice-to-have

오픈 전 **반드시** 확인할 것:
1. **서비스가 돌아간다** — 핵심 기능이 오류 없이 작동
2. **결제가 된다** (결제 서비스라면) — 돈이 오가는 흐름 확인
3. **개인정보 처리방침** — 법적 의무
4. **연락처** — 문제 시 사용자가 연락할 방법
5. **보안 체크리스트 통과** — 기본 보안 완료

**나중에 해도 되는 것**:
- 완벽한 디자인, 모든 기능 완성, SEO 최적화, 다국어 지원

### 공유 최적화: Meta Tags + OG Image

링크를 카카오톡이나 SNS에 공유하면 미리보기가 나옵니다. 이 미리보기를 결정하는 것이 **Meta Tags**와 **OG Image**입니다.

- **og:title** — 링크 제목
- **og:description** — 한 줄 설명
- **og:image** — 미리보기 이미지

이걸 설정하지 않으면 공유했을 때 빈 박스만 나옵니다. 설정하면 깔끔한 미리보기가 나와서 클릭률이 높아집니다.

### LAUNCH의 진짜 의미

오픈은 끝이 아닙니다. **새로운 시작**입니다.

오픈 → 실제 사용자 피드백 → 개선 → 더 많은 사용자 → 더 많은 피드백 → ...

이 사이클을 시작하는 것이 LAUNCH입니다.

## EXECUTE

### 단계 1: Meta Tags + OG Image 설정

Claude Code에서:
```
내 서비스의 Meta Tags와 OG Image를 설정해줘.

설정할 것:
- og:title — 서비스 이름 + 한 줄 설명
- og:description — 서비스가 뭘 하는지 60자 이내
- og:image — 서비스 대표 이미지 (없으면 스크린샷 사용)
- favicon 설정
- 모바일 viewport 메타 태그 확인

카카오톡과 SNS에서 공유했을 때 미리보기가 잘 나오게 해줘.
```

### 단계 2: 모바일 테스트

폰에서 라이브 URL에 접속합니다:
1. 서비스 URL을 폰 브라우저에 입력
2. 핵심 기능을 처음부터 끝까지 테스트
3. 글자가 너무 작거나, 버튼이 눌리지 않거나, 레이아웃이 깨지는 곳 확인

문제가 있으면:
```
모바일에서 [문제 설명]. 수정해줘.
```

### 단계 3: 카운트다운 의식

모든 준비가 끝났습니다. 잠시 멈추고 이 순간을 느껴봅니다.

당신은:
- 아이디어를 생각했고 (C1)
- 기획서를 만들었고 (C2)
- 코드를 작성했고 (C3)
- 디자인하고, 보안 점검하고, 배포했습니다 (C4)

**카운트다운:**
```
3... 서비스가 돌아갑니다
2... 보안 점검을 마쳤습니다
1... 세상이 기다리고 있습니다
🚀 LAUNCH!
```

### 단계 4: 첫 공유 메시지 작성

Claude에게:
```
내 서비스를 처음 공유할 메시지를 써줘.
카카오톡, SNS, 커뮤니티에 올릴 수 있게.

포함할 것:
- 서비스 이름과 한 줄 설명
- 누구를 위한 서비스인지
- 서비스 URL
- 피드백 요청 (부담 없이 써달라는 느낌)

자연스럽고 진정성 있는 톤으로 써줘.
```

작성된 메시지를 복사해서 주변에 공유합니다. 가족, 친구, 동료 — 누구든 좋습니다.
