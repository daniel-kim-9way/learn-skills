## EXPLAIN

### [선택] 이 레슨은 선택사항입니다

이 레슨은 **선택 레슨**입니다. 결제 기능이 필요한 서비스만 진행하면 됩니다.

무료 서비스를 운영할 계획이라면 건너뛰어도 됩니다. 나중에 결제가 필요해지면 그때 돌아와서 진행하면 됩니다.

### "실수로 돈이 빠져나가면 어쩌지?"

결제 연동이라고 하면 가장 먼저 드는 걱정이 있습니다.

> "실수로 돈이 빠져나가면 어쩌지?"
> "내 카드가 결제되면 어쩌지?"
> "코드를 잘못 짜서 이중 결제가 되면?"

**걱정하지 마세요. 테스트 모드에서는 실제 돈이 절대 움직이지 않습니다.**

테스트 모드는 **게임 머니**와 같습니다. 진짜 돈이 아닌 가짜 카드 번호를 사용하고, 결제가 승인되어도 실제 청구는 없습니다. 100% 안전합니다.

실제로 수백 번 결제를 테스트해도 통장 잔고에는 아무 변화가 없습니다.

테스트 모드에서 충분히 연습한 후, 서비스를 정식으로 오픈할 때 "실제 모드"로 전환하면 됩니다. 그 전환도 키 하나만 바꾸면 끝입니다.

### 결제는 어떻게 일어나는가?

결제 흐름을 **택배 시스템**으로 설명합니다:

```
[1] 고객이 "결제하기" 버튼을 누른다
     ↓
[2] 결제 게이트웨이(Toss Payments)가 결제창을 띄운다
    "카드 번호를 입력하세요"
     ↓
[3] 고객이 카드 정보를 입력한다
    (카드 정보는 Toss가 처리 — 내 서버에는 저장 안 됨!)
     ↓
[4] Toss가 카드사에 "이 결제 승인해주세요" 요청
     ↓
[5] 카드사: "승인!" (또는 "잔액 부족!")
     ↓
[6] Toss가 내 서비스에 결과를 알려준다
    "결제 성공했어요!" 또는 "실패했어요"
     ↓
[7] 내 서비스: 성공이면 → 주문 완료 화면
              실패면 → 다시 시도 안내
     ↓
[8] (며칠 후) Toss가 내 통장에 정산 입금
```

**핵심 포인트**: 고객의 카드 정보는 **내 서버에 저장되지 않습니다.** Toss Payments가 모든 카드 처리를 대신합니다. 보안 걱정을 할 필요가 없습니다.

직접 카드사와 계약하면 수억 원과 1년 이상이 필요합니다. PCI-DSS라는 국제 보안 인증도 받아야 합니다. 결제 게이트웨이가 이 모든 걸 대신해줍니다. 수수료는 결제 금액의 2~3% 수준입니다.

### 왜 Toss Payments인가?

한국에서 결제 게이트웨이를 고를 때 Toss Payments를 추천하는 이유:

| 장점 | 설명 |
|------|------|
| **개발자 친화적 문서** | 설명이 쉽고 예제가 풍부합니다. 비개발자도 이해할 수 있는 수준 |
| **테스트 모드** | 실제 돈 없이 전 과정을 안전하게 테스트 |
| **빠른 심사** | 개인 사업자도 빠른 심사가 가능 |
| **국내 최적화** | 카카오페이, 네이버페이, 삼성페이 모두 지원 |
| **합리적 수수료** | 일반 카드 결제 약 3.3% |

다른 옵션(아임포트, 부트페이 등)도 있지만, 처음 시작하기에는 Toss가 가장 쉽습니다.

### 테스트 모드 vs 실제 모드

| | 테스트 모드 | 실제 모드 |
|--|------------|----------|
| **API 키** | test_ck_xxx / test_sk_xxx | live_ck_xxx / live_sk_xxx |
| **돈** | 가짜 (테스트 카드 번호) | 진짜 돈 |
| **목적** | 기능 확인, 버그 찾기 | 서비스 운영 |
| **전환 방법** | 테스트 키 → 실제 키 교체 (한 줄 변경) | |
| **필요 조건** | 없음 (바로 사용) | 사업자등록 + Toss 심사 |

**지금은 테스트 모드에만 집중합니다.** 실제 모드는 서비스를 정식 오픈할 때 전환하면 됩니다.

### 사업자등록 안내 (실제 결제시 필요)

실제 돈을 받으려면 **사업자등록**이 필요합니다. 지금 당장 할 필요는 없지만, 미리 알아두면 좋습니다.

**개인사업자 등록 절차:**
1. 국세청 홈택스 (hometax.go.kr) 접속
2. "사업자등록 신청" → 온라인으로 가능
3. 업종 선택: "정보통신업" 또는 "소프트웨어 개발업"
4. 처리 기간: 보통 1~3 영업일
5. 비용: 무료

사업자등록 후 Toss Payments에서:
1. "실제 모드 신청"
2. 사업자등록번호 입력
3. 정산 받을 계좌 입력
4. 심사 (보통 1~3 영업일)
5. 승인 후 실제 키 발급

> **꿀팁**: 사업자등록은 무료이고, 연 매출이 작으면 세금도 거의 없습니다. "사업자"라는 단어가 부담스럽지만, 실제로는 온라인에서 10분이면 끝나는 간단한 절차입니다.

## EXECUTE

### 단계 1: Toss Payments 테스트 계정 설정

1. https://developers.tosspayments.com 접속
2. "시작하기" → 회원가입 (이메일 또는 소셜 로그인)
3. 로그인 후 대시보드에서 "개발 정보" 확인
4. "테스트 키" 확인:
   - **클라이언트 키**: `test_ck_`로 시작 (브라우저에서 사용)
   - **시크릿 키**: `test_sk_`로 시작 (서버에서 사용)
5. 두 키를 **모두** 복사

이제 .env 파일에 저장합니다:
```
# .env 파일에 추가
TOSS_CLIENT_KEY=test_ck_xxxxx여기에키붙여넣기
TOSS_SECRET_KEY=test_sk_xxxxx여기에키붙여넣기
```

> **중요**: 시크릿 키는 절대 브라우저 코드(JavaScript)에 넣지 마세요. 서버 코드에서만 사용합니다. Claude에게 맡기면 알아서 올바른 위치에 넣어줍니다.

### 단계 2: 가격표 페이지 만들기

결제 전에 사용자가 "뭘 결제하는지" 알아야 합니다. 가격표 페이지를 먼저 만듭니다.

Claude Code에서:
```
서비스 가격표 페이지를 만들어줘.

요구사항:
- 요금제 카드 형태로 표시 (1~3개)
- 각 카드에: 요금제 이름, 가격, 포함 기능 목록
- "결제하기" 버튼
- 인기 요금제에 "추천" 배지
- Tailwind CSS로 깔끔하게
- 모바일 반응형
```

### 단계 3: 결제 플로우 구현

이제 핵심인 결제 연동입니다:

```
Toss Payments 테스트 모드로 결제 기능을 연동해줘.

환경 변수:
- TOSS_CLIENT_KEY (클라이언트 키)
- TOSS_SECRET_KEY (시크릿 키)

구현할 것:
1. 가격표 페이지에서 "결제하기" 클릭
2. Toss Payments 결제창 오픈 (카드/간편결제 선택)
3. 결제 성공 시 → 성공 페이지로 이동 + 주문 기록 저장
4. 결제 실패 시 → 실패 페이지로 이동 + 에러 메시지 표시
5. 결제 취소 시 → 가격표 페이지로 복귀

참고: Toss Payments 공식 문서 https://docs.tosspayments.com
```

> **에러가 나면**: 결제 연동에서 가장 흔한 오류는 "키가 올바르지 않습니다"입니다. test_ck_와 test_sk_ 키가 제대로 들어갔는지 확인하세요. 앞뒤에 공백이 있으면 안 됩니다.

### 단계 4: 테스트 결제 실행

드디어 테스트 결제를 해봅니다:

1. 서비스에서 가격표 페이지로 이동
2. "결제하기" 버튼 클릭
3. Toss 결제창이 뜨면 → **테스트 카드 정보 입력**:
   - 카드 번호: `4330000000000000`
   - 유효기간: 아무 미래 날짜 (예: 12/28)
   - CVC: 아무 3자리 (예: 123)
   - 비밀번호 앞 2자리: 아무 2자리 (예: 12)
4. "결제하기" 클릭
5. **결제 성공!** → 성공 페이지가 나오는지 확인

성공 페이지가 나왔나요? **축하합니다!** 결제 연동이 완성되었습니다.

> **꿀팁**: 실패 케이스도 테스트하세요. 잘못된 카드 번호를 넣어보고, 실패 페이지가 제대로 나오는지 확인합니다. 성공만 테스트하면 실패할 때 사용자가 막막해집니다.

### 단계 5: Toss 대시보드에서 확인

https://developers.tosspayments.com 대시보드에서:
1. "결제 내역" 탭 클릭
2. 방금 테스트한 결제가 기록되어 있는지 확인
3. 결제 금액, 상태, 시간 등 정보 확인

이 대시보드가 앞으로 결제 관리의 중심이 됩니다.

### 단계 6: 결제 경험 다듬기

기본 결제가 되면, 사용자 경험을 다듬습니다:

```
결제 사용자 경험을 개선해줘.

개선할 것:
- 결제 성공 페이지에 주문 번호, 결제 금액 표시
- 결제 실패 페이지에 "다시 시도" 버튼
- 결제 진행 중 로딩 표시
- 가격표 페이지에서 가격 표시 형식 (예: ₩9,900/월)
- 결제 내역을 볼 수 있는 마이페이지 (선택)
```

> **막혔을 때**: 결제 관련 오류는 Toss Payments 공식 문서(docs.tosspayments.com)에 에러 코드별 해결 방법이 있습니다. 오류 코드를 Claude에게 알려주면 문서를 참고해서 해결해줍니다.
