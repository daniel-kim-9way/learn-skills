## EXPLAIN

### "테스트 모드에서는 실제 돈이 안 오갑니다"

결제 연동이라고 하면 "실수로 돈이 빠져나가면 어쩌지?" 걱정이 되실 수 있습니다. 걱정 마세요. **테스트 모드**에서는 가짜 카드 번호를 사용하고, 실제 돈이 이동하지 않습니다. 100% 안전합니다.

테스트 모드에서 충분히 연습한 후, 실제 서비스를 오픈할 때 "실제 모드"로 전환하면 됩니다.

### 결제는 어떻게 일어나는가?

결제 흐름을 택배 시스템으로 설명하면:

```
고객 (카드 정보 입력)
   ↓
결제 게이트웨이 (Toss Payments) — "이 택배 받아서 처리할게요"
   ↓ 카드사와 협상
카드사 (결제 승인)
   ↓
결제 게이트웨이 → 나의 서비스 "승인됐어요!" 알림
   ↓
나의 서비스: 주문 완료 처리
   ↓ (며칠 후)
내 통장에 정산 입금
```

직접 카드사와 계약하면 수억 원과 1년 이상이 필요합니다. 결제 게이트웨이가 이 모든 걸 대신해줍니다. 수수료는 결제 금액의 2~3% 수준입니다.

### 왜 Toss Payments인가?

| 장점 | 설명 |
|------|------|
| 개발자 친화적 문서 | 설명이 쉽고 예제가 풍부합니다 |
| 테스트 모드 | 실제 돈 없이 전 과정을 테스트합니다 |
| 빠른 심사 | 개인 사업자도 빠른 심사가 가능합니다 |
| 국내 최적화 | 카카오페이, 네이버페이, 삼성페이 지원 |

### 테스트 모드 vs 실제 모드

| | 테스트 모드 | 실제 모드 |
|--|------------|----------|
| 돈 | 가짜 (테스트 카드) | 실제 돈 |
| 목적 | 기능 확인 | 서비스 운영 |
| 전환 | 테스트 키 → 실제 키 교체 | |

실제 모드 전환 시 **사업자등록** 또는 개인 계좌 인증이 필요합니다. 지금은 테스트 모드에만 집중합니다.

## EXECUTE

### 단계 1: Toss Payments 테스트 계정 등록

1. https://developers.tosspayments.com 접속
2. "시작하기" → 회원가입
3. 대시보드에서 "테스트 키" 확인:
   - 클라이언트 키 (test_ck_로 시작)
   - 시크릿 키 (test_sk_로 시작)
4. 두 키를 복사해서 .env 파일에 보관

```
# .env
TOSS_CLIENT_KEY=test_ck_xxxxx
TOSS_SECRET_KEY=test_sk_xxxxx
```

### 단계 2: 가격 페이지 + 결제 흐름 구현

Claude Code에서:
```
Toss Payments 테스트 모드로 결제 기능을 연동해줘.

요구사항:
- 가격 페이지: 서비스 요금을 보여주는 페이지
- 결제 버튼을 누르면 Toss 결제창 오픈
- 결제 성공 시 완료 페이지로 이동
- 결제 실패 시 오류 메시지 표시

환경 변수:
- TOSS_CLIENT_KEY
- TOSS_SECRET_KEY

공식 문서: https://docs.tosspayments.com
```

### 단계 3: 테스트 결제 실행

1. 서비스에서 결제 화면으로 이동
2. 결제 버튼 클릭 → Toss 결제창 확인
3. 테스트 카드 입력:
   - 카드 번호: `4330000000000000`
   - 유효기간: 아무 미래 날짜 (예: 12/28)
   - CVC: 아무 3자리 (예: 123)
4. 결제 완료 → 성공 페이지 확인
5. Toss 대시보드에서 테스트 결제 내역 확인

실제 서비스 오픈 시 참고:
- 사업자등록 필요 (개인사업자도 가능)
- 테스트 키를 실제 키로 교체
- 정산 주기: 보통 D+7 (결제 후 7일 뒤 입금)
