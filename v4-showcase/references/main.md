## EXPLAIN

### 엘리베이터 피치란?

엘리베이터를 타는 30초 동안 투자자를 만났을 때 서비스를 설명할 수 있는 능력을 **엘리베이터 피치**라고 합니다.

30초 피치 구조:

```
"[공감 질문] 혹시 [이런 경험] 있으신가요?
[해결책] 저는 [이런 사람들]을 위해 [서비스 이름]을 만들었습니다.
[핵심 기능] [한 가지 기능]으로 [결과/혜택]을 줍니다.
[행동 촉구] 한번 써보시겠어요?"
```

예시:
"혹시 책을 읽고 나서 내용을 금방 잊어버리는 경험 있으신가요? 저는 독서광들을 위해 '북메모리'를 만들었습니다. AI가 핵심 내용을 정리하고 복습 알림을 보내서 책 내용을 오래 기억할 수 있게 도와드립니다. 한번 써보시겠어요?"

### 피치의 심리: 스토리가 이긴다

데이터: "독서 앱, 기억 보조 기능 포함"
스토리: "책을 읽고 나서 90%를 잊어버리는 고통을 해결합니다"

같은 서비스이지만 스토리가 훨씬 강력합니다. 듣는 사람이 자기 경험을 투영하기 때문입니다.

### Showcase에서의 첫인상

Vuild Showcase는 서비스 빌더들이 만든 서비스를 공유하는 공간입니다. 투자자, 잠재 사용자, 다른 빌더들이 둘러봅니다.

서비스 목록에서 클릭을 받으려면 **제목과 한 줄 설명**이 전부입니다. 5~10개를 훑어보고 흥미로운 것만 클릭합니다.

나쁜 예: "독서 관리 앱입니다"
좋은 예: "책 읽고 나서 90%를 잊어버리는 분들을 위한 AI 독서 노트"

### C1~C4 여정 돌아보기

잠시 멈추고 여기까지 온 길을 돌아봅시다:

- **C1**: 아이디어를 발견하고, 문제를 정의하고, 서비스 이름을 지었습니다
- **C2**: 기획서를 만들고, 기능을 설계했습니다
- **C3**: Claude Code를 설치하고, 실제로 코드를 작성하고, 서비스를 만들었습니다
- **C4**: 디자인하고, 보안 점검하고, 배포하고, 세상에 LAUNCH했습니다

몇 주 전만 해도 "코딩을 모르는 나"였습니다. 지금은 **실제 서비스를 만들어서 인터넷에 올린 사람**입니다.

## EXECUTE

### 단계 1: 엘리베이터 피치 작성

Claude에게:
```
내 서비스의 30초 엘리베이터 피치를 써줘.
구조:
1. 공감 질문으로 시작
2. 서비스 이름 + 대상 소개
3. 핵심 기능 + 혜택
4. 행동 촉구

자연스럽게 말할 수 있는 구어체로 써줘.
3가지 버전을 만들어줘.
```

3가지 중 가장 마음에 드는 것을 골라 소리 내어 읽어봅니다. 30초 안에 끝나는지 시간을 재봅니다.

### 단계 2: Vuild Showcase 등록

1. https://vuild.kr/showcase 접속
2. "내 서비스 등록" 클릭
3. 정보 입력:
   - 서비스 이름
   - 한 줄 설명 (60자 이내)
   - 서비스 URL
   - 대표 이미지 (스크린샷)
   - 카테고리

한 줄 설명 도움:
```
내 서비스의 Showcase 한 줄 설명을 3가지 버전으로 써줘.
- 대상: [타겟 사용자]
- 해결하는 문제: [문제]
- 핵심 기능: [기능]
60자 이내로.
```

### 단계 3: 여정 회고

Claude와 함께 여정을 돌아봅니다:
```
내가 이 서비스를 만들면서 가장 어려웠던 순간과
가장 뿌듯했던 순간을 정리해줘.
CLAUDE.md와 지금까지의 기록을 바탕으로.
```

### 단계 4: "당신은 서비스 빌더입니다"

아이디어에서 시작해서 실제 서비스를 만들고, 배포하고, 세상에 공유했습니다.

**당신은 서비스 빌더입니다.**

이제 코딩은 낯선 것이 아닙니다. Claude와 함께 무엇이든 만들 수 있습니다. 다음 아이디어가 떠오르면 또 만들면 됩니다. 이번에는 훨씬 빠르게.

### 단계 5: C5~C8 독립 트랙 미리보기

C1~C4는 필수 여정이었습니다. 여기서부터는 **독립 트랙**입니다. 관심 있는 것을 골라서 순서 상관없이 학습합니다:

- **C5: 성장** — 사용자 분석, A/B 테스트, 마케팅 기초
- **C6: 수익화** — 구독 모델, 프리미엄 기능, 가격 전략
- **C7: 팀빌딩** — 협업 도구, 코드 리뷰, 프로젝트 관리
- **C8: 스케일링** — 성능 최적화, 인프라 확장, 모니터링

필요할 때 돌아오면 됩니다.
