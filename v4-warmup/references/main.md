## EXPLAIN

### 쉬었다 돌아온 당신에게

며칠, 혹은 몇 주가 지났을 수 있습니다. 컴퓨터를 켜고 프로젝트 폴더를 열었을 때 "내가 뭘 만들었더라?" 하는 막막함이 밀려올 수 있습니다.

**이건 완전히 정상입니다.**

프로 개발자도 월요일 아침에 같은 경험을 합니다. 금요일에 짜던 코드가 월요일에는 남의 코드처럼 보입니다. 뇌는 자주 쓰지 않는 정보를 뒤로 밀어두기 때문입니다. 잊어버린 것이 아니라, 잠시 잠든 것뿐입니다.

그래서 필요한 것이 **워밍업**입니다. 축구 선수가 경기 전에 스트레칭하듯, 우리도 코딩 전에 감각을 깨우는 시간을 가집니다.

### 컨텍스트 회복이란?

컨텍스트(context)는 "맥락"이란 뜻입니다.

내가 만든 서비스의 맥락이란:
- **왜** 만들었는지 (C1에서 발견한 문제)
- **어디까지** 했는지 (C3에서 만든 기능들)
- **다음에 뭘** 해야 하는지 (C4: LAUNCH를 향한 여정!)

이 감각을 되찾는 것이 **컨텍스트 회복**입니다.

가장 빠른 방법 두 가지가 있습니다:

**1. 직접 실행해보기**
눈으로 보면 바로 기억이 돌아옵니다. "아, 맞다. 이 버튼을 누르면 이렇게 되는 거였지." 손이 기억을 불러옵니다.

**2. Claude에게 설명 시키기**
"우리가 뭘 만들었어?"라고 물어보면 Claude가 코드를 읽고 기능 중심으로 요약해줍니다. 기술 용어 없이, "사용자가 로그인하면 이 화면이 나오고, 버튼을 누르면 저 기능이 동작해요" 같은 방식으로 듣는 것이 포인트입니다.

### 작은 변경 = 손 풀기

운동 선수도 경기 전에 스트레칭을 합니다. 코딩도 마찬가지입니다.

버튼 하나의 색을 바꾸거나, 텍스트를 수정하는 **아주 작은 변경**을 해봅니다. "아, 이렇게 하는 거였지"라는 감각이 돌아옵니다. 이 작은 성공 하나가 자신감을 되살려줍니다.

오늘은 **LAUNCH를 향한 C4 여정의 첫 걸음**입니다.

C4의 목표는 명확합니다:
> **작동한다 → 멋져보인다 → 세상이 본다 → LIVE!**

가볍게 시작합시다.

### 오늘 하루의 여정 미리보기

C4는 이렇게 흘러갑니다:

```
[워밍업] 손 풀기, 감각 회복 ← 지금 여기
   ↓
[디자인] 내 서비스에 신뢰감 입히기
   ↓
[랜딩] 처음 방문한 사람을 사로잡는 페이지
   ↓
[보안] 사용자를 보호하는 방패
   ↓
[배포] localhost → 인터넷! ★ THE MOMENT
   ↓
[런치] 세상에 공유하기
   ↓
[쇼케이스] 서비스 빌더로 데뷔
```

이 여정의 끝에서 당신은 **실제 URL을 가진 서비스의 주인**이 됩니다. 그 URL을 카카오톡으로 보내면 누구든 볼 수 있습니다.

설레지 않나요? 시작합시다.

## EXECUTE

### 단계 1: 서비스 실행하기

프로젝트 폴더에서 서비스를 시작합니다:

```bash
bin/rails server
# 또는
bin/dev
```

브라우저에서 `http://localhost:3000`을 열어봅니다.

천천히 클릭해보세요. 메인 페이지부터 시작해서, 로그인도 해보고, 기능도 눌러보세요. "아, 이런 것도 만들어뒀었구나" 하는 순간이 올 겁니다.

> **막혔을 때**: 서버가 안 켜지면 오류 메시지를 그대로 복사해서 Claude에게 붙여넣으세요. 대부분 한 번에 해결됩니다.

### 단계 2: Claude에게 서비스 요약 요청

Claude Code를 열고 아래와 같이 말합니다:

```
우리가 만든 서비스를 기능 중심으로 요약해줘.
기술 용어 없이, 사용자 입장에서 "이걸 누르면 이게 된다" 식으로 설명해줘.
현재 있는 페이지 목록과 각 페이지에서 할 수 있는 것도 정리해줘.
```

Claude가 요약하면 맞는지 확인합니다. 틀린 부분이 있으면 정정해줍니다. "아니, 그건 아직 안 만들었어" 하고 알려주면 됩니다.

이 과정을 통해 "아, 여기까지 만들었구나"가 선명해집니다.

> **꿀팁**: Claude의 요약을 읽으면서 "C4에서 이걸 개선해야겠다" 하는 아이디어가 떠오르면 메모해두세요. 곧 쓸 거예요.

### 단계 3: 작은 수정 — 손 풀기

감각을 되살리기 위해 작은 변경 하나를 합니다:

```
메인 페이지에서 가장 눈에 띄는 버튼 색을 바꿔줘. 지금 느낌과 다른 색으로.
```

변경 후 브라우저를 새로고침(F5)해서 확인합니다.

바뀌었나요? **축하합니다. 워밍업 완료입니다.**

이 작은 변경 하나가 증명하는 것이 있습니다:
- 나는 서비스를 수정할 수 있다
- Claude가 내 의도를 이해하고 코드를 바꿔준다
- 브라우저에서 바로 결과를 확인할 수 있다

이 감각이 C4 전체를 관통합니다. 이제 본격적으로 LAUNCH를 향해 달려봅시다.

### 단계 4: C4 여정 확인

마지막으로 Claude에게 물어봅니다:

```
지금 내 서비스를 "인터넷에 공개"하려면 부족한 것이 뭐야?
디자인, 기능, 보안, 배포 관점에서 각각 정리해줘.
```

이 답변이 C4에서 해야 할 일의 로드맵이 됩니다. 앞으로의 레슨에서 하나씩 해결해 나갈 거예요.
