## EXPLAIN

### 쉬고 나면 왜 기억이 흐릿해질까?

며칠 또는 몇 주 동안 손을 놓으면 "내가 뭘 만들었더라?" 하는 막막함이 찾아옵니다. 프로 개발자도 같은 경험을 합니다. 뇌는 자주 쓰지 않는 정보를 뒤로 밀어두기 때문입니다.

완전히 잊어버린 것이 아닙니다. 잠시 잠든 것뿐입니다. 그래서 필요한 것이 **워밍업**입니다.

### 컨텍스트 회복이란?

컨텍스트(context)는 "맥락"이란 뜻입니다. 내가 만든 서비스의 맥락 — 왜 만들었는지, 어디까지 했는지, 다음에 뭘 해야 하는지 — 이 감각을 되찾는 것이 **컨텍스트 회복**입니다.

가장 빠른 방법 두 가지:
1. **직접 실행해보기** — 눈으로 보면 바로 기억이 돌아옵니다
2. **Claude에게 설명 시키기** — "우리가 뭘 만들었어?"라고 물어보면 코드를 읽고 기능 중심으로 요약해줍니다

기술 용어 없이, "사용자가 로그인하면 이 화면이 나오고, 버튼을 누르면 저 기능이 동작해요" 같은 방식으로 듣는 것이 포인트입니다.

### 작은 변경 = 손 풀기

운동 선수도 경기 전에 스트레칭을 합니다. 코딩도 마찬가지입니다. 버튼 하나의 색을 바꾸거나 텍스트를 수정하는 **아주 작은 변경**을 해보면 "아, 이렇게 하는 거였지"라는 감각이 돌아옵니다.

오늘은 LAUNCH를 향한 여정의 첫 걸음입니다. 가볍게 시작합시다.

## EXECUTE

### 단계 1: 서비스 실행하기

프로젝트 폴더에서 서비스를 시작합니다:

```bash
bin/rails server
# 또는
bin/dev
```

브라우저에서 `http://localhost:3000`을 열어 직접 클릭해보며 어디까지 만들었는지 확인합니다.

### 단계 2: Claude에게 서비스 요약 요청

Claude Code를 열고 아래와 같이 말합니다:

```
우리가 만든 서비스를 기능 중심으로 요약해줘.
기술 용어 없이, 사용자 입장에서 "이걸 누르면 이게 된다" 식으로 설명해줘.
```

Claude가 요약하면 맞는지 확인합니다. 틀린 부분이 있으면 정정해줍니다.

### 단계 3: 작은 수정 — 손 풀기

감각을 되살리기 위해 작은 변경 하나를 합니다:

```
메인 페이지에서 가장 눈에 띄는 버튼 색을 바꿔줘. 지금 느낌과 다른 색으로.
```

변경 후 브라우저를 새로고침(F5)해서 확인합니다. 바뀌었으면 워밍업 완료입니다.
