## EXPLAIN

### 무엇을 측정해야 하는가?

서비스를 운영하면 수많은 숫자가 나옵니다. 방문자 수, 페이지뷰, 클릭 수... 하지만 **모든 숫자가 중요한 건 아닙니다.**

### 허영 지표 vs 실행 지표

**허영 지표(Vanity Metrics)**: 보기엔 좋지만 의사결정에 도움이 안 되는 숫자
- 총 가입자 수 (탈퇴한 사람 포함)
- 페이지뷰 (같은 사람이 100번 방문해도 100)
- SNS 좋아요 수

**실행 지표(Actionable Metrics)**: 실제 의사결정에 도움이 되는 숫자
- **활성 사용자(DAU/WAU)** — 실제로 서비스를 쓰는 사람 수
- **기능 사용률** — 핵심 기능을 사용하는 비율
- **이탈률** — 가입 후 다시 안 오는 비율
- **전환율** — 방문자 중 가입하는 비율

### 핵심 지표 5가지

서비스 초기에 집중해야 할 지표:

1. **DAU (일간 활성 사용자)** — 매일 서비스를 쓰는 사람 수
2. **리텐션율** — 이번 주 사용자 중 다음 주에도 오는 비율
3. **핵심 기능 사용률** — 사용자의 몇 %가 핵심 기능을 사용하는가
4. **전환 퍼널** — 방문 → 가입 → 핵심 기능 사용 각 단계의 전환율
5. **NPS (순추천지수)** — "이 서비스를 추천하시겠습니까?" 0~10 점수

### 데이터에서 패턴 찾기

데이터를 볼 때 주목할 패턴:
- **시간대 패턴** — 사용자가 주로 언제 접속하는가?
- **이탈 지점** — 어디서 사용자가 떠나는가?
- **성공 경로** — 결제까지 가는 사용자는 어떤 경로를 거치는가?
- **코호트 분석** — 가입 시기별로 행동이 다른가?

## EXECUTE

### 단계 1: 현재 데이터 확인

이전 블록에서 설치한 Ahoy 데이터를 확인합니다. Claude에게 요청합니다:

```
Ahoy에 쌓인 분석 데이터를 확인해줘.
- 최근 방문 기록 (visits)
- 사용자 행동 이벤트 (events)
- 가장 많이 방문한 페이지 Top 5
- 요일별, 시간대별 방문 패턴
```

아직 데이터가 충분하지 않다면 시뮬레이션 데이터로 연습합니다.

### 단계 2: 핵심 지표 대시보드 만들기

Claude에게 요청합니다:

```
관리자용 분석 대시보드 페이지를 만들어줘.
표시할 지표:
- 일간/주간 활성 사용자 수
- 핵심 기능 사용 횟수
- 신규 가입자 수 추이
- 가장 인기 있는 페이지 Top 5
간단한 숫자와 표로 보여주면 돼.
```

### 단계 3: 패턴 분석 및 인사이트 도출

데이터를 보고 Claude와 함께 패턴을 분석합니다:

```
분석 데이터를 바탕으로 패턴을 찾아줘.
- 사용자 이탈이 가장 많은 지점은 어디인가?
- 핵심 기능까지 도달하는 사용자 비율은?
- 개선하면 가장 효과가 클 부분은?
```

### 단계 4: 인사이트 문서 작성

Claude에게 요청합니다:

```
분석 결과를 my-analytics-insights.md 파일로 정리해줘.
포함할 내용:
- 현재 핵심 지표 현황
- 발견한 패턴 3가지
- 개선 우선순위 리스트 (영향도 기준 정렬)
- 다음 2주간 실험할 항목
```
