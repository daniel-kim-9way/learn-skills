## EXPLAIN

### STAR-M 프레임워크: 프로젝트를 임팩트 있게 정리하기

면접에서 "프로젝트 소개해주세요"라고 했을 때, 대부분 "Rails로 만든 웹서비스입니다"라고 답합니다. 이건 아무 정보도 전달하지 못합니다.

**STAR-M 프레임워크**는 프로젝트를 체계적으로 설명하는 방법입니다:

- **S (Situation)** — 어떤 상황/문제가 있었는가?
- **T (Task)** — 내가 맡은 역할/목표는?
- **A (Action)** — 구체적으로 무엇을 했는가?
- **R (Result)** — 어떤 결과를 만들었는가?
- **M (Meaning)** — 이 경험이 왜 의미 있는가?

예시:
> **S**: 소상공인들이 재고 관리를 엑셀로 하느라 실수가 잦았다
> **T**: AI 기반 재고 관리 SaaS를 기획부터 배포까지 1인 개발
> **A**: Rails + Claude API로 자동 발주 추천 기능 구현, Toss Payments 결제 연동
> **R**: 베타 사용자 15명 확보, 월 평균 재고 실수 40% 감소
> **M**: 비개발자도 AI를 활용해 실제 문제를 해결할 수 있다는 것을 증명

### Before/After 숫자의 힘

"좋은 서비스를 만들었습니다"보다 "사용자의 업무 시간을 주당 3시간 줄였습니다"가 **100배 강력합니다.**

숫자로 보여줄 수 있는 것들:
- 사용자 수 (0명 → 15명)
- 해결한 문제의 규모 (엑셀 실수 40% 감소)
- 개발 기간 (6주 만에 기획~배포)
- 기술 스택 범위 (프론트 + 백엔드 + AI + 결제)

숫자가 없어도 괜찮습니다. "측정하고 개선하려 했다"는 자세 자체가 차별점입니다.

### 커리어 활용 전략

서비스 빌딩 경험은 다양한 커리어 방향에 활용됩니다:

1. **창업/사이드 프로젝트** — 이미 검증된 서비스를 성장시키기
2. **취업 포트폴리오** — "혼자 기획~배포~운영까지 한 경험" 강조
3. **프리랜서** — 실제 서비스 운영 경험이 신뢰도를 높임
4. **사내 프로젝트** — AI 활용 역량 증명으로 리더십 기회

### 이력서에 쓸 때 팁

- "했다"가 아니라 **"달성했다"**로 표현
- 기술 나열이 아니라 **문제 해결 스토리**로 구성
- 과정의 실패와 극복도 가치 있음 (성장 마인드셋)
- GitHub 링크 + 배포 URL 함께 첨부

## EXECUTE

### 단계 1: STAR-M으로 프로젝트 정리

Claude에게 요청합니다:

```
내 서비스 프로젝트를 STAR-M 프레임워크로 정리해줘.
- S: 어떤 문제를 해결하려 했는지
- T: 내 역할과 목표
- A: 구체적으로 한 일 (기획, 개발, 배포, 운영)
- R: 달성한 결과 (숫자 포함)
- M: 이 경험의 의미

내 서비스 정보:
- 서비스명: [서비스 이름]
- 주요 기능: [핵심 기능]
- 사용자: [타겟 사용자]
```

### 단계 2: Before/After 숫자 정리

Claude에게 요청합니다:

```
내 서비스 프로젝트의 Before/After를 숫자로 정리해줘.
- 개발 전 상태 (문제 상황)
- 개발 후 결과 (개선된 상태)
- 수치화할 수 있는 항목 모두 정리
- 수치화가 어려운 것은 정성적으로 표현

카테고리:
- 사용자 관련 숫자
- 기술 관련 숫자 (개발 기간, 기능 수 등)
- 비즈니스 관련 숫자 (수익, 전환율 등)
```

### 단계 3: 포트폴리오용 프로젝트 설명 작성

Claude에게 요청합니다:

```
이력서와 포트폴리오에 넣을 프로젝트 설명을 3가지 버전으로 만들어줘.
1. 한 줄 소개 (30자 이내) — 엘리베이터 피치용
2. 3줄 요약 — 이력서용
3. 상세 설명 (10줄) — 포트폴리오 페이지용

각 버전에 Before/After 숫자를 포함해줘.
```

### 단계 4: 포트폴리오 문서 완성

Claude에게 요청합니다:

```
my-portfolio.md 파일을 만들어줘.
포함할 내용:
- 프로젝트 한 줄 소개
- STAR-M 프레임워크 기반 상세 설명
- Before/After 비교표
- 사용 기술 스택과 각 기술의 선택 이유
- 스크린샷 위치 표시 (나중에 추가)
- GitHub 링크와 배포 URL
- 커리어 활용 방향 (본인이 생각하는 다음 단계)
```
