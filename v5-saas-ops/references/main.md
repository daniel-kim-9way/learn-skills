## EXPLAIN

### 수익 쌓기 (Revenue Stacking)

월 100만원은 하나의 큰 고객이 아니라, **여러 수익원을 쌓아서** 만듭니다.

수익 쌓기의 기본 구조:

| 수익원 | 예시 | 월 목표 |
|--------|------|---------|
| 기본 구독 | 월 9,900원 x 30명 | 297,000원 |
| 프리미엄 구독 | 월 29,900원 x 10명 | 299,000원 |
| 일회성 판매 | 템플릿, 가이드 등 | 200,000원 |
| 부가 서비스 | 컨설팅, 커스터마이징 | 200,000원 |

**핵심**: 한 가지 수익원에 의존하지 않고, 여러 층으로 쌓는 것입니다.

### 이탈 방지 (Churn Prevention)

SaaS에서 **이탈률(Churn Rate)**은 가장 중요한 지표입니다. 매달 10%가 이탈하면, 아무리 신규 가입을 받아도 성장하지 못합니다.

**이탈률 계산**: 월초 100명 → 월말 92명 = 이탈률 8%

이탈의 주요 원인:
1. **가치를 못 느낌** — 핵심 기능을 사용하지 않음
2. **온보딩 실패** — 처음에 사용법을 모름
3. **경쟁 서비스 등장** — 더 좋은 대안 발견
4. **가격 불만** — 비용 대비 가치를 못 느낌

이탈 방지 전략:
- **온보딩 이메일** — 가입 후 3일, 7일, 14일에 사용 가이드 발송
- **핵심 기능 유도** — "아직 이 기능을 안 써보셨네요!" 알림
- **이탈 예측** — 2주 이상 미접속 사용자에게 연락
- **해지 인터뷰** — 떠나는 이유를 묻고 개선

### 가격 최적화

가격은 "원가 + 마진"이 아니라, **고객이 느끼는 가치**로 정합니다.

가격 전략 3가지:
1. **앵커링** — 가장 비싼 플랜을 먼저 보여줘서, 중간 플랜이 합리적으로 보이게
2. **3단계 가격** — Free / Basic / Pro (대부분 중간을 선택)
3. **연간 할인** — 월 결제보다 연간 결제 시 20% 할인 (이탈 방지 효과)

### 자동화로 운영 효율 높이기

1인 SaaS 운영자가 모든 것을 수동으로 하면 24시간이 모자랍니다. 반복 작업은 자동화합니다:

- **이메일 자동화** — 가입 환영, 사용 가이드, 결제 알림
- **모니터링 자동화** — 서버 오류 시 슬랙 알림
- **고객 지원 자동화** — FAQ 챗봇, 자동 응답
- **결제 자동화** — 구독 갱신, 결제 실패 재시도

## EXECUTE

### 단계 1: 수익 모델 설계

Claude에게 요청합니다:

```
내 서비스의 수익 모델을 설계해줘.
- 현재 서비스 기능 기반으로 Free / Basic / Pro 플랜 구성
- 각 플랜의 기능 차이 (Feature Gate)
- 월 100만원 달성을 위한 목표 고객 수 계산
- 가격 앵커링이 적용된 가격표 디자인 제안
```

### 단계 2: 이탈 방지 전략 수립

Claude에게 요청합니다:

```
이탈 방지를 위한 자동 이메일 시스템을 설계해줘.
- 가입 후 Day 1, 3, 7 온보딩 이메일 내용
- 핵심 기능 미사용 사용자에게 보낼 넛지 메시지
- 2주 미접속 사용자 재참여 이메일
- Action Mailer로 구현할 수 있는 구조
```

### 단계 3: 운영 자동화 설정

Claude에게 요청합니다:

```
서비스 운영 자동화를 설정해줘.
- 에러 발생 시 알림 (예외 추적)
- 일간/주간 운영 리포트 자동 생성
- Solid Queue를 활용한 백그라운드 작업 설정
```

### 단계 4: 운영 전략 문서 작성

Claude에게 요청합니다:

```
my-ops-strategy.md 파일을 만들어줘.
포함할 내용:
- 수익 모델 (플랜별 가격과 기능)
- 월 100만원 달성 로드맵 (3개월, 6개월, 12개월)
- 이탈 방지 전략 체크리스트
- 자동화 항목 목록과 우선순위
- 주간/월간 운영 루틴
```
