## EXPLAIN

### SEO란 무엇인가?

**SEO(Search Engine Optimization, 검색엔진 최적화)**는 내 서비스가 구글이나 네이버 검색 결과에서 더 잘 보이게 만드는 작업입니다.

아무리 좋은 서비스를 만들어도, 검색했을 때 나오지 않으면 사람들이 찾아올 수 없습니다. SEO는 "가게 앞에 간판을 잘 세우는 것"과 같습니다.

### 검색엔진이 내 사이트를 이해하는 방법

검색엔진은 **크롤러(봇)**를 보내 웹사이트를 읽습니다. 이 봇이 이해하기 쉬운 사이트가 검색 결과 상위에 노출됩니다.

봇이 보는 것:
- **메타 태그** — 페이지 제목과 설명 (검색 결과에 직접 노출)
- **사이트맵** — 사이트의 전체 구조 (봇에게 "여기 이런 페이지가 있어요"라고 알려줌)
- **페이지 속도** — 느린 사이트는 검색 순위가 떨어짐
- **모바일 친화성** — 모바일에서 잘 보이는지

### 소셜 미디어 공유 최적화 (OG 태그)

카카오톡이나 페이스북에 링크를 공유하면 **미리보기 카드**가 나옵니다. 이 카드의 내용을 설정하는 것이 **OG(Open Graph) 태그**입니다.

OG 태그를 설정하지 않으면 링크를 공유해도 밋밋하게 URL만 보입니다. 예쁜 카드가 나오면 클릭률이 2~3배 높아집니다.

### 페이지 속도가 중요한 이유

구글은 "3초 안에 로딩되지 않으면 사용자의 53%가 떠난다"는 연구 결과를 발표했습니다. 속도는 사용자 경험이자 검색 순위 요소입니다.

속도 개선 기본:
- 이미지 최적화 (크기 줄이기)
- CSS/JS 압축
- 브라우저 캐싱 활용

## EXECUTE

### 단계 1: 메타 태그 설정

Claude에게 요청합니다:

```
내 Rails 서비스의 SEO 메타 태그를 설정해줘.
- 각 페이지마다 고유한 title과 description
- 기본 메타 태그 레이아웃 설정
- 한국어 서비스에 맞는 charset, language 설정
```

### 단계 2: 사이트맵 생성

Claude에게 요청합니다:

```
내 서비스의 사이트맵(sitemap.xml)을 생성해줘.
- sitemap_generator gem 사용
- 주요 페이지 모두 포함
- robots.txt에 사이트맵 위치 명시
```

### 단계 3: OG 태그로 소셜 공유 최적화

Claude에게 요청합니다:

```
소셜 미디어 공유용 OG 태그를 설정해줘.
- og:title, og:description, og:image 설정
- 카카오톡, 페이스북, 트위터에서 예쁜 미리보기 카드가 나오게
- 각 페이지마다 다른 OG 태그 설정 가능하도록
```

### 단계 4: 키워드 기초 조사

Claude에게 요청합니다:

```
내 서비스와 관련된 검색 키워드를 조사해줘.
- 서비스 관련 주요 키워드 10개 제안
- 각 키워드의 경쟁 강도 예상
- 콘텐츠에 자연스럽게 키워드를 녹이는 방법
```

### 단계 5: SEO 체크리스트 작성

Claude에게 요청합니다:

```
my-seo-checklist.md 파일을 만들어줘.
포함할 내용:
- 완료한 SEO 작업 체크리스트
- 아직 남은 SEO 작업
- 타겟 키워드 목록
- 소셜 공유 설정 상태
- 월간 SEO 점검 항목
```
