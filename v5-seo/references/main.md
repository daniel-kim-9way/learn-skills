## EXPLAIN

### SEO: "좋은 서비스인데 아무도 모르는" 문제 해결하기

좋은 서비스를 만들었습니다. 하지만 사람들이 어떻게 찾아올까요?

친구에게 직접 알려줄 수는 있지만, 그건 한계가 있습니다. **검색해서 찾아오는 사람**이 있어야 서비스가 스스로 성장합니다.

SEO(Search Engine Optimization, 검색엔진 최적화)는 구글이나 네이버에서 검색했을 때 **내 서비스가 위에 나오게** 만드는 작업입니다.

비유하자면:
- 서비스를 만든 것 = 가게를 연 것
- SEO를 안 한 것 = 간판 없이 골목 안에 숨어 있는 것
- SEO를 한 것 = 큰 길가에 눈에 띄는 간판을 세운 것

### 검색엔진이 내 사이트를 이해하는 방법

구글과 네이버는 **크롤러(봇)**라는 프로그램을 보내서 웹사이트를 읽습니다. 이 봇에게 "우리 서비스는 이런 곳이에요"라고 잘 알려주는 것이 SEO의 핵심입니다.

봇이 확인하는 것들:

| 항목 | 봇이 보는 것 | 비유 |
|------|-------------|------|
| **Meta Tags** | 페이지 제목, 설명 | 책의 표지와 목차 |
| **Sitemap** | 사이트 전체 구조 | 건물 안내도 |
| **페이지 속도** | 로딩 시간 | 문 여는 속도 |
| **모바일 대응** | 모바일에서 잘 보이는지 | 모바일 전용 입구 |
| **콘텐츠 품질** | 유용한 내용이 있는지 | 가게 안에 좋은 상품이 있는지 |

### Meta Tags: 검색 결과에서 보이는 "첫인상"

네이버에서 뭔가를 검색하면 결과 목록이 나오죠? 각 결과에는 **제목**과 **설명**이 있습니다. 이것을 정하는 게 Meta Tags입니다.

```
[검색 결과 예시]

🔍 "동네 맛집 추천"으로 검색

─────────────────────────────────────
동네 맛집 찾기 - 내 주변 숨은 맛집을 발견하세요   ← title 태그
우리 동네 진짜 맛집만 모았습니다. 실제 방문자        ← description 태그
리뷰 기반으로 매주 업데이트되는 맛집 추천 서비스.
─────────────────────────────────────

vs. Meta Tags를 설정 안 하면:

─────────────────────────────────────
MyApp                                               ← 기본값
localhost:3000                                      ← 설명 없음
─────────────────────────────────────
```

어떤 걸 클릭하고 싶으세요? 당연히 위쪽이죠. Meta Tags 하나만 잘 설정해도 클릭률이 2~3배 달라집니다.

**좋은 title 태그 작성법:**
- 핵심 키워드를 앞에 배치: "동네 맛집 찾기 - 내 주변 숨은 맛집"
- 30자 이내로 간결하게
- 각 페이지마다 다른 제목

**좋은 description 태그 작성법:**
- 서비스의 핵심 가치를 한 문장으로
- 80자 이내 (모바일에서 잘리지 않게)
- "지금 시작하세요", "무료로 체험" 같은 행동 유도 문구 포함

### OG Tags: 카카오톡에서 공유했을 때 예쁘게 나오는 법

친구에게 카카오톡으로 링크를 보내면 **미리보기 카드**가 나옵니다. 이게 OG(Open Graph) 태그입니다.

```
[OG 태그 설정 안 한 경우]
┌─────────────────────┐
│ https://myapp.com   │
│                     │
│ (텅 빈 미리보기)     │
└─────────────────────┘
→ "이게 뭐야?" 클릭 안 함

[OG 태그 잘 설정한 경우]
┌─────────────────────┐
│ 🖼️ [예쁜 대표 이미지]│
│                     │
│ 동네 맛집 찾기       │
│ 우리 동네 진짜 맛집  │
│ 만 모았습니다.       │
└─────────────────────┘
→ "오 뭔데? 눌러봐야지" 클릭!
```

**OG 태그의 종류:**

| 태그 | 역할 | 예시 |
|------|------|------|
| `og:title` | 미리보기 제목 | "동네 맛집 찾기" |
| `og:description` | 미리보기 설명 | "우리 동네 진짜 맛집만 모았습니다" |
| `og:image` | 미리보기 이미지 | 1200x630 사이즈 권장 |
| `og:url` | 페이지 주소 | "https://myapp.com" |
| `og:type` | 페이지 유형 | "website" |

**플랫폼별 특징:**

| 플랫폼 | 이미지 비율 | 특이사항 |
|--------|------------|----------|
| 카카오톡 | 1:1 또는 2:1 | 한번 캐시되면 갱신이 느림 |
| 페이스북 | 1.91:1 | Facebook Debugger로 미리보기 확인 |
| 트위터/X | 2:1 | `twitter:card` 태그 별도 필요 |
| 네이버 블로그 | 다양 | og 태그를 잘 읽어줌 |

> **꿀팁**: OG 이미지는 **1200 x 630 픽셀**로 만들면 대부분의 플랫폼에서 예쁘게 나옵니다. Canva(무료)에서 5분이면 만들 수 있습니다.

> **막혔을 때**: "카카오톡에서 OG 태그를 바꿨는데 예전 이미지가 계속 나와요." 카카오톡은 한번 읽은 OG 정보를 캐시합니다. [카카오 캐시 초기화 도구](https://developers.kakao.com/tool/clear/og)에서 URL을 입력하면 새로 읽어갑니다.

### Sitemap: 검색엔진에게 "지도" 제공하기

Sitemap(사이트맵)은 내 서비스의 모든 페이지 목록을 검색엔진에게 알려주는 파일입니다.

사이트맵이 없으면 검색엔진 봇이 링크를 하나하나 따라가면서 페이지를 발견해야 합니다. 시간이 오래 걸리고, 일부 페이지는 아예 못 찾을 수 있습니다.

사이트맵이 있으면 "우리 서비스에는 이런 페이지들이 있어요, 전부 읽어가세요!"라고 한 번에 알려줄 수 있습니다.

```xml
<!-- sitemap.xml 예시 -->
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://myapp.com/</loc>           <!-- 홈페이지 -->
    <changefreq>daily</changefreq>
  </url>
  <url>
    <loc>https://myapp.com/about</loc>      <!-- 소개 페이지 -->
    <changefreq>monthly</changefreq>
  </url>
  <url>
    <loc>https://myapp.com/restaurants</loc> <!-- 맛집 목록 -->
    <changefreq>daily</changefreq>
  </url>
</urlset>
```

### 네이버/구글 검색 등록: 존재를 알리기

사이트맵을 만들었으면 검색엔진에 **직접 알려줘야** 합니다.

**구글 Search Console:**
1. https://search.google.com/search-console 접속
2. 내 도메인 등록 (HTML 파일 업로드로 소유권 확인)
3. 사이트맵 URL 제출 (https://myapp.com/sitemap.xml)
4. 며칠 후부터 구글에 노출되기 시작

**네이버 Search Advisor:**
1. https://searchadvisor.naver.com 접속
2. 내 도메인 등록 (HTML 태그로 소유권 확인)
3. 사이트맵 URL 제출
4. RSS 피드도 등록하면 더 빠르게 반영

> **꿀팁**: 구글은 등록 후 1~2주, 네이버는 2~4주 정도 걸립니다. "등록했는데 왜 안 나와요?"라고 당황하지 마세요. 기다리면 됩니다.

### 페이지 속도: 느리면 사람도 봇도 떠남

구글은 "페이지 로딩이 3초를 넘으면 사용자의 53%가 이탈한다"고 발표했습니다. 속도는 사용자 경험이자 검색 순위 요소입니다.

**속도 확인 방법:**
- https://pagespeed.web.dev 에서 내 URL 입력
- 모바일/데스크톱 점수 확인

**빠르게 할 수 있는 속도 개선:**

| 개선 항목 | 효과 | 난이도 |
|-----------|------|--------|
| 이미지 압축 (WebP 포맷) | 로딩 40~60% 감소 | 쉬움 |
| CSS/JS 압축 | 파일 크기 20~30% 감소 | 쉬움 |
| 브라우저 캐싱 설정 | 재방문 시 즉시 로딩 | 보통 |
| 불필요한 JS 제거 | 초기 로딩 개선 | 보통 |

> **막혔을 때**: "PageSpeed 점수가 30점이에요..." 점수에 너무 집착하지 마세요. 초기 서비스에서는 **50점 이상**이면 충분합니다. 80점 이상을 만들려고 시간을 쏟는 것보다, 콘텐츠와 기능 개선이 더 중요합니다.

- OG 태그: Open Graph 태그는 페이스북이 만든 표준으로, 링크를 공유할 때 제목, 설명, 이미지를 지정합니다. 카카오톡, 페이스북, 트위터 등 대부분의 플랫폼이 이 태그를 읽어 미리보기 카드를 만듭니다. 설정하지 않으면 URL만 덩그러니 표시되어 클릭률이 급감합니다.
- 검색엔진 등록: 먼저 사이트맵(sitemap.xml)을 만들고, 구글 Search Console이나 네이버 Search Advisor에 내 사이트를 등록한 뒤, 사이트맵 URL을 제출합니다. SEO는 광고와 다르게 비용이 들지 않습니다. 키워드를 부자연스럽게 반복하는 것은 "키워드 스터핑"이라고 하며, 오히려 검색 순위 하락의 원인이 됩니다.
- OG 이미지 사이즈: 1200 x 630 픽셀(1.91:1 비율)이 Facebook, 카카오톡, 트위터 등 대부분의 플랫폼에서 잘 보이는 표준 사이즈입니다. Canva 같은 무료 디자인 도구에서 "소셜 미디어 이미지" 템플릿을 선택하면 이 사이즈로 자동 설정됩니다.
