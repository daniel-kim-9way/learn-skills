# V6: Clarify — 모호한 요청을 명확하게

## 학습 목표
- 모호한 프롬프트와 명확한 프롬프트의 차이를 직접 비교한다
- Context/Constraint/Criteria 3C 패턴으로 프롬프트를 구조화할 수 있다
- 자신의 서비스에 맞는 프롬프트 라이브러리를 만든다

## 사전 조건
- C3 BUILD 완료 (Claude Code 사용 경험)
- 코드 에디터 + Claude Code 확장 프로그램 설치

## 대화형 학습 프로토콜

이 스킬은 대화형 코칭 방식을 따릅니다. Before/After 비교를 통해 직접 체감합니다.

### 진행 방식
1. references/main.md를 읽고 대화 흐름에 따라 진행한다
2. Bad prompt vs Good prompt 비교 체험 후 3C 패턴을 설명한다
3. 학생이 자신의 서비스에 적용해 연습한다
4. Before/After를 비교하고 프롬프트 라이브러리를 저장한다
