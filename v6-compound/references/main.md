## EXPLAIN

### 학습 복리란? — 매일 1%씩 성장하면 1년 후 37배

은행 복리를 아시나요?

- **단리**: 원금 100만원, 연 10% → 10년 후 200만원
- **복리**: 원금 100만원, 연 10% → 10년 후 259만원

처음 몇 년은 차이가 미미합니다. 1년 차에 단리 110만원 vs 복리 110만원 — 차이가 없죠. 하지만 시간이 지날수록 격차가 **폭발적으로** 벌어집니다.

이걸 학습에 적용하면:

```
매일 1% 성장 (복리) = 1.01^365 = 37.78배
매일 1% 감소 (퇴보) = 0.99^365 = 0.03배 (거의 0)
```

**매일 1%씩 성장하면 1년 후 37배가 됩니다.**

하지만 핵심은 "열심히 하면 된다"가 아닙니다. **어떻게** 쌓느냐가 중요합니다.

---

### 단리 학습 vs 복리 학습

**단리 학습**: 매번 따로따로
```
월요일: "React 상태관리 배웠다" → 노트에 적음 → 까먹음
화요일: "SQL 조인 배웠다" → 북마크에 저장 → 안 봄
수요일: "React 상태관리가 뭐였더라..." → 다시 검색
```

각 지식이 **독립적**으로 존재합니다. 서로 연결이 없으니 잊어버리기 쉽고, 잊으면 처음부터 다시 배워야 합니다.

**복리 학습**: 연결하며 축적
```
월요일: "React 상태관리 배웠다" 
  → "이게 내 서비스의 장바구니 기능과 연결되겠네"
  → CLAUDE.md에 기록: "장바구니 상태 = React useState로 관리"

화요일: "SQL 조인 배웠다"
  → "이게 어제 배운 React와 어떻게 연결되지?"
  → "백엔드에서 조인으로 가져온 데이터를 → React 상태로 표시"
  → CLAUDE.md에 추가: "주문 목록 = SQL 조인 → API → React 상태"

수요일: 새로운 것을 배우면 기존 지식 위에 올림
  → 학습 속도가 점점 빨라짐 (기존 지식이 발판이 되므로)
```

복리 학습의 비밀은 **연결**입니다. 새 지식이 기존 지식과 연결될 때, **두 지식 모두** 강화됩니다.

---

### 왜 대부분의 학습은 단리로 끝나는가

좋은 글을 읽었을 때 보통 이렇게 합니다:

1. 북마크에 저장한다 → 다시 안 봄
2. 노트에 복사한다 → 검색하지 않음
3. "나중에 정리해야지" → 안 함
4. 블로그에 정리한다 → 써놓고 끝, 실천 안 함

문제는 **캡처(저장)**와 **연결(기존 지식과 통합)** 사이의 간극입니다.

```
캡처(저장)    ←── 쉬움. 클릭 한 번이면 됨
    ↓
연결(통합)    ←── 어려움. 에너지가 필요함. 이 단계를 건너뜀
    ↓
활용(실천)    ←── 연결 없이는 불가능
```

저장만 하고 연결하지 않으면, 복리 효과가 0입니다. 1000개의 북마크보다 10개의 연결된 인사이트가 더 강력합니다.

---

### content-digest: 캡처 + 연결을 자동화하기

content-digest는 이 문제를 해결하는 패턴입니다. Claude를 활용해서 "연결" 단계를 자동화합니다.

**과정:**
1. 콘텐츠(URL, 텍스트, 영상 메모 등)를 Claude에게 전달
2. Claude가 **핵심 인사이트**를 추출
3. **내 기존 지식/서비스와 어떻게 연결되는지** 분석
4. **바로 실천할 수 있는 액션**으로 변환
5. 내 학습 문서에 추가

**일반 요약 vs content-digest:**

| | 일반 요약 | content-digest |
|---|---------|---------------|
| 하는 일 | 긴 글을 짧게 줄임 | 내 맥락에서 의미를 추출하고 연결 |
| 결과 | "이 글은 A, B, C를 다룹니다" | "이 글의 B 개념이 내 서비스의 X 기능에 적용 가능" |
| 가치 | 시간 절약 (읽는 시간 단축) | 지식 연결 = 복리 효과 |
| 축적 | 요약본이 쌓임 (다시 안 봄) | 인사이트 네트워크가 성장 (점점 가치 상승) |

**실제 content-digest 예시:**

입력:
```
"클린 코드에서 함수는 한 가지 일만 해야 한다"는 원칙에 대한 글을 읽었어.
```

일반 요약 결과:
```
"클린 코드의 단일 책임 원칙에 대한 글. 함수가 한 가지 일만 해야 유지보수가 쉬움."
```

content-digest 결과:
```
핵심 인사이트: 함수의 단일 책임 원칙
연결점: 우리 서비스의 OrderController#create가 현재 5가지 일을 함
       (검증 + 생성 + 결제 + 이메일 + 로깅)
       → 각각 분리하면 버그 발생 시 원인 파악이 5배 빨라짐
실천 액션: 이번 주에 OrderController#create를 5개 메서드로 분리
```

차이가 느껴지시나요? 같은 내용을 읽었지만, content-digest는 **내 상황에서 무엇을 할지**까지 연결해줍니다.

### CLAUDE.md를 "두 번째 뇌"로 활용하기

여러분의 프로젝트에 있는 `CLAUDE.md` 파일은 단순한 설정 파일이 아닙니다. **Claude가 매 세션마다 읽는 기억 장치**입니다.

여기에 학습한 내용을 기록하면:

- 작업할 때 관련 지식을 자동으로 참고
- 시간이 지날수록 Claude가 여러분을 더 잘 이해

**CLAUDE.md를 학습 허브로 만드는 구조:**

이 문서가 쌓이면 Claude는:
- "이전에 N+1 해결 경험이 있으시니, 비슷한 패턴이 여기도 있습니다" 라고 연결해줌
- "지난번에 Tailwind으로 하셨으니 이번에도 같은 방식으로 하겠습니다" 라고 일관성 유지
- "배우고 계신 SQL 최적화와 관련된 개선 기회가 이 코드에 있습니다" 라고 학습 기회 제안

### 학습 캡처 시스템의 전체 구조

효과적인 학습 캡처 시스템은 3단계로 구성됩니다:

**1단계: 인박스 (수집)** — 좋은 콘텐츠를 발견하면 일단 모아두는 곳

형식은 자유입니다. URL 목록, 메모, 스크린샷 등. 핵심은 **빠르게 캡처**하는 것.

```
## 인박스 (처리 전)
- [ ] https://example.com/clean-code-tips (2026-04-01 발견)
- [ ] 동료가 알려준 SQL 최적화 팁: "인덱스는 WHERE절 기준으로"
- [ ] YouTube: "Rails 8 Turbo Streams 완벽 가이드" 핵심 메모
```

**2단계: 다이제스트 (처리)** — 주 1~2회, 모아둔 콘텐츠를 content-digest로 처리

이 단계가 **가장 중요하고, 가장 많이 건너뛰는** 단계입니다. Claude로 자동화하는 이유가 여기에 있습니다.

```
인박스의 3개 콘텐츠를 content-digest 처리:
→ 핵심 인사이트 추출
→ 내 프로젝트/목표와 연결
→ 실천 액션으로 변환
```

**3단계: 컴파운드 문서 (축적)** — 처리된 인사이트가 쌓이는 살아있는 문서

시간이 지날수록 풍부해지고, Claude가 작업 시 참고합니다.

```
[인박스]        →    [content-digest 처리]    →    [컴파운드 문서]
 수집 (쉬움)          연결 + 변환 (핵심)            축적 + 활용
 5초/건              5분/건 (Claude 도움)          자동 축적
```

---

### 자동 요약 습관 만들기 — 하루 5분의 마법

습관은 **트리거(언제) + 행동(무엇을) + 보상(결과)**으로 만들어집니다.

**추천 루틴:**

**매일 (2분):**
- 트리거: 작업 종료 전
- 행동: Claude에게 "오늘 배운 것 한 줄 요약해서 CLAUDE.md에 추가해줘"
- 보상: 내일 아침 Claude가 이어서 작업

**매주 (15분, 주 1회):**
- 트리거: 금요일 퇴근 전 / 일요일 저녁
- 행동: 인박스의 콘텐츠를 content-digest로 처리
- 보상: 다음 주에 더 빠르게 학습

**매월 (20분, 월 1회):**
- 트리거: 월말
- 행동: 컴파운드 문서 전체 리뷰 — 패턴 발견, 불필요한 것 정리
- 보상: 한 달간의 성장을 시각적으로 확인

**가장 중요한 규칙: 완벽하지 않아도 됩니다.**

하루에 "오늘 배운 것: includes로 N+1 해결" 한 줄이면 충분합니다. 이 한 줄이 365일 쌓이면 365개의 인사이트가 됩니다. 그중 10%만 실제로 연결되어도 36개의 실전 노하우가 생깁니다.


판단하지 않습니다. 대부분의 사람이 같은 상황입니다. 중요한 건 지금부터입니다.

결과를 읽고 확인합니다:
- "이건 몰랐네" 하는 순간이 있었는가?
- "이렇게 연결되는구나" 하는 순간이 있었는가?
- 실천 액션이 구체적이고 실행 가능한가?

이것이 일반 요약과 content-digest의 차이입니다.

정답:
- 1번: 새 지식을 기존 지식과 **연결**하는 것. 단순히 많이 읽는 것(단리)이 아니라, 읽은 것을 기존에 알던 것과 연결할 때(복리) 학습이 가속됨. 연결 없이 100개를 배우면 100개가 따로 놀지만, 연결하면 서로를 강화해서 200이 아니라 300, 500의 효과가 됨.
- 2번: 일반 요약은 콘텐츠를 짧게 줄이는 것이고, content-digest는 **내 맥락**(서비스, 목표, 기존 지식)에서 콘텐츠의 의미를 추출하고 연결점을 찾는 것. 요약은 시간을 절약하지만 지식 성장에 기여 안 함. digest는 지식을 연결해서 복리 효과를 만듦.
- 3번: **다이제스트(처리)** 단계. 콘텐츠를 수집(인박스)하는 것은 클릭 한 번으로 쉽고, 축적(컴파운드)은 처리 결과만 추가하면 됨. 하지만 수집한 콘텐츠를 내 맥락에서 처리하는 작업은 에너지가 필요해서 자꾸 미루게 됨. 이 단계를 Claude로 자동화하는 것이 핵심.
- 4번: 일반 노트앱은 기록만 하고 끝이지만, CLAUDE.md에 기록하면 Claude가 매 세션마다 읽어서 작업에 자동으로 활용함. "지난번에 이런 걸 배우셨는데 여기에 적용할 수 있습니다"처럼 능동적으로 연결해줌. 노트앱은 내가 찾아야 하지만, CLAUDE.md는 Claude가 알아서 참고.
- 5번: 완벽을 추구하면 시작하지 못하거나 며칠 하다 포기하기 때문. "오늘 배운 것 한 줄"이면 충분. 이 한 줄이 365일 쌓이면 365개. 하루 10분짜리 완벽한 기록을 시도하면 3일 만에 "오늘은 바빠서..."가 되고, 그 뒤로 안 하게 됨. 작고 불완전해도 매일 하는 게 복리의 핵심.
