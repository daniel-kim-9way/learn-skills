## EXPLAIN

### 학습 복리란?

은행 복리를 아시나요?

- **단리**: 원금 100만원, 연 10% → 10년 후 200만원
- **복리**: 원금 100만원, 연 10% → 10년 후 259만원

처음 몇 년은 차이가 미미합니다. 하지만 시간이 지날수록 격차가 폭발적으로 벌어집니다.

학습도 똑같습니다.

**단리 학습**: 새로운 걸 배우고 → 따로 저장하고 → 잊어버리고 → 다시 배움
**복리 학습**: 새로운 걸 배우고 → 기존 지식과 연결하고 → 다음 학습이 더 빨라지고 → 또 연결

복리 학습의 비밀은 **연결**입니다. 새 지식이 기존 지식과 연결될 때, 두 지식 모두 강화됩니다.

### 왜 대부분의 학습은 단리로 끝나는가

좋은 글을 읽었을 때 보통 이렇게 합니다:
1. 북마크에 저장한다 → 다시 안 봄
2. 노트에 복사한다 → 검색 안 함
3. "나중에 정리해야지" → 안 함

문제는 **캡처(저장)**와 **연결(기존 지식과 통합)** 사이의 간극입니다. 저장만 하고 연결하지 않으면, 복리 효과가 0입니다.

### content-digest: 캡처 + 연결을 자동화

content-digest는 이 문제를 해결하는 패턴입니다:

1. 콘텐츠(URL, 텍스트, 영상 메모)를 Claude에게 전달
2. Claude가 **핵심 인사이트**를 추출
3. **내 기존 지식/서비스와 어떻게 연결되는지** 분석
4. **실천 가능한 액션**으로 변환
5. 내 학습 문서에 추가

일반 요약과의 차이:
| | 일반 요약 | content-digest |
|---|---------|---------------|
| 하는 일 | 긴 글을 짧게 줄임 | 내 맥락에서 의미를 추출 |
| 결과 | "이 글은 A, B, C를 다룹니다" | "이 글의 B 개념이 내 서비스의 X 기능에 적용 가능" |
| 가치 | 시간 절약 | 지식 연결 = 복리 효과 |

### 학습 캡처 시스템의 구조

효과적인 학습 캡처 시스템은 3가지로 구성됩니다:

**1. 인박스 (수집)**
- 좋은 콘텐츠를 발견하면 일단 모아두는 곳
- 형식: URL 목록, 메모, 스크린샷

**2. 다이제스트 (처리)**
- 주 1~2회, 모아둔 콘텐츠를 content-digest로 처리
- Claude가 핵심을 추출하고 연결점을 찾아줌

**3. 컴파운드 문서 (축적)**
- 처리된 인사이트가 쌓이는 살아있는 문서
- 시간이 지날수록 풍부해지고, Claude가 작업 시 참고

```
[인박스] → [content-digest 처리] → [컴파운드 문서]
 수집          연결 + 변환           축적 + 활용
```

## EXECUTE

### Step 1: 현재 학습 방식 점검 (3분)

잠시 돌아봅니다:
- 최근 한 달간 북마크에 저장한 글이 몇 개인가?
- 그중 다시 열어본 것은 몇 개인가?
- 읽은 내용이 실제 작업에 영향을 준 적이 있는가?

솔직한 답을 Claude에게 공유합니다. 판단하지 않습니다 — 대부분의 사람이 같은 상황입니다.

### Step 2: content-digest 체험 (10분)

최근 읽은 좋은 글이나 영상을 하나 떠올립니다. Claude에게 요청합니다:

```
content-digest를 실행해줘.

콘텐츠:
[URL 또는 핵심 내용 붙여넣기]

내 컨텍스트:
- 만들고 있는 서비스: [서비스 설명]
- 최근 관심사: [관심 주제]
- 현재 수준: [초보/중급/고급]

추출해줘:
1. 핵심 인사이트 3개 (원문의 키포인트)
2. 내 서비스에 적용할 수 있는 점 (연결)
3. 바로 실천할 수 있는 액션 1개 (변환)
```

결과를 읽고, "이건 몰랐네" 또는 "이렇게 연결되네"라는 순간이 있었는지 확인합니다.

### Step 3: 학습 캡처 시스템 구축 (7분)

나만의 시스템을 만듭니다. Claude에게 요청합니다:

```
나만의 학습 캡처 시스템을 만들어줘.

포함할 것:
1. 인박스 템플릿 (콘텐츠 수집용)
2. content-digest 실행 프롬프트 템플릿
3. 컴파운드 문서 초기 구조

내 상황:
- 관심 분야: [분야 1], [분야 2]
- 학습 목표: [목표]
- 주간 학습 시간: 약 [시간]

결과를 my-learning-system.md로 저장해줘.
```

### Step 4: 첫 다이제스트 기록 (5분)

Step 2에서 만든 content-digest 결과를 컴파운드 문서에 추가합니다:

```
방금 만든 content-digest 결과를 my-learning-system.md의 컴파운드 섹션에 추가해줘.
날짜, 출처, 인사이트, 연결점, 액션을 포함해서.
```

이것이 첫 번째 복리 이자입니다. 매주 1~2개씩 추가하면 한 달 후, 석 달 후에 놀라운 차이를 경험하게 됩니다.

## QUIZ

다음 질문에 답해봅니다:

1. 학습 복리에서 "복리 효과"가 생기는 핵심 조건은 무엇인가요?

2. content-digest와 일반 요약의 가장 큰 차이는?

3. 학습 캡처 시스템의 3단계(인박스, 다이제스트, 컴파운드)에서 사람들이 가장 많이 빠뜨리는 단계는 어디이고, 왜 그런가요?

정답:
- 1번: 새 지식을 기존 지식과 "연결"하는 것. 단순히 많이 읽는 것(단리)이 아니라, 읽은 것을 기존에 알던 것과 연결할 때(복리) 학습이 가속됨.
- 2번: 일반 요약은 콘텐츠를 짧게 줄이는 것이고, content-digest는 내 맥락(서비스, 목표, 기존 지식)에서 콘텐츠의 의미를 추출하고 연결점을 찾는 것. 요약은 시간을 절약하지만, digest는 지식을 연결해서 복리 효과를 만듦.
- 3번: 다이제스트(처리) 단계. 콘텐츠를 수집(인박스)하는 것은 쉽고, 축적(컴파운드)은 처리 결과만 추가하면 됨. 하지만 수집한 콘텐츠를 내 맥락에서 처리하는 작업은 에너지가 필요해서 자꾸 미루게 됨. 이 단계를 Claude로 자동화하는 것이 핵심.
