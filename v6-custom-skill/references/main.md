## EXPLAIN

### 스킬이란?

Claude Code에서 스킬은 **재사용 가능한 지시서**입니다.

매번 같은 설명을 반복하는 대신, 한 번 스킬로 만들어두면 `/스킬이름`으로 바로 실행할 수 있습니다.

일상에서 비유하면:
- **스킬 없이**: 매번 "커피는 에스프레소 2샷, 우유 150ml, 60도로 데워서, 시나몬 뿌려줘"
- **스킬 있으면**: "내 카페라떼 만들어줘" (레시피가 저장되어 있으니까)

### Template + STUB 패턴

스킬은 두 부분으로 구성됩니다:

**Template (뼈대)**: 변하지 않는 구조
```markdown
---
name: STUB_NAME
description: "STUB_DESCRIPTION"
---

# STUB_TITLE

## 이 스킬이 하는 일
STUB_PURPOSE

## 실행 단계
1. STUB_STEP_1
2. STUB_STEP_2
3. STUB_STEP_3

## 출력 형식
STUB_OUTPUT_FORMAT
```

**STUB (채울 부분)**: 상황에 맞게 바꾸는 부분
- `STUB_NAME`: 스킬 이름 (예: `daily-report`)
- `STUB_DESCRIPTION`: 한 줄 설명
- `STUB_PURPOSE`: 이 스킬이 해결하는 문제
- `STUB_STEP_1~3`: 구체적인 실행 단계
- `STUB_OUTPUT_FORMAT`: 결과물 형식

### 좋은 스킬의 3가지 조건

1. **반복성**: 일주일에 2회 이상 하는 작업인가?
2. **구조성**: 매번 비슷한 패턴으로 진행되는가?
3. **구체성**: 실행 단계가 명확한가?

예시 — 좋은 스킬 후보:
- 매일 아침 PR 리뷰 요약 만들기
- 새 기능 개발 시 보일러플레이트 생성
- 코드 리뷰 체크리스트 작성
- 주간 회의록 정리

예시 — 스킬이 아닌 것:
- "좋은 코드 짜줘" (너무 모호)
- 한 번만 하는 작업 (반복성 없음)
- 매번 완전히 다른 판단이 필요한 작업 (구조성 없음)

### 실전 스킬 예시: daily-standup

```markdown
---
name: daily-standup
description: "매일 아침 스탠드업 준비. 어제 작업 요약, 오늘 할 일 정리."
---

# Daily Standup 준비

## 이 스킬이 하는 일
git log와 이슈 목록을 기반으로 스탠드업 내용을 자동 정리합니다.

## 실행 단계
1. 어제의 git commit 목록을 가져와서 "완료한 일"로 요약
2. 현재 브랜치에서 진행 중인 작업을 "진행 중"으로 정리
3. GitHub Issues에서 나에게 할당된 이슈를 "오늘 할 일"로 정리
4. 막힌 부분이 있으면 "블로커"로 표시

## 출력 형식
### 어제 완료
- [커밋 기반 요약]

### 오늘 예정
- [이슈 기반 목록]

### 블로커
- [있으면 작성, 없으면 "없음"]
```

## EXECUTE

### Step 1: 반복 작업 발견하기 (5분)

지난 일주일을 돌아봅니다. Claude에게 다음과 같이 요청합니다:

```
내가 지난 일주일간 Claude에게 반복적으로 요청한 것 같은 패턴을 찾아줘.
채팅 히스토리를 보고 2회 이상 비슷하게 요청한 것을 알려줘.

없으면, 아래 후보 중에서 내 상황에 맞는 걸 추천해줘:
- 코드 리뷰 체크리스트 생성
- 커밋 메시지 작성
- 버그 리포트 템플릿 작성
- 회의록 정리
- 데일리 작업 요약
```

### Step 2: STUB 함께 채우기 (15분)

선택한 반복 작업을 스킬로 변환합니다. Claude에게 요청합니다:

```
[선택한 작업]을 스킬로 만들고 싶어.

아래 Template의 STUB 부분을 함께 채워줘.
하나씩 질문하면서 진행해줘 (한 번에 다 완성하지 말고).

Template:
---
name: STUB_NAME
description: "STUB_DESCRIPTION"
---

# STUB_TITLE

## 이 스킬이 하는 일
STUB_PURPOSE

## 실행 단계
1. STUB_STEP_1
2. STUB_STEP_2
3. STUB_STEP_3

## 출력 형식
STUB_OUTPUT_FORMAT
```

Claude와 대화하며 각 STUB을 채웁니다. 중요한 것은 **Claude가 물어보고, 내가 답하는** 과정입니다.

### Step 3: 스킬 저장 및 테스트 (10분)

완성된 스킬을 저장합니다:

```
완성된 스킬을 .claude/skills/[스킬이름]/SKILL.md로 저장해줘.
```

저장 후 바로 테스트합니다:

```
/[스킬이름]
```

결과를 확인하고 부족한 부분을 메모합니다.

### Step 4: 반복 개선 (10분)

테스트 결과를 바탕으로 스킬을 개선합니다:

```
방금 스킬을 테스트했는데 이런 부분이 아쉬워:
- [개선점 1]
- [개선점 2]

스킬 파일을 수정해줘.
```

2~3회 반복하면 실용적인 스킬이 완성됩니다. 완벽할 필요 없습니다. **쓰면서 개선**하는 것이 핵심입니다.

## QUIZ

다음 질문에 답해봅니다:

1. Template + STUB 패턴에서 "Template"과 "STUB"의 역할 차이는 무엇인가요?

2. 다음 중 스킬로 만들기 적합한 작업은? (모두 선택)
   - (a) 새 프로젝트 셋업 가이드
   - (b) 오늘 기분에 따라 음악 추천
   - (c) PR 생성 시 체크리스트 자동 작성
   - (d) 한 번만 하는 데이터 마이그레이션

3. 스킬을 만든 후 바로 실전에 투입하기 전에 꼭 해야 할 일은?

정답:
- 1번: Template는 모든 스킬에 공통인 변하지 않는 구조(뼈대), STUB는 각 스킬의 목적에 맞게 채워 넣는 구체적인 내용(살). Template가 있으면 새 스킬을 빈 화면에서 시작하지 않아도 됨.
- 2번: (a)와 (c). (a)는 프로젝트마다 반복되고 구조가 비슷함. (c)는 PR마다 반복되는 체크리스트. (b)는 구조성이 없고, (d)는 반복성이 없음.
- 3번: 테스트 실행. 실제로 한 번 돌려보고 결과를 확인한 뒤 부족한 부분을 개선해야 함. 스킬은 한 번에 완성되지 않고 "작성 → 테스트 → 개선" 사이클로 다듬어짐.
