## EXPLAIN

### MCP란? — 레스토랑 배달 비유

Claude Code는 뛰어난 셰프입니다. 하지만 셰프가 아무리 실력이 좋아도 **재료가 없으면** 요리를 못 합니다.

MCP(Model Context Protocol)는 **배달 서비스**입니다.

- **셰프** = Claude Code (요리 실력은 뛰어남)
- **식재료 공급업체** = 외부 도구 (Slack, Notion, GitHub, DB 등)
- **배달 서비스** = MCP (식재료를 셰프에게 가져다주는 표준 배달 시스템)

MCP가 없으면: 셰프에게 "어제 팀 회의에서 뭐 나왔어?"라고 물어도, 셰프는 Slack을 못 봐서 대답 못 합니다.

MCP가 있으면: 셰프가 배달 서비스를 통해 Slack에서 어제 회의 내용을 가져와서 "이런 논의가 있었고, 이렇게 정리할 수 있습니다"라고 답합니다.

### MCP의 핵심 구조

```
[Claude Code] ←→ [MCP 프로토콜] ←→ [외부 도구]
   (셰프)         (배달 시스템)       (식재료 공급)
```

기술적으로는:
- **MCP Server**: 외부 도구와 통신하는 중개 프로그램
- **MCP Client**: Claude Code 내부에서 MCP Server를 호출하는 부분
- **Tools**: MCP Server가 제공하는 기능들 (메시지 읽기, 파일 생성 등)

하지만 여러분이 기억할 건 이것뿐입니다: **MCP = Claude가 외부 도구를 사용할 수 있게 해주는 연결 통로**

### 연결 가능한 도구 예시

| 도구 | 할 수 있는 일 | 활용 예시 |
|------|-------------|----------|
| Slack | 메시지 읽기/보내기 | "어제 #개발 채널 요약해줘" |
| Notion | 페이지 읽기/작성 | "회의록을 Notion에 정리해줘" |
| GitHub | 이슈/PR 관리 | "이 버그 이슈 만들어줘" |
| Google Calendar | 일정 조회 | "오늘 일정 알려줘" |
| 데이터베이스 | 데이터 조회 | "이번 달 가입자 수 알려줘" |

### 언제 MCP가 유용하고, 언제 불필요한가?

**MCP가 유용한 경우:**
- Claude가 외부 데이터를 읽어야 할 때
- 반복 작업을 자동화하고 싶을 때
- 여러 도구를 연결해서 워크플로우를 만들고 싶을 때

**MCP가 불필요한 경우:**
- 코드 작성, 리팩토링 등 로컬 작업
- 파일 시스템 내에서 해결되는 작업 (이미 기본 제공)
- 단순 질문/답변 대화

## EXECUTE

### Step 1: 연결할 도구 선택 (5분)

먼저 자신의 작업 흐름을 생각합니다:
- 매일 어떤 도구를 쓰는가?
- Claude가 접근할 수 있으면 편해질 도구가 있는가?

추천 순서 (설정 난이도 기준):
1. **GitHub** — 가장 설정이 쉬움 (이미 gh CLI가 있다면)
2. **Slack** — 팀 협업에 바로 활용 가능
3. **Notion** — 문서 자동화에 유용

### Step 2: MCP Server 설정 (15분)

Claude Code에서 다음과 같이 요청합니다:

```
[선택한 도구] MCP를 설정하고 싶어.

내 환경:
- OS: [macOS/Windows/Linux]
- Claude Code: 코드 에디터 확장 프로그램 사용 중

다음을 안내해줘:
1. 필요한 MCP Server 패키지
2. settings.json에 추가할 설정
3. 인증/토큰 설정 방법
4. 연결 확인하는 방법
```

Claude의 안내에 따라 하나씩 설정합니다. 막히는 부분이 있으면 바로 Claude에게 물어봅니다.

### Step 3: 연결 테스트 (5분)

설정이 끝나면 간단한 명령으로 테스트합니다:

```
[도구 이름]에서 최근 데이터를 읽어와줘.
```

예시:
- GitHub: "내 최근 이슈 목록을 보여줘"
- Slack: "최근 메시지를 읽어와줘"
- Notion: "내 페이지 목록을 가져와줘"

동작하면 성공입니다.

### Step 4: 활용 아이디어 정리 & 저장 (5분)

Claude에게 요청합니다:

```
방금 연결한 [도구 이름] MCP를 활용해서 내 작업을 자동화할 수 있는 아이디어를 5가지 제안해줘.

내 서비스/업무 컨텍스트:
[간단한 서비스 또는 업무 설명]

결과를 my-mcp-setup.md로 저장해줘.
포함할 내용:
- 연결한 도구와 설정 요약
- 활용 아이디어 5가지
- 앞으로 추가 연결하고 싶은 도구
```

## QUIZ

다음 질문에 답해봅니다:

1. MCP를 레스토랑 비유로 설명할 때, "MCP Server"의 역할은 무엇인가요?

2. 다음 상황에서 MCP가 필요한 경우와 불필요한 경우를 구분해보세요:
   - (a) "이 함수를 리팩토링해줘"
   - (b) "Slack #bugs 채널에서 오늘 올라온 버그 목록을 정리해줘"
   - (c) "README.md를 업데이트해줘"
   - (d) "Notion의 스프린트 보드에서 이번 주 할 일을 가져와줘"

3. MCP 연결 후 가장 먼저 해야 할 일은 무엇인가요?

정답:
- 1번: MCP Server는 외부 도구와 Claude Code 사이를 중개하는 "배달 서비스" 역할. 외부 도구의 데이터를 Claude가 이해할 수 있는 형태로 가져다줌.
- 2번: (a) 불필요 — 로컬 파일 작업. (b) 필요 — Slack 데이터 접근. (c) 불필요 — 로컬 파일 작업. (d) 필요 — Notion 데이터 접근.
- 3번: 간단한 읽기 명령으로 연결이 정상 동작하는지 테스트해야 함. 동작 확인 없이 복잡한 작업을 시도하면 설정 문제와 로직 문제를 구분하기 어려움.
