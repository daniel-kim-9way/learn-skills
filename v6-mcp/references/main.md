## EXPLAIN

### MCP란? — "배달 서비스" 비유

Claude Code는 뛰어난 셰프입니다. 어떤 요리든 만들 수 있는 실력이 있습니다. 하지만 셰프가 아무리 실력이 좋아도 **재료가 부엌에 없으면** 요리를 못 합니다.

MCP(Model Context Protocol)는 **배달 서비스**입니다.

```
🏪 식재료 공급업체들          🚚 배달 서비스       👨‍🍳 셰프
┌─────────────┐
│ Slack       │
│ Notion      │──────→  MCP  ──────→  Claude Code
│ Google Sheets│       (배달 시스템)     (요리 시작!)
│ GitHub      │
└─────────────┘
```

- **셰프** = Claude Code (요리 실력은 이미 충분)
- **식재료 공급업체** = 외부 도구 (Slack, Notion, GitHub, DB 등)
- **배달 서비스** = MCP (식재료를 셰프에게 가져다주는 표준 배달 시스템)

**MCP가 없을 때:**
여러분: "어제 팀 회의에서 나온 할 일 정리해줘"
Claude: "죄송합니다. 저는 Slack에 접근할 수 없어서 회의 내용을 모릅니다."

**MCP가 있을 때:**
여러분: "어제 팀 회의에서 나온 할 일 정리해줘"
Claude: (Slack MCP를 통해 #general 채널 확인) "어제 회의에서 나온 할 일 3가지입니다: 1) 랜딩 페이지 수정... 2) 결제 테스트... 3) 사용자 피드백 정리..."

---

### MCP의 핵심 구조 — 기술적으로 이해하기

어렵지 않습니다. 3가지만 알면 됩니다.

```
[Claude Code] ←→ [MCP Server] ←→ [외부 서비스]
   (클라이언트)      (중개자)         (데이터 소스)
```

**MCP Client (클라이언트)**: Claude Code 안에 내장되어 있습니다. 여러분이 설정할 필요 없습니다.

**MCP Server (서버)**: 외부 서비스와 Claude Code 사이에서 통역사 역할을 합니다. 각 서비스마다 전용 MCP Server가 있습니다. 이것을 설치하고 설정하는 게 여러분이 할 일입니다.

**Tools (도구)**: MCP Server가 제공하는 구체적인 기능들입니다. 예를 들어 Slack MCP Server는 "메시지 읽기", "메시지 보내기", "채널 목록 조회" 같은 도구를 제공합니다.

배달 비유로 다시 설명하면:
- MCP Client = 셰프가 전화기를 들고 주문하는 행위
- MCP Server = 배달 앱 (배민, 쿠팡이츠)
- Tools = 배달 앱의 메뉴들 (주문, 취소, 배달 추적 등)

---

### 연결 가능한 도구 목록

현재 MCP로 연결할 수 있는 주요 도구들입니다:

| 도구 | 할 수 있는 일 | 실전 활용 예시 | 설정 난이도 |
|------|-------------|--------------|-----------|
| **Slack** | 메시지 읽기/보내기, 채널 관리 | "어제 #bugs 채널 내용 정리해줘" | 중간 |
| **Notion** | 페이지 읽기/작성, DB 조회 | "스프린트 보드에서 이번 주 할 일 가져와줘" | 중간 |
| **Google Sheets** | 시트 읽기/쓰기, 수식 적용 | "매출 데이터 시트에서 이번 달 요약해줘" | 중간 |
| **GitHub** | 이슈/PR 관리, 코드 검색 | "이 버그 이슈 만들고 라벨 붙여줘" | 쉬움 |
| **Google Calendar** | 일정 조회/생성 | "오늘 회의 일정 알려줘" | 중간 |
| **PostgreSQL/MySQL** | DB 직접 조회 | "이번 달 신규 가입자 수 알려줘" | 어려움 |
| **Figma** | 디자인 파일 조회 | "이 디자인 컴포넌트 스펙 읽어줘" | 어려움 |

**처음이라면 GitHub 또는 Slack부터** 시작하세요. 설정이 비교적 간단하고, 바로 효과를 느낄 수 있습니다.

---

### MCP 연결 전체 과정 미리보기

1단계에서 4단계까지, 전체 흐름을 먼저 그림으로 봅니다:

```
1. 도구 선택           2. MCP Server 설치       3. 인증 설정          4. 연결 확인
"Slack 연결할래"  →   npm install ...     →   토큰 발급/설정   →   "테스트 읽기"
                      settings.json 수정       .env에 저장            성공!
```

복잡해 보이지만, Claude Code가 각 단계를 안내해줍니다. 여러분은 Claude의 안내를 따라 하나씩 진행하면 됩니다.

---

### 언제 MCP가 유용한가 — 판단 기준 5가지

모든 작업에 MCP가 필요한 건 아닙니다. 아래 체크리스트로 판단합니다:

**MCP가 필요한 신호 (하나라도 해당되면 고려):**

1. **외부 데이터 접근**: Claude가 Slack, Notion, Google Sheets 등의 데이터를 읽어야 할 때
2. **반복 수동 작업**: 매번 수동으로 복사-붙여넣기 하는 정보가 있을 때
3. **도구 간 연결**: "Slack 내용을 Notion에 정리" 같은 워크플로우를 자동화하고 싶을 때
4. **실시간 데이터**: 최신 상태(이슈 목록, 일정 등)를 기반으로 작업해야 할 때
5. **팀 협업 자동화**: 여러 사람이 쓰는 도구의 정보를 종합해야 할 때

**MCP가 불필요한 경우:**

- **로컬 파일 작업**: 코드 작성, 리팩토링, 파일 수정 (이미 기본 제공)
- **단순 질문/답변**: "이 코드 뭐하는 건지 설명해줘"
- **일회성 작업**: 한 번만 하면 끝나는 작업 (수동이 더 빠름)
- **보안 민감 데이터**: 접근 권한 관리가 복잡한 내부 시스템

**실전 판단 예시:**

| 상황 | MCP 필요? | 이유 |
|------|----------|------|
| "이 함수를 리팩토링해줘" | 불필요 | 로컬 파일 작업 |
| "Slack #bugs에서 오늘 버그 목록 정리해줘" | 필요 | 외부 데이터 접근 |
| "README.md를 업데이트해줘" | 불필요 | 로컬 파일 작업 |
| "Notion 스프린트 보드에서 할 일 가져와줘" | 필요 | 외부 데이터 접근 |
| "Google Sheets 매출 데이터를 분석해줘" | 필요 | 외부 데이터 접근 |
| "이 에러 메시지가 뭔 뜻이야?" | 불필요 | 단순 질문 |

---

### MCP 보안 알아두기

MCP는 외부 서비스에 접근하는 만큼, 보안을 신경 써야 합니다:

**API 토큰 관리:**
- 토큰은 `.env` 파일에 저장하고, `.gitignore`에 `.env`를 추가
- GitHub에 절대 토큰을 올리지 않기
- 팀원과 토큰을 공유할 때는 안전한 채널(1Password 등) 사용

**권한 최소화:**
- Slack MCP를 설정할 때 "모든 채널 접근"이 아니라 "특정 채널만" 설정
- GitHub MCP는 "읽기 전용"으로 시작하고, 필요할 때 "쓰기" 추가
- 불필요한 MCP는 비활성화

**Claude가 도구를 사용할 때:**
- Claude는 MCP 도구를 사용하기 전에 여러분에게 허락을 구합니다
- "Slack에서 메시지를 읽겠습니다" 같은 확인 메시지가 나옵니다
- 의도하지 않은 동작이면 거절할 수 있습니다


## EXECUTE

### Step 1: 연결할 도구 선택하기 (5분)

먼저 자신의 일상 작업 흐름을 돌아봅니다:
- 매일 어떤 도구를 여는가?
- Claude가 접근할 수 있으면 편해질 정보가 있는가?
- 반복적으로 복사-붙여넣기 하는 데이터가 있는가?

다음 중 하나를 선택합니다 (추천 순서):

1. **GitHub** — 가장 쉬움. gh CLI가 설치되어 있다면 바로 시작 가능
2. **Slack** — 팀 협업에 즉시 활용. 회의록 정리, 채널 요약에 강력
3. **Notion** — 문서 자동화에 유용. 스프린트 보드, 회의록 관리
4. **Google Sheets** — 데이터 분석, 보고서 자동화

결정이 안 되면 Claude에게 물어봅니다:

```
나는 [나의 역할/업무 설명]이야.
매일 쓰는 도구는 [도구 목록].
MCP로 처음 연결하기에 가장 효과적인 도구를 추천해줘.
```

### Step 2: MCP Server 설정하기 (15분)

선택한 도구의 MCP Server를 설치합니다. Claude에게 다음과 같이 요청합니다:

```
[선택한 도구] MCP를 설정하고 싶어.

내 환경:
- OS: [macOS/Windows/Linux]
- Claude Code: 코드 에디터 확장 프로그램 사용 중
- [선택한 도구]는 이미 사용 중

다음을 단계별로 안내해줘:
1. 필요한 MCP Server 패키지 이름과 설치 명령어
2. settings.json에 추가할 설정 (복사-붙여넣기할 수 있게)
3. API 토큰/인증 설정 방법 (스크린샷 위치 포함)
4. .env 파일에 토큰 저장하는 방법
5. 연결 확인하는 테스트 명령어

중요: 각 단계가 끝나면 다음 단계로 넘어가기 전에 
"여기까지 됐어" / "안 돼, 이런 에러가 나" 알려줄게.
```

Claude의 안내를 따라 하나씩 설정합니다. **한 단계씩** 진행하세요. 여러 단계를 한꺼번에 하면 어디서 문제가 생겼는지 찾기 어렵습니다.

**자주 만나는 문제와 해결:**

| 문제 | 원인 | 해결 |
|------|------|------|
| "MCP Server를 찾을 수 없습니다" | 패키지 미설치 또는 경로 오류 | npm list -g로 설치 확인 |
| "인증 실패" | 토큰 만료 또는 권한 부족 | 토큰 재발급, 권한 확인 |
| "연결 시간 초과" | 네트워크 문제 또는 방화벽 | VPN 확인, 프록시 설정 |
| settings.json 위치를 모르겠음 | OS마다 경로가 다름 | Claude에게 OS 알려주고 물어보기 |

### Step 3: 연결 테스트하기 (5분)

설정이 끝나면 간단한 읽기 명령으로 테스트합니다:

**GitHub 연결 시:**
```
내 GitHub 저장소에서 최근 이슈 3개를 보여줘.
```

**Slack 연결 시:**
```
Slack [채널명]에서 오늘 올라온 메시지를 읽어와줘.
```

**Notion 연결 시:**
```
내 Notion 워크스페이스의 페이지 목록을 가져와줘.
```

**Google Sheets 연결 시:**
```
[시트 이름]의 첫 번째 시트에서 데이터를 읽어와줘.
```

동작하면 성공입니다! 동작하지 않으면:

```
[도구 이름] MCP 테스트가 실패했어.
에러 메시지: [붙여넣기]
어떻게 해결해야 해?
```

### Step 4: 활용 아이디어 정리 및 저장 (5분)

연결에 성공했으면, 실제로 활용할 시나리오를 정리합니다:

```
방금 연결한 [도구 이름] MCP를 활용해서 
내 업무를 자동화할 수 있는 아이디어를 5가지 제안해줘.

내 상황:
- 서비스/업무: [설명]
- 매일 하는 작업: [반복 작업 설명]
- 가장 시간이 많이 드는 작업: [설명]

결과를 my-mcp-setup.md로 저장해줘.
포함할 내용:
- 연결한 도구와 설정 요약 (나중에 다시 설정할 때 참고)
- 활용 아이디어 5가지 (구체적인 프롬프트 예시 포함)
- 앞으로 추가 연결하고 싶은 도구와 이유
```


## QUIZ

다음 질문에 답해봅니다:

1. MCP를 배달 서비스 비유로 설명할 때, MCP Server, MCP Client, Tools 각각의 역할은?

2. 다음 상황에서 MCP가 필요한 경우와 불필요한 경우를 구분하세요:
   - (a) "이 함수를 리팩토링해줘"
   - (b) "Slack #bugs 채널에서 오늘 올라온 버그 목록을 정리해줘"
   - (c) "README.md를 업데이트해줘"
   - (d) "Notion의 스프린트 보드에서 이번 주 할 일을 가져와줘"
   - (e) "Google Sheets의 매출 데이터를 분석해줘"

3. MCP 설정 시 API 토큰을 왜 .env 파일에 저장하고 .gitignore에 추가해야 하나요?

4. MCP 연결 후 가장 먼저 해야 할 일은 무엇이고, 왜 그런가요?

5. MCP를 처음 설정한다면 GitHub, Slack, Notion 중 어느 것부터 시작하는 게 좋을까요? 이유는?

정답:
- 1번: MCP Client는 셰프가 전화로 주문하는 행위(Claude Code 내장, 설정 불필요). MCP Server는 배달 앱(외부 서비스와 Claude 사이의 중개자, 설치 필요). Tools는 배달 앱의 메뉴들(메시지 읽기, 파일 생성 같은 구체적 기능).
- 2번: (a) 불필요 — 로컬 파일 작업. (b) 필요 — Slack 데이터 접근. (c) 불필요 — 로컬 파일 작업. (d) 필요 — Notion 데이터 접근. (e) 필요 — Google Sheets 데이터 접근.
- 3번: API 토큰은 비밀번호와 같음. .env에 저장하면 코드와 분리되고, .gitignore에 추가하면 GitHub에 실수로 올라가지 않음. 토큰이 유출되면 누구나 내 Slack/Notion/GitHub에 접근할 수 있어서 심각한 보안 문제가 됨.
- 4번: 간단한 읽기 명령으로 연결이 정상 동작하는지 테스트해야 함. 동작 확인 없이 복잡한 작업을 시도하면 "설정 문제"와 "사용법 문제"를 구분하기 어려움. 항상 단순한 것부터 확인하고 복잡한 것으로 나아가야 함.
- 5번: GitHub부터 시작하는 것을 추천. gh CLI가 이미 설치되어 있으면 설정이 가장 간단하고, 이슈/PR 관리를 바로 활용할 수 있음. 성공 경험을 빨리 얻는 것이 MCP 학습 동기를 유지하는 데 중요.
